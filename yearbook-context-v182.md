## ✅ DONE — Два фикса из теста «Аква меч»

### ФИКС 1 — портрет/имя/цитата не привязывались в личном разделе (БАГ)

**Симптом.** В тесте «Аква меч» личный раздел собирался, фото с друзьями
вставлялись, но портрет и имя ученика — нет (фрейм пустой).

**Причина.** Биндер одиночного ученика `bindSingleStudent` в
`lib/rule-engine/sections/students.ts` матчил метки портрета/имени/цитаты
ТОЛЬКО без номера (`studentportrait`, `studentname`, `studentquote`).
А мастера E-Universal/E-Standard «Аква меч» именуют их С номером
(`studentportrait_1`, `studentname_1`, `studentquote_1`). Точное сравнение
не проходило → в data этих ключей не было → пусто. Фото с друзьями работали,
потому что искались regex'ом с номером.

**Фикс.** В `bindSingleStudent` заменили точное сравнение на регексы с
опциональным номером:
- `/^studentportrait(_\d+)?$/` → `student.portrait`
- `/^studentname(_\d+)?$/` → `student.full_name`
- `/^studentquote(_\d+)?$/` → `student.quote`

Номер здесь — НЕ индекс ученика (на одностраничном мастере всегда один
ученик), просто часть имени слота. Это отличает биндер одиночного ученика
от **grid-биндера** (≈ строка 1135), где `studentportrait_N → students[N-1]`
и номер = индекс. Grid-биндер и friend-фото (`studentphoto_N`/`friendphoto_N`)
НЕ трогали.

**Тесты:** `sections-students-page-semantic.test.ts` — добавлен блок «ФИКС 1»:
мастер с `studentportrait_1`/`studentname_1`/`studentquote_1` (1 ученик) →
привязались; мастер без номера → регресс ок. Grid 1..12 покрыт прежними
grid-тестами.

### ФИКС 2 — человекочитаемые подписи мастеров в пикерах

**Симптом.** В пикере общего раздела заголовки групп человеческие, а карточка
подписана техимем `J-Collage-4` / `J-Sixth-6` — непонятно партнёру.

**Фикс.** Новая чистая функция `humanMasterLabel(master)` в
`lib/album-builder/master-label.ts` — строит подпись по составу слотов:

| Слоты | Подпись |
|---|---|
| `classphotoframe` | Общее фото класса (1 фото на страницу) |
| `halfphoto`×N | Половина класса (N фото на страницу) |
| `quarterphoto`×N | Четверть класса (N фото на страницу) |
| `sixthphoto`×N | 1/6 класса (N фото на страницу) |
| `collagephoto`×N | Коллаж — N фото на страницу |
| `spreadphoto` | Фото на весь разворот |
| прочее / личный / учительский | техимя (fallback) |

Приоритет совпадает с `classifyMaster` в JMasterPicker (spread → sixth →
collage → quarter≥2 → half≥2 → full → fallback).

**Где применено** (подпись = человекочитаемо, техимя = мелкий серый
подзаголовок, который скрывается если совпадает с подписью):
- `app/super/presets/_components/JMasterPicker.tsx` (пикер общего раздела)
- `app/super/presets/_components/CommonRequiredPagesEditor.tsx` (список
  страниц общего раздела — заменили прежний `summary`)
- `app/app/_components/TemplatePickerModal.tsx` (пикер мастеров в редакторе
  заказа, тоже партнёрский)

`app/super/master-catalog/page.tsx` НЕ трогали — там уже своя система
`display_label` (инлайн-редактор) + техимя серым; это супер-админский каталог.

**Раздел↔мастер.** Фильтр `isCommonMaster` в JMasterPicker уже отсекает
student/teacher мастера (по `NON_COMMON_PAGE_ROLES` + префиксам
`studentportrait_`/`studentname_`/`teacherphoto_`/`teachername_`/`headteacher*`).
Не меняли.

**Что НЕ делали:** техимена `J-Collage-4` и т.п. в БД/IDML НЕ переименовывали —
это системные имена, меняли только отображаемую подпись в UI.

**Тесты:** `lib/album-builder/__tests__/master-label.test.ts` — по типу слота
ожидаемая строка, `J-Collage-3/5/6` → «Коллаж — 3/5/6 фото на страницу»,
fallback на техимя для student-мастера.
