# Ветка + preview перед main (yearbook-v2)

## Зачем

У нас `main` = прод с авто-деплоем Vercel. Чтобы Сергей мог посмотреть
изменение ВЖИВУЮ до того, как его увидят партнёры/родители, видимые
изменения катятся через feature-ветку → Vercel даёт временную preview-ссылку
→ после ОК мёрж в main.

## Когда через ветку (обязательно)

- Видимые/поведенческие: вёрстка, кнопки, панели, экраны, канвас редактора,
  кабинет партнёра, родительский вход
- Всё, что Сергей захочет «посмотреть глазами»

## Когда можно прямо в main

- Мелкие/внутренние правки: рефакторинг, переименования, конфиги, доки —
  то, что глазами всё равно не проверишь. В спорных случаях — спросить.

## Процесс

### 1. Завести ветку под задачу

```bash
git checkout -b feat/краткое-описание   # или fix/ , chore/ , refactor/
```

### 2. Сделать изменение + 3 проверки

`npx vitest run` / `npx tsc --noEmit` / `npx next build` зелёные.

### 3. Закоммитить и запушить ветку

Наш формат коммитов: русский, короткий, с этапом
(`feat обложка.3: ...`). Co-Authored-By Claude Opus.

```bash
git add <файлы> && git commit -m "..." && git push -u origin <branch>
```

### 4. Дать Сергею preview-ссылку

Vercel автоматически собирает preview на любую ветку кроме main и выдаёт
временную ссылку. Передать её Сергею, сказать какой сценарий смотреть.

### 5. Дождаться «ок»

Если не так — править в этой же ветке (к пользователям ничего не утекло).

### 6. Мёрж в main

После ОК — влить в main (push в main → Vercel катит на прод). Перед
мерджем уместен навык `pre-deploy-check`.

## Важный нюанс — база общая

Preview-сайт смотрит в БОЕВУЮ базу Supabase. Для «как выглядит / удобно ли
нажимать» — безопасно. Но изменения, которые ПИШУТ/УДАЛЯЮТ данные
(пересборка альбома, перенос заказов, миграции) — на preview трогают
реальные данные партнёров. Полноценная тестовая база — это будущий Вариант 1
(см. память project_dev_environment_strategy), к нему придём с ростом числа
реальных пользователей.

## Анти-паттерны

- Гонять через ветку тривиальную внутреннюю правку (лишняя морока)
- Считать preview безопасной песочницей для записи в БД
- Мёрж в main без взгляда Сергея на preview (для видимых изменений)
