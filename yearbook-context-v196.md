## Что делали

ТЗ «смена фона и скрытие элементов в редакторе обложки». Дали менеджеру из
редактора обложек (`/app/album/[id]/cover`) две новые операции — без новой
миграции, всё на уже существующих `cover_edits` (две глубины: шаблонно на тип /
поштучно на ученика) и существующем рендере обложки.

## ✅ DONE (миграции НЕТ)

### 1. Смена фона (служебный ключ `__bg__` в cover_edits.data)
- **lib/cover/editor-merge.ts**: `resolveCoverBackground(data, masterBg)` +
  константы `COVER_BG_KEY='__bg__'`, `COVER_BG_NONE='none'`. Правила:
  `__bg__`=URL → этот фон; `'none'`/`''` → явно без фона; ключа нет / `null` →
  фон мастера (`covers.background_url`).
- **lib/cover/preview-svg.ts**: `renderCoverPreviewSvg` теперь сам учитывает
  `__bg__` (через resolveCoverBackground) → миниатюры ленты и превью Обзора
  показывают правку фона.
- **app/app/_components/CoverCanvas.tsx**: `backgroundUrl` = resolveCoverBackground
  (data, master.background_url) — холст редактора и «Вид» применяют фон.
- **lib/cover/load-editor.ts**: новое поле `available_backgrounds` — фоны обложек
  того же дизайна (`covers.background_url` где `template_set_id` = дизайн заказа)
  + фоны мастеров, уже назначенных на обложки. Уникальны по URL.
- **Загрузка нового фона** (обход 413): новый action `cover_bg_sign` в
  `/api/tenant` (assertAlbumAccess) подписывает заливку прямо в публичный bucket
  `template-backgrounds` под `album-covers/<albumId>/<uuid>.<ext>` и возвращает
  публичный URL. Клиент — `app/app/album/[id]/cover/coverBgUpload.ts`
  (`uploadAlbumCoverBackground`). Эталон обложки в библиотеке НЕ трогаем — URL
  кладётся в `cover_edits.__bg__`.
- **UI**: `app/app/_components/CoverBackgroundPanel.tsx` (кнопка «Фон» в шапке).
  Плитки: «Фон дизайна» (вернуть мастер) / «Без фона» / доступные фоны /
  загрузить новый. Для портретной (ученик) по умолчанию поштучно; чекбокс
  «применить ко всем такого типа» → шаблонно на тип (поштучный фон ученика
  снимаем, чтобы стало единым).

### 2. Ручное скрытие элементов (служебный ключ `__hidden__<label>`)
- Механика скрытия на холсте УЖЕ была (`hiddenOverridesFromData` →
  `placeholderOverrides` → AlbumSpreadCanvas прячет любой плейсхолдер).
- **preview-svg.ts**: добавлено сокрытие `__hidden__<label>` в SVG (миниатюры +
  Обзор) — гасит даже заполненный элемент, отдельно от авто-скрытия пустых
  (`hide_empty_slots` оставлен как есть).
- **UI**: `app/app/_components/CoverVisibilityPanel.tsx` (кнопка «Элементы»).
  Списки «На обложке» (глазик → Скрыть) и «Скрытые (N)» (→ Показать). Контекст —
  как у фона (поштучно/шаблонно + чекбокс «ко всем такого типа»). Имена
  элементов — `app/app/album/[id]/cover/elementLabels.ts`.

### 3. Превью Обзора (lib/cover/preview-album.ts)
- `buildAlbumCoverPreviews` теперь подмешивает `cover_edits` (через
  `mergeCoverEditsInto`) перед рендером — Обзор заказа совпадает с редактором
  (фон, скрытие, а также кроп/тексты, которых раньше в Обзоре не было).
