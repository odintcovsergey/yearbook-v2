# Формат типографии: safe в адаптации + сверка пикселей

Две точечные доработки поверх существующего движка адаптации под формат
(`lib/format-adapt`, типографская выгрузка `action=export_typography`). Движок
адаптации (uniform-scale + центрирование внутри одного `format_family`) НЕ
переписывался — только две вставки.

## 1. safe в work-зону (feat `b6456db`)

**Было:** `safe_mm` формата читался только в админке, в рендере/адаптации не
использовался (мёртвое поле). Масштаб считался от `work_w/h_mm` (а у «Булгака»
work=0 → фолбэк на размер страницы, без отступа от края).

**Стало:** `resolveWorkZone(pageMm, safeMm, explicitWorkMm)` (`lib/format-adapt/index.ts`):

```
1) явный work>0 → он (типография задала рабочую зону вручную);
2) work=0/пусто и safe>0 → page − 2×safe (контент отступает от края на safe);
3) оба 0 → page (прежнее поведение, без регрессии).
```

Применён в `adaptTemplateToFormat` (мастер) и `adaptCoverToFormat` (обложка).
Центрирование не менялось: work = page − 2×safe симметрична, центр work = центр
страницы → контент уже кладётся в центр рабочей зоны с полями ≥ safe.

**Семантика `work_w_mm`/`work_h_mm` зафиксирована: НА СТРАНИЦУ (per-page)** — как и
`page_w/h_mm`; масштаб делится на размер одной страницы. ⚠️ В seed «Фабрики
Фотокниги» `work_w_mm=410` при `page=210` похоже на разворот, а не страницу —
**ОТДЕЛЬНАЯ задача проверить seed** (формулу не трогаем, чиним понимание поля).
safe-путь безопасен (page − 2×safe всегда per-page).

**Изменение поведения (сознательное):** у форматов с `safe>0` (все «Булгак»,
«Окей Книга») контент станет меньше и с отступом от края. Затрагивает любую
типографскую выгрузку с такими форматами.

## 2. Сверка пикселей (feat в `dc2337a`, helper `lib/pdf-export/format-px-check.ts`)

**Расчёт размера файла остаётся мм×dpi** (px НЕ источник истины). Добавлена
СВЕРКА: совпадает ли размер файла-разворота, который мы реально отдаём, с
эталоном px типографии (`PrinterFormat.spread_*_px`).

`checkFormatPx(...)` → `PdfWarning | null`. Разворот = 2 страницы + вылеты:
```
expected_w_px = round((2×page_w_mm + 2×bleed_mm) × dpi / 25.4)
expected_h_px = round((page_h_mm + 2×bleed_mm) × dpi / 25.4)
```
Расхождение с эталоном > **±1px** → warning (код `format_px_mismatch`) в
существующий канал `PdfWarning` (тот же, что adapt/font-warnings →
`album_exports`). Текст с числами: «Размер не совпадает с эталоном типографии:
разворот ожидается {эталон}px, а по заданным мм и {dpi} dpi выходит
{вычислено}px. Проверьте размеры в мм или DPI».

**Пропуски (молча, без warning):** формат не выбран / адаптация не применилась /
режим приёма НЕ `spread` (per-page эталона у формата нет — можно добавить
позже) / эталон px = 0 (как у «Булгака»). DPI берётся из вызова (не хардкод 300).

На реальных данных «Булгак 22×30» (page 220×300, bleed 5, эталон 5339×3602) сверка
дала бы: ожидается 5339×3602px, выходит 5315×3661px → **ловит заглушечный px**
(он скопирован от другого дизайна, не от этого формата).

## Границы
- Только типографский путь (`action=export_typography`). Цельный PDF
  (`action=export`) формат не применяет и НЕ трогается — печать идёт через
  типографскую выгрузку.
- Seed «Фабрики» (work=410) — НЕ трогали, отдельная задача (проверить: разворот
  там или страница).
