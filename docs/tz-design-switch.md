# Смена дизайна в редакторе (V1) — прогресс

Кнопка «Сменить дизайн» в редакторе альбома: выбрать другой дизайн → проверить
совместимость → если ОК, переложить каждый разворот на мастера нового дизайна,
**сохранив контент и ручные правки (кропы, тексты)**, а оформление (шрифты, цвета,
рамки, фоны, расположение слотов) взять из нового дизайна. Несовместимо — понятный
отказ.

Подход — **remap (per-spread), НЕ rebuild** (rebuild затирает ручные правки;
remap их сохраняет). Распределение учеников НЕ пересчитываем. Обложки в V1 не
трогаем.

## Что переносим / что из нового дизайна
- Переносим: контент `data[<label>]` (тексты, фото) + кропы
  `__scale__/__offset__/__rotate__`.
- Из нового дизайна: стиль текста (`__fontSize__/__color__/__font__/__halign__/
  __valign__` — НЕ переносим), рамки, фоны, декор, расположение слотов.
- Балансировка старого мастера (`__hidden__/__pos__`) отбрасывается.

## Этапы
- **Этап 1 — coverCheck (✅ ГОТОВ).** Проверка совместимости по варианту **B** —
  по ТЕКУЩИМ разворотам альбома (а не пере-прогоном движка).
  - `lib/design-switch/compatibility.ts` — чистая логика:
    `findReplacementMaster` (подбор мастера нового дизайна по page_role +
    slot_capacity; **переиспользуется remap'ом на Этапе 2**), `describeSpreadType`,
    `checkCoverage`.
  - `lib/design-switch/cover-check.ts` — загрузчик (читает `album_layouts` +
    мастера двух дизайнов через `loadTemplateSetById`).
  - V1-упрощения: «потребность» = номинальная `slot_capacity` текущего мастера;
    `photos_friend` — мягкая ёмкость (не блокирует, как и движок); роли строго
    в рамках семейства (генерик `student_grid` ⇒ замена для left/right);
    `applies_to_configs` не фильтруем.
  - Тексты отказа — человеческим языком, правило: **называть и нужное (5), и
    максимум, что есть (4)**. Пример (живой прогон «Тест 24» → «Белый»): «…в вашем
    альбоме есть страница-коллаж из 5 фотографий, а в этом дизайне максимум 4
    фотографии в коллаже…». Если в новом дизайне типа нет вовсе → «таких страниц
    нет». Несколько типов → маркированный список.
  - НЕ подключено к API/UI (чистый модуль + 17 тестов). Прогнано на проде
    скриптом: текущий дизайн → OK; «Белый» → отказ по 5-местному коллажу (у
    «Белого» только 4-местный) — отказ настоящий.

- **Этап 2 — remap на весь альбом (✅ ГОТОВ, без UI).**
  - `remapData` (lib/template-replace) расширен аддитивным параметром
    `contentKeyPrefixes` (дефолт = прежний список → смена мастера через
    TemplatePickerModal не изменилась).
  - `lib/design-switch/remap.ts` — чистое ядро `remapAlbumSpreads`: для каждого
    разворота `findReplacementMaster` → `remapData` с кроп-онли списком. Список
    переноса `DESIGN_SWITCH_CARRY_PREFIXES = __scale__/__offset__/__rotate__/
    __hidden__`. Решение: **`__hidden__` ПЕРЕНОСИМ** (скрытие пустого слота — про
    наличие контента, не про геометрию; иначе после смены дизайна вылезают
    скрытые пустые слоты). `__pos__` и стили — НЕ переносим. Граничный случай
    (новый мастер с бОльшим числом слотов → лишние пустые не скрыты) — V1 ок.
  - `lib/design-switch/remap-album.ts` — загрузчик: guard через `coverCheck`,
    `write:false` = DRY-RUN (ничего не пишет), `write:true` пишет
    `album_layouts.spreads`+`template_set_id` и `albums.template_set_id`;
    `has_user_edits` НЕ трогает (правки сохраняются). Фото переносятся КАК ЕСТЬ
    (V1, без пере-подписи — отдельный долг).
  - Прогнан DRY-RUN на проде «Тест 24» (Акварельные→Акварельные): 8/8 разворотов,
    **lost:0**, контент+кропы целы, стили/__pos__ сброшены. ⚠️ Это проверка
    МЕХАНИКИ; перенос между РАЗНЫМИ раскладками не проверен (нет второго
    совместимого дизайна — нужен фикс «Белого»/J-Collage-5 или 3-й дизайн).
  - Запись в прод — только по явному «ок» Сергея или из UI (Этап 3).
- **Этап 3 — UI «Сменить дизайн»** (кнопка + пикер + coverCheck-гейт + применение).
- **Этап 4 — хвосты** (фоны нового дизайна, пере-подпись, граничные случаи).

## Границы (что НЕ делаем в V1)
Не через rebuild/buildAlbum; не пересчитываем распределение; не трогаем обложки;
не переносим стилевые `__*__`; не «причёсываем» соседний код.
