# Fix: согласованность цитат на combined-хвосте (РЭ.37.9 после РЭ.40)

**Дата:** 25.06.2026
**Файлы:** `lib/rule-engine/sections/students.ts`, тест `lib/rule-engine/__tests__/sections-students-grid-semantic.test.ts`
**Происхождение:** найдено аудитом кода (`docs/audit-report.md`), переменная `effectiveHasQuote` числилась «неиспользуемой».

## Что было не так

В сеточной вёрстке (`buildGridSemantic`) есть fallback: если партнёр включил цитаты под учениками, но в дизайне нет сеточного мастера с цитатами на нужный размер — движок берёт мастер **без** цитат (warning `students_quote_fallback`) и помечает `effectiveHasQuote = false`.

Смысл `effectiveHasQuote` — чтобы **хвостовой combined-мастер** тоже подбирался без цитат, иначе сетка без цитат, а хвостовая страница с цитатами = рассогласование.

- В РЭ.37.9 (коммит `7b48270`) переменная была **подключена правильно** — хвостовые мастера искались с `hasQuote: effectiveHasQuote`.
- Рефакторинг РЭ.40 (коммит `aaf4910`) заменил хвостовой поиск на `decideDistribution` + ручной цикл подбора combined-мастера. Этот цикл фильтровал кандидатов по `photos_full / has_portrait / has_name / students`, но **цитату перестал учитывать**. Два чтения `effectiveHasQuote` удалили, а две записи забыли — переменная осиротела.

Связывание цитат (`bindGridStudents`) — placeholder-driven: цитата рисуется, если у выбранного мастера есть слот `studentquote_N`. Поэтому если quote-слепой цикл выбирал combined **с** цитатами при сетке без цитат — на хвосте появлялись цитаты, которых в сетке нет.

## Когда проявлялось

Узкий, но теперь реальный путь (после выката «смены дизайна в редакторе»): фотограф переключается на неполный/кастомный дизайн, где:
1. включены цитаты,
2. нет quote-сеточного мастера на нужный размер (→ fallback),
3. есть combined-мастер **с** quote-слотами.

Тогда: страницы-сетки без цитат, а combined-хвост — с цитатами. Поломки данных нет (фото на местах), но визуальное рассогласование.

На полном `okeybook-default` (quote-сетки есть) fallback не срабатывает → в проде на дефолтном дизайне не проявлялось.

## Что сделано

Восстановлена гарантия РЭ.37.9 — **жёсткий** фильтр прямо в сборе combined-кандидатов:

```
masterHasQuote === effectiveHasQuote
```

Combined-кандидат проходит, только если его цитатность совпадает с фактической цитатностью сетки. Если совпадения нет (например, fallback без цитат, а combined только с цитатами) — combined не выбирается, и хвост уходит в базовую сетку без цитат (существующий путь `tail_padded`). Симметрично для `effectiveHasQuote=true`.

## Проверка

- Новый тест «quote fallback + combined С цитатами → хвост уходит в сетку БЕЗ цитат».
- Проверено в обе стороны: с фиксом — зелёный; без фикса падает (`expected 'id-L-Combined-Q' to be 'id-L-Grid-Page'`).
- Полный регресс движка: 1178/1178. tsc 0, next build 0.
- Существующий путь combined-хвоста без цитат (РЭ.40) не затронут (combined с `has_quote=false` при `effectiveHasQuote=false` проходит фильтр как раньше).
