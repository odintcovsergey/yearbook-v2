# ТЗ для дизайнера мастеров InDesign

> Накопительный документ. Сюда попадают **все** запросы на новые/доработанные
> мастера, которые выявляются в ходе разработки и тестирования. Каждый раз
> когда engine упирается в отсутствие нужного мастера или партнёр просит
> компоновку которой нет — добавляем сюда.
>
> **Не код-задача**, а **дизайн-задача** для Сергея и его дизайнера.
> Engine уже умеет находить мастер по `slot_capacity` через
> `findStudentGridMaster` — если мастер есть в template_set, engine его
> подхватит автоматически.

## Раздел 1 — Хвостовые мастера для grid-сеток учеников

**Контекст:** Сергей хочет чтобы при `student_distribution='auto'` хвост
не выпадал в крупный мастер другой density. Идеал — engine выбирает
**визуально гармоничный** мастер с нужной компоновкой.

### Mini (12 учеников на полной странице, сетка 4×3)

Базовый мастер: `N-Grid-Page` (12 слотов) — ✓ есть.
Combined-мастер: `N-Combined-Page` (4 студента + 1 общее фото) — ✓ есть.

**Нужны дополнительные мастера с компоновкой 4+3+3 / 4+3 / 3+3:**

| Имя | Слотов | Компоновка | Применение |
|---|---|---|---|
| `N-Grid-Page-10` | 10 | 4+3+3 (центрированные ряды 2 и 3) | N=20→[10,10]; N=30→[10,10,10] |
| `N-Grid-Page-9` | 9 | 3+3+3 | N=18→[9,9]; N=27→[9,9,9] |
| `N-Grid-Page-8` | 8 | 4+4 (без 3-й строки) или 3+3+2 (центр) | N=16→[8,8] |
| `N-Grid-Page-7` | 7 | 4+3 (центр 2-й) | N=14→[7,7] |
| `N-Grid-Page-6` | 6 | 3+3 (упрощённая) | N=18→[6,6,6] |

**Принципы для всех новых мастеров:**
- Размер портретов **тот же** что в `N-Grid-Page` (ключевое требование Сергея —
  чтобы выглядело как «недозаполненная» страница, не «выделение»)
- Шрифт имени тот же, отступ тот же
- Центрирование: если в строке нечётное число слотов — равные отступы

### Light (6 учеников на полной странице)

Базовый: `L-Grid-Page` (6 слотов) — ✓ есть.
Combined: `L-Combined-Page` (3 студента + 1 фото) — ✓ есть.

| Имя | Слотов | Компоновка | Применение |
|---|---|---|---|
| `L-Grid-Page-5` | 5 | 3+2 (центр) | N=10→[5,5] |
| `L-Grid-Page-4` | 4 | 2+2 | N=8→[4,4] |
| `L-Grid-Page-3` | 3 | 3 (одна строка) | хвосты |

## Раздел 2 — Семантика slot_capacity (важно для дизайнера)

Engine различает категории мастеров по полям `slot_capacity` в БД. При
создании мастеров в IDML-конвертере **обязательно** правильно задавайте
эти флаги:

### Student-grid мастера (для секции students)

| Тип | photos_full | has_portrait | has_name | students |
|---|---|---|---|---|
| `*-Grid-Page` (полная сетка) | 0 | true | true | N (12/6/4) |
| `*-Grid-Page-N` (хвостовая) | 0 | true | true | N (фактическое) |
| `*-Combined-Page` (хвост с фото) | 1 | true | true | N |

### Transition combo (для секции transition / симметризация)

| Тип | photos_full | has_portrait | has_name | students |
|---|---|---|---|---|
| `J-Combined-Tail-N` | 1 | (нет/false) | (нет/false) | N |

**Критически важно:** у `J-Combined-Tail-*` НЕ должно быть
`has_portrait=true` и `has_name=true` — иначе engine путает его со
student-combined и использует не по назначению (баг фикс в коммите
29b85a9).

Аналогично для student-combined (`L-Combined-Page`, `N-Combined-Page`)
`has_portrait` и `has_name` обязательно `true` — иначе engine не
подхватит мастер при поиске для combined-tail.

## Раздел 3 — Combo-мастера для симметризации (greedy режим)

**Контекст:** В режиме `greedy` с включённой галкой «симметризировать
хвост» в пресете срабатывает legacy-симметризация (РЭ.37.4). Она
ищет combo-мастер по жёсткому имени `J-Combined-Tail-N` и кладёт на
переходную страницу 1-фото-вверху + N-учеников-снизу.

**Проблема найдена 2026-05-25:** У Сергея сейчас один комплект
`J-Combined-Tail-2/3/4` нарисованный под Mini-density (12 учеников на
странице → маленькие портреты). Для Light шаблона (6 на странице →
крупные портреты) на combo получается визуальный диссонанс: ученики
на хвосте меньше учеников на остальных страницах.

**Два пути решения:**

### Путь A — отдельные combo для каждой density (дизайн)

| Имя | Слотов | Стиль (размер портретов) | Когда применяется |
|---|---|---|---|
| `L-J-Combined-Tail-2` | 2 | Light (как L-Grid-Page) | Light + greedy + симм, tail=1 |
| `L-J-Combined-Tail-3` | 3 | Light | Light + greedy + симм, tail=2 |
| `N-J-Combined-Tail-2` | 2 | Mini (как N-Grid-Page) | Mini + greedy + симм, tail=1 |
| `N-J-Combined-Tail-3` | 3 | Mini | Mini + greedy + симм, tail=2 |
| `N-J-Combined-Tail-4` | 4 | Mini | Mini + greedy + симм, tail=3 |

Требует **изменение кода** в `findComboMaster` — добавить префикс по
density.

### Путь B — использовать `*-Combined-Page` (код)

Engine научить в режиме симметризации использовать обычный
`*-Combined-Page` (`L-Combined-Page` для Light, `N-Combined-Page` для
Mini) — он уже имеет правильный стиль портретов по density.

Это **код-задача** (переделать `trySymmetrizeTail` чтобы использовал
combined-мастер найденный через `findStudentGridMaster` с `photos_full=1`,
а не `findComboMaster` по жёсткому имени).

Решение за Сергеем — что проще: нарисовать 5 новых мастеров или
переделать движок.
