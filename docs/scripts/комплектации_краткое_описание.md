# Комплектации выпускных альбомов — краткая справка

> Документ для использования в смежных проектах (SaaS `yearbook-v2` и др.).
> Описывает продуктовую линейку, которую обслуживает скрипт автовёрстки `yearbook-indesign-autoscript`.

---

## Школьные комплектации

| # | Название | Стр./ученик | Контент страницы ученика | Объём альбома |
|---|---|---|---|---|
| 1 | **Стандарт** | 1 стр. | портрет + имя + цитата | зависит от N учеников |
| 2 | **Универсал** | 1 стр. | портрет + имя + цитата + 2 фото с друзьями | зависит от N |
| 3 | **Максимум** | 1 разворот (2 стр.) | левая: портрет/имя/цитата · правая: 4 фото с друзьями | зависит от N |
| 4 | **Медиум** | сетка 2×2 (4 на стр.) | портрет + имя + цитата | зависит от N |
| 5 | **Лайт** | адаптивная сетка | портрет + имя | **строго 12 стр.**, до 32 учеников |
| 6 | **Мини** | адаптивная сетка | портрет + имя | до 36 учеников, 6–7 стр. |

**Адаптивная сетка Лайт** (4 базовые страницы):
≤8 → 2×1, 9–12 → 3×1, 13–16 → 2×2, 17–24 → 3×2; 25+ → overflow на доп. страницы.

**Адаптивная сетка Мини** (2 базовые страницы):
≤9 → 2×2, 10–12 → 2×3, 13–18 → 3×3, 19–36 → 4×3.

## Дошкольная комплектация

| # | Название | Структура |
|---|---|---|
| 7 | **Индивидуальный** (детский сад) | 1 разворот учителей + 1 личный разворот ребёнка (уникальный) + N разворотов виньеток (как Мини) + общий раздел. Один документ на группу, разбивка при экспорте PDF. |

---

## Дополнительный продукт

**Трюмо (фотопапка)** — 2 документа:

- **Общий** (1 раз на группу): классрук + виньетки учеников
- **Персональный** (на каждого ученика): 2 стр. общих фото + персональная стр. с портретом

---

## Структура альбома (общая для всех школьных)

```
[1 разворот учителей]
[N страниц/разворотов учеников — зависит от комплектации]
[О-1: 2×полкласса или 6×⅙ класса]   ─┐
[О-2: 4×¼ класса]                    ├─ обязательные
[О-3: 6×⅙ или 1×общее фото]          ─┘
[Д-1, Д-2: дополнительные развороты] ─── опциональные
```

---

## Логика разворота с учителями (единая для школ)

Классрук = первый учитель с непустым полем `Текст` в CSV.
Предметники — все остальные учителя.

| Предметников | Левая страница | Правая страница |
|---|---|---|
| Нет классрука | Разворот не создаётся | — |
| 0 | Классрук + общее фото | 2×полкласса → 1×общее → пусто |
| 1–4 | Классрук + до 4 предм., 1 ряд | 2×полкласса → 1×общее → пусто |
| 5–8 | Классрук + до 8 предм., 2 ряда | 2×полкласса → 1×общее → пусто |
| 9 | Классрук + общее фото | Сетка 3×3 |
| 10–12 | Классрук + общее фото | Сетка 4×3, пустые скрываются |
| 13–16 | Классрук + общее фото | Сетка 4×4, пустые скрываются |
| 17–24 | Классрук + до 8 предм., 2 ряда | Сетка 4×4, пустые скрываются |

## Форматы листов

- **Layflat (плотные)** — стандарт, альбом начинается с разворота
- **Мягкие листы** — добавляется вступительная стр. 1 (`introText` + `introPhoto`);
  из обязательного раздела убирается О-3

---

## Источники фото в скрипте

- **Портреты учеников** — из CSV (колонка `Портрет_страница`)
- **Фото с друзьями** — из CSV (`Фото_друзья_1..4`)
- **Общие фото класса** — из папок `common/` рядом с проектом:
  - `full_class/`, `half_left/`, `half_right/`,
  - `quarter_left/`, `quarter_right/`,
  - `sixth_left/`, `sixth_right/`
  - Файлы берутся в алфавитном порядке, счётчик сквозной по альбому

---

## Что важно для SaaS-проекта (yearbook-v2)

CSV-экспорт должен содержать поля под все комплектации сразу.

**Текущие колонки CSV:**
- `Ученик` — ФИО
- `Текст` — цитата/пожелание ученика, у учителей — текст пожелания (признак классрука)
- `Портрет_страница` — имя файла портрета
- `Фото_друзья_1..4` — имена файлов с друзьями (для Универсал/Максимум/Индивидуальный)
- блок учителей: ФИО, должность, портрет, текст

**Принцип:** CSV для всех комплектаций **один и тот же** —
скрипт сам решает, какие колонки использовать в зависимости от выбранной
комплектации (выбор делает оператор при запуске скрипта в InDesign).

Поэтому добавлять в SaaS поле «комплектация» **необязательно** —
достаточно гарантировать наличие всех полей в экспорте.
