# Исходные Indesign-скрипты

Исходные Indesign-скрипты которые album-builder реализует в коде.
Не редактируем — это источник истины для логики автовёрстки.
Скрипты разработаны Сергеем для ручной работы в Indesign в 2024-2025.
В фазе 0 продукта B мы переносим эту логику в TypeScript album-builder.

## Файлы

- `build_album.jsx` — главный скрипт сборки альбома (ученики + учителя + общий раздел).
  Логика по комплектациям (Стандарт/Универсал/Максимум/Медиум/Лайт/Мини/Индивидуальный)
  и по soft/layflat — здесь.
- `build_cover.jsx` — сборка обложки альбома.
- `build_tryumo.jsx` — отдельный продукт «трюмо» (вне фазы 0 продукта B,
  см. memory `project_phase0_tryumo_separate_masters`).
- `комплектации_краткое_описание.md` — продуктовая спецификация комплектаций,
  снимок на 07.05.2026. Копия из `docs/templates/`.

## Как использовать в работе над album-builder

1. При планировании очередного подэтапа — открыть соответствующий блок
   `build_album.jsx` (поиск по имени мастера или по `configType`).
2. Перенести логику в TypeScript, не пытаясь сохранить ExtendScript-стиль —
   код в album-builder идиоматичный TS, без global state.
3. Если в скрипте обнаружена дыра (как с subjects ≥25) — задокументировать
   в `docs/templates/master-cleanup-tz.md` и решить как закрывать.
