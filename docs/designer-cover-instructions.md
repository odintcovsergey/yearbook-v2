# Инструкция дизайнеру: как рисовать обложку альбома (IDML)

Коротко, как система ждёт макет обложки. Это нужно, чтобы парсер правильно
разобрал полотно на зоны и метки. Пока не загружаем в прод — сначала тестовый
макет для проверки механизма.

## 1. Обложка — разворот из ДВУХ страниц (facing pages)

В InDesign обложка рисуется как обычный **разворот из 2 страниц** (facing pages):

```
┌──────────┬──────────┐
│  ЗАДНЯЯ   │ ПЕРЕДНЯЯ  │
│ (слева)   │ (справа)  │
│           │           │
│           │           │
└──────────┴──────────┘
        ↑
   здесь система сама вставит КОРЕШОК при печати
```

- **Левая** страница — задняя обложка.
- **Правая** страница — передняя обложка.
- **Корешок отдельно рисовать НЕ нужно** — система сама вставит его между
  задней и передней и посчитает ширину из толщины альбома (число листов + тип
  бумаги). Контент на корешок в этом варианте не помещается.

Имя мастера-обложки начинай с префикса **`C-`**, например `C-Cover-Portrait`,
`C-Cover-ClassPhoto`, `C-Cover-NoPhoto`. По первой букве система понимает, что
это обложка.

**Если нужен текст НА КОРЕШКЕ** (метка `spine_text`) — тогда рисуй разворот из
**3 страниц**: задняя | узкий корешок-заглушка | передняя. Систему это тоже
понимает. Для большинства обложек хватает 2 страниц.

## 2. Метки слотов (Script Label на фрейме)

Как и во внутренних мастерах, на каждый фрейм вешается Script Label. Имена:

**Передняя обложка (правая страница) — фото:**
- `cover_portrait` — рамка под портрет ученика (персональная обложка).
- `cover_common_photo` — рамка под общее фото класса.

**Передняя обложка — текстовые подписи** (каждую можно ставить отдельным фреймом,
система подставит данные из заказа):
- `cover_student_name` — ФИО выпускника (персональная — у каждого своя).
- `cover_school_name` — название учебного заведения.
- `cover_city` — город.
- `cover_year` — год выпуска.
- `cover_class` — класс / группа.
- `cover_title` — общий заголовок (если нужен свободный текст).
- `cover_subtitle` — подзаголовок (свободный).

**Корешок (средняя страница):**
- `spine_text` — текст на корешке (можно повернуть вертикально — поворот
  система учитывает).

**Задняя обложка (левая страница):**
- `back_logo` — рамка под логотип фотостудии.
- `back_contacts` — текст: контакты.
- `back_common_photo` — рамка под фото на задней обложке (опционально).
- `back_qr` — рамка под QR-код (на будущее, для «оживающих» фото).

Любой **текстовый** фрейм с такой меткой система прочитает как текст (со
шрифтом, размером, цветом, выравниванием из макета). Любой **прямоугольник /
овал** — как рамку под фото.

## 3. Декор — как везде

Привязанный декор работает теми же суффиксами, что и во внутренних мастерах:
`<метка>__under` (под слотом), `<метка>__over` (над слотом), `__fg` / `__fg_1`
(передний план поверх всего). Картинку декора встраивай прямо в IDML.

## 4. Чего пока НЕ нужно

- Не нужно вручную считать корешок, bleed, выступ, загиб — это параметры
  печати, система подставит их сама.
- Достаточно ОДНОГО тестового мастера обложки (например `C-Cover-Portrait`),
  чтобы проверить разбор. Галерею вариантов нарисуем после проверки механизма.

## Открытые вопросы к типографии (для Сергея)

Точные миллиметры (подставим в пресет печати позже, в макете не нужны):
толщина листа, прослойки 0.4 / 0.7 мм, запас на корешок, bleed, выступ обложки,
загиб на внутреннюю сторону.
