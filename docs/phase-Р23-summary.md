# Фаза РЭ.23 — Визуальный каталог мастеров

**Статус**: ✅ Закрыта
**Сроки**: 20.05.2026 (одна сессия, 5 подэтапов 23.0..23.4)
**Главная цель**: создать инфраструктуру для будущих РЭ.24/25 — автогенерация SVG-превью мастеров + admin-tool каталог для разметки человеко-читаемых названий.

## Что было до фазы

После РЭ.22 engine работал через семантические теги (`page_role + slot_capacity`), но **визуально** партнёр и сам OkeyBook-админ не могли увидеть свою библиотеку мастеров. Чтобы понять «какой layout доступен под N учеников» — нужно было открыть InDesign-файл или смотреть placeholders в БД.

Это блокировало следующие фазы:
- **РЭ.24** (готовые пресеты) — внутри пресета должен быть визуальный анонс
- **РЭ.25** (конструктор «галерея вариантов») — без превью галерея бессмысленна

## Что сделано

### 1. Поле `display_label` в БД

Миграция `2026-05-20-spread-templates-display-label.sql`: добавлена колонка `TEXT NULL` для человеко-читаемого названия мастера. NULL допустим — UI делает fallback на `name`.

### 2. Чистая функция `renderPreviewSvg`

`lib/album-builder/render-preview-svg.ts` — генерирует SVG-схему мастера из placeholder-координат:
- Серые прямоугольники для фото-слотов
- Круги (`<circle>`) для учительских аватарок (`is_circle=true`)
- Серые линии 80% ширины для текст-слотов
- Контур страницы + правильный aspectRatio
- Для разворотных мастеров (`is_spread=true`) — две страницы рядом + сгиб 4mm
- Поддержка `rotation_deg` через `transform="rotate(...)"`

Без зависимостей от React/Konva/DOM — тестируется в vitest. Размер SVG ~1-3 KB на мастер (< 5 KB для N-Grid из 36 placeholders).

9 unit-тестов покрывают: базовую структуру SVG, разворотные мастера, круглые vs прямоугольные фото, текст-слоты, rotation, пустые placeholders, форматирование чисел.

### 3. API endpoints

В `app/api/tenant/route.ts` два новых action'а:
- **`template_set_list_with_previews`** (GET) — возвращает все мастера template_set'а с готовыми SVG. Tenant-aware. Доступ admin/superadmin.
- **`template_set_update_display_label`** (POST) — обновляет `display_label` мастера. Tenant-aware с проверкой `template_sets.tenant_id`. Глобальный мастер → только superadmin, чужой тенант'овский → 404.

### 4. UI страница `/super/master-catalog`

`app/super/master-catalog/page.tsx` — admin-tool для разметки:
- Grid карточек мастеров (responsive 1-4 колонки)
- SVG-превью с правильным aspectRatio (разворотные шире одиночных)
- Inline-редактор `display_label` (onBlur-сохранение, оптимистичное обновление с откатом при ошибке)
- Бейджи: `page_role`, `slot_capacity` (students=4, photos_full=1, has_quote, и т.д.), «разворот» для is_spread
- Фильтр по `page_role` (кнопки-табы): «Все», «Без роли», + по одной кнопке на каждую уникальную роль с количеством
- Ссылка в верхнем меню `/super`: «📚 Каталог мастеров →»

## Что НЕ сделано в фазе (намеренно отложено)

### РЭ.23.5 — массовая разметка `display_label` (отложена)

Изначально планировалось: OkeyBook-админ заходит в каталог и проставляет название каждому мастеру.

Решение по ходу: лучше сначала **дорисовать недостающих мастеров** для стартовой линейки пресетов (мини/лайт/медиум/стандарт/универсал/максимум/индивидуальный), потом разметить всю библиотеку **в один присест**. Иначе:
- Двойная работа (размечать сейчас → дорисовывать → снова размечать)
- Имена даны в вакууме (без видения всей линейки)

Это **не блокер ни для чего** — `display_label` опциональный, UI fallback на `name` работает везде. Каталог `/super/master-catalog` остаётся доступным, разметка возможна в любой момент.

### Замечание по данным: L-Grid-12

При тестировании каталога обнаружен мастер `L-Grid-12` с подозрительными данными:
- `is_spread=true`, но placeholders сосредоточены только в левой половине разворота
- Правая страница пустая

Это не баг превью (превью точно отражает БД), это аномалия данных в БД. Когда будем пересохранять template_set из InDesign — исправить либо `is_spread` (если мастер односторонний), либо координаты placeholder'ов (если двухсторонний с правой пустой).

## Состояние после фазы

**Что работает прямо сейчас:**
- `/super/master-catalog` показывает все мастера OkeyBook с SVG-превью
- Superadmin может разметить `display_label` каждого мастера через inline-редактор
- Никаких изменений для конечного пользователя (партнёра/учителя/родителя)
- Никаких изменений в engine — engine не использует превью

**Готовность к следующим фазам:**
- `renderPreviewSvg` доступен из любого слоя приложения (импортируется как обычная функция)
- API `template_set_list_with_previews` готов отдавать массив мастеров с превью — РЭ.24 будет показывать ключевые развороты пресета, РЭ.25 — галерею вариантов

## Что дальше

**РЭ.24** — готовые пресеты для быстрого старта партнёра. Стартовая линейка Сергея (мини/лайт/медиум/стандарт/универсал/максимум/индивидуальный) уже в БД как обычные пресеты. РЭ.24 — это:
- Пометить часть из них как «рекомендованные»
- На главной странице партнёра показать карточки этих пресетов с превью ключевых разворотов (через `renderPreviewSvg` из РЭ.23)
- Клонирование пресета партнёром в свой личный список

**РЭ.25** — конструктор «галерея вариантов». Партнёр редактирует свой пресет, выбирает альтернативный layout для каждой роли из доступной библиотеки.
