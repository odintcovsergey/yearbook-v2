# Дизайн-гайд OkeyBook — редизайн системы

## Философия

Вдохновение: Notion (кабинет — чисто, плотно, эффективно) + Pixieset (отбор
фото — просторно, тепло, фото в центре). Одна ДНК, разная плотность.

Принцип: много белого пространства, мягкие скругления, спокойная типографика,
акцент через ЦВЕТ а не через рамки/линии. Чисто, легко, современно.

ВАЖНО: это РЕДИЗАЙН внешнего вида. Логику, расположение функций, работу
отбора/сборки/submit — НЕ менять. Только цвета, отступы, скругления, иконки,
типографику.

================================================================================
# ГЕОМЕТРИЯ И ПРОСТРАНСТВО
================================================================================

## Скругления (мягче чем сейчас)
- Мелкие элементы (кнопки, инпуты, пилюли): 8px
- Карточки, модалки: 12px
- Крупные блоки/фото-превью: 16px
- Пилюли статусов: 999px

## Тени (мягкие, ненавязчивые — как Pixieset)
- Карточки: 0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)
- Модалки: 0 10px 30px rgba(0,0,0,0.12)
- Hover карточек: чуть усилить тень + поднять (transform translateY(-2px))
- НЕ использовать жёсткие тёмные тени.

## Отступы (больше воздуха чем сейчас)
- Внутри карточек: 20-24px padding
- Между карточками: 16px gap
- Секции страницы: 32px между блоками
- Кабинет — плотнее (16-20px), отбор фото — просторнее (24-32px)

================================================================================
# ТИПОГРАФИКА
================================================================================

## Шрифт
Чистый гротеск без засечек. Рекомендую Inter или Manrope (есть в Google Fonts,
бесплатные, современные, отличная кириллица). Manrope чуть теплее/округлее —
ближе к дружелюбному вайбу. Inter — нейтральнее, чище. Выбрать один.

## Размеры (вес: 400 обычный, 500/600 для акцентов; не жирнее 700)
- H1 (заголовок страницы):  24px / 600
- H2 (секция):              18px / 600
- H3 (карточка):            16px / 500
- Body (основной текст):    14px / 400
- Small (подписи, мета):    13px / 400
- Tiny (статусы, бейджи):   12px / 500

## Правила
- Sentence case (не Title Case, не CAPS).
- Line-height 1.5-1.6 для текста.
- Цифры в метриках — крупно (24-28px/600).

«узнаваемые иконки» что хотел Сергей).

ВАЖНО: убрать эмодзи из рабочих интерфейсов. Сейчас в табах 🚀 (Производство),

Размер иконок: 16-20px инлайн, цвет наследуется от текста или мятный для
активных.

# ТРИ ЗОНЫ (одна ДНК, разная плотность)
================================================================================

## Зона 1 — Кабинет партнёра / админ (стиль Notion)
- Плотнее, эффективнее. Данные, таблицы, карточки заказов.
- Цветовые статусы заказов (пилюли) — с одного взгляда видно этап.
- Метрики сверху (Активных альбомов / Учеников / Завершили / Заявок) —
  чистые карточки с крупной цифрой.
- Карточки заказов: белый фон, мягкая тень, скругление 12px, статус-пилюля,
  прогресс-бар мятный, hover-подъём.
- Табы — иконки lucide + текст, активный таб подчёркнут мятным.
НАЧАТЬ РЕДИЗАЙН С ЭТОЙ ЗОНЫ (проще, быстрая видимая победа).

## Зона 2 — Редактор / выбор дизайна / шаблоны (стиль Pixieset)
- Просторнее, фото и макеты дышат.
- Превью дизайнов — крупные карточки с обложкой, мягкая тень.
- Лёгкость, минимум UI вокруг фото.

## Зона 3 — Отбор фото родителями (стиль Pixieset, максимум тепла)
- Фото — ГЛАВНЫЙ герой. Крупные превью, много воздуха.
- Тёплый, простой, для неподготовленного пользователя (часто с телефона).
- Минимум текста, крупные понятные кнопки выбора.
- Мятные акценты на выбранных фото (рамка/галочка #0D9488).
ДЕЛАТЬ ВТОРОЙ (тоньше по ощущению, требует аккуратности).

# ПРОЦЕСС РЕАЛИЗАЦИИ
================================================================================

1. ОТДЕЛЬНАЯ ВЕТКА (не main) + preview-ссылка Vercel. Обязательно.
2. По ОДНОМУ экрану. Начать с кабинета партнёра (дашборд + карточки заказов).
3. Сначала привести к новой ДНК (палитра, скругления, иконки, воздух, шрифт) —
   это 80% эффекта. Тонкую доводку — потом, не уходить в перфекционизм.
4. Смотреть preview, сравнивать со старым, править, потом мержить.
5. Логику НЕ трогать — только внешний вид.

## Подключение
- Шрифт: Google Fonts (Inter или Manrope) — добавить в проект.
- Иконки: lucide-react (уже есть).
- shadcn/ui — можно подключить для базовых компонентов (кнопки, карточки,
  бейджи, табы), он на Tailwind, ложится органично. Опционально — Claude Code
  решит нужно ли, или хватит правки существующих Tailwind-классов.
- Magic UI — НЕ в систему (только для будущего лендинга okeybook).

## Что НЕ трогать
- Логику отбора/сборки/submit/draft.
- Расположение функций (не переставлять, только перекрашивать/причёсывать).
- Родительские страницы деструктивно (отбор причёсываем аккуратно, Зона 3).
- main-ветку (всё на отдельной ветке через preview).
