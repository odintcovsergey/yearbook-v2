## Цель фазы

Партнёр заходит в `/app/templates/[designId]`, видит **два раздела**:
- **«Готовые от OkeyBook»** — пресеты-шаблоны от OkeyBook (бейдж «Глобальная»)
- **«Мои шаблоны»** — свои пресеты партнёра (бейдж «Моя»)

Создаёт новый шаблон через `+ Создать свой шаблон`, выбирая:
- Режим (page / spread / grid)
- Параметры (друзья, цитата, число в сетке)
- Структуру секций

Шаблон сразу работает — без скрытых селектов «Section Structure», без поля «Плотность портретов», без альтернативных полей «Комплектация + Тип печати» в форме альбома.

**Без винегрета. Один путь к сборке.**

## Этап Б — миграция данных (3 коммита)

### Б.1 — SQL миграция: глобальные пресеты на семантику

**Файл**: `migrations/2026-05-21-presets-to-semantic.sql`

Что делает:
1. Обнуляет `density` у всех 7 глобальных пресетов
2. Заполняет `student_layout_mode + параметры` (mapping таблица выше)
3. Заполняет `template_set_id = 08baf556-...` («Белый плотные разворотами»)
4. Дорабатывает `individual` (mode='spread', friends=4, quote=true как maximum)
5. Снимает `applies_to_configs` у E-* мастеров (UPDATE spread_templates SET applies_to_configs='{}' WHERE name LIKE 'E-%')

**Идемпотентность**: каждый UPDATE с WHERE по id, безопасно повторить.

⚠️ **Многие UPDATE'ы уже применены вручную в Supabase 21.05.2026.** Миграция приводит к согласованному финальному состоянию (тест-кейс: применяется на чистой БД и на текущей с одинаковым результатом).

**Коммит**: `feat(РЭ.30.1): миграция глобальных пресетов на семантическую модель`

### Б.2 — Fix PresetEditorModal: не записывать смешанные пресеты

**Файл**: `app/super/presets/_components/PresetEditorModal.tsx`

Сейчас на сохранении (~строка 218) записываются **оба** поля одновременно: `density` И `student_layout_mode`. Это позволяет создать смешанный пресет (как «Мой шаблон» и «Мой Мини» Сергея).

Что делаем:
- Удаляем сохранение `density` совсем. Только `student_layout_mode` + параметры.
- Удаляем сохранение `sheet_type`. Тип переплёта живёт на уровне альбома (РЭ.27) и template_set.
- UI селект «Плотность портретов» убираем (см. этап В.2 — связано).

После этого новые пресеты создаются только в семантической модели. Старые legacy-пресеты в БД остаются как были, но больше не сохраняются с новыми значениями.

**Коммит**: `feat(РЭ.30.2): PresetEditorModal — не записывать density/sheet_type`

### Б.3 — SQL миграция: очистить смешанные пресеты Сергея

**Файл**: `migrations/2026-05-21-presets-cleanup-mixed.sql`

Что делает:
```sql
-- Если у пресета есть student_layout_mode — обнуляем density.
-- Применяется к смешанным записям (blank-w7lygmuy, custom-l34kwu6p).
UPDATE presets
SET density = NULL, sheet_type = NULL
WHERE student_layout_mode IS NOT NULL
  AND (density IS NOT NULL OR sheet_type IS NOT NULL);
```

После этого все «здоровые» пресеты — только на семантической модели. Чистые legacy-пресеты (с `density` и `layout_mode=NULL`) — не трогаем.

**Коммит**: `feat(РЭ.30.3): очистка смешанных пресетов`
