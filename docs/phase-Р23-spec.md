# Фаза РЭ.23 — Визуальный каталог мастеров и автогенерация превью

**Спецификация v1.0**
**Дата:** 20.05.2026
**Статус:** утверждена 20.05.2026 (browser-чат с Сергеем).
**Зависит от:** РЭ.22 (семантический engine, page_role + slot_capacity на всех мастерах OkeyBook).
**Эстимация:** ~6 коммитов на main + 1 миграция. Половина дня работы.

---

## 1. Зачем фаза существует

После РЭ.22 engine работает через семантические теги, но **партнёр** (и сам OkeyBook-админ) всё ещё не видят свою библиотеку мастеров глазами. Чтобы понять «какой layout доступен под "4 ученика на страницу"» — нужно открыть InDesign-файл или смотреть placeholders в БД.

Это блокирует следующие фазы:
- **РЭ.24** (готовые пресеты) — внутри пресета должен быть визуальный анонс что в нём.
- **РЭ.25** (конструктор «галерея вариантов») — без превью галерея бессмысленна.

РЭ.23 — **инфраструктурная фаза**: создать механизм автоматической генерации SVG-превью из placeholder-координат мастера + страницу `/super/master-catalog` где OkeyBook-админ видит все свои мастера и размечает их человеко-читаемыми названиями («Вариант 4: четыре ученика»).

Партнёр в этой фазе ничего нового не видит. Все изменения — на стороне OkeyBook-админа (Сергея).

---

## 2. Что добавляется в БД

Одно новое поле в `spread_templates`:

```sql
ALTER TABLE spread_templates ADD COLUMN display_label TEXT NULL;
```

- Человеко-читаемое название мастера для UI («Вариант 4: четыре ученика»). Партнёр и админ видят его в галерее, не технический `name`.
- Когда NULL — UI показывает `name` как fallback.
- Превью НЕ хранится в БД. Генерируется на лету при запросе (SVG-строка). Кэшируется на клиенте.

---

## 3. SVG-превью

Простая SVG-схема — прямоугольники где расположены слоты, без реальных фото:
- Фото-слоты — серые прямоугольники (`fill: #d1d5db`)
- Текст-слоты — серая горизонтальная линия (для текстов с `auto_fit` — длиннее, иначе короче)
- Соблюдены пропорции страницы (`width_mm × height_mm`) и относительное расположение слотов
- 1 страница = 1 SVG. Двухстраничные мастера (`is_spread=true`) рендерятся как «обе страницы рядом» (см. развилку D).

Утилита `lib/album-builder/render-preview-svg.ts`. **Чистая функция**: `placeholders + page-size → SVG string`. Никаких зависимостей от React/Konva. Тестируема vitest'ом без браузера.

**Почему именно SVG (а не PNG/Konva-render):**
- Маленький размер (1-3KB на превью)
- Чёткие линии на любом масштабе
- Генерируется мгновенно, без headless-браузера
- Подходит и для каталога, и для будущего конструктора, и для писем (можно встроить в email)

## 4. UI страница `/super/master-catalog`

Новая страница в `/super`:
- Список всех мастеров template_set OkeyBook (через `template_set_list_with_previews`)
- Фильтры: по `page_role` (student_left/right/grid, teacher_left/right, intro/final, common, cover)
- Каждая карточка показывает:
  - SVG-превью (через `<div dangerouslySetInnerHTML>` или `<svg>` напрямую)
  - `name` (технический)
  - `display_label` — редактируемый inline, default fallback на `name`
  - `page_role`
  - Ключевые `slot_capacity` теги (students=4, has_quote=true и т.п.)
- Сохранение `display_label` через новый API endpoint

UI минималистичный — admin-tool, не партнёрский UX. Tailwind, без декора. По образцу существующих `/super/*` страниц.

## 5. API endpoints

В `app/api/tenant/route.ts` добавить два action'а:

**`template_set_list_with_previews`** — возвращает массив:
```ts
{
  id: string;
  name: string;
  display_label: string | null;
  page_role: PageRole | null;
  slot_capacity: SlotCapacity | null;
  is_spread: boolean;
  preview_svg: string; // готовая SVG-строка
}
```
Только для super-admin (защита через существующий middleware).

**`template_set_update_display_label`** — обновляет `display_label` мастера:
```ts
{ template_id: string, display_label: string | null }
```
Только для super-admin.

## 7. Архитектурные развилки (зафиксированы)

**A. SVG vs Canvas-PNG для превью.** ✅ **Решение: SVG** (см. §3 «почему»).

**B. Где хранить display_label.** ✅ **Решение: новое поле** `spread_templates.display_label` (Вариант 1). `audit_notes` для другого назначения.

**C. Видимость для партнёра.** ✅ **Решение: отложить** до появления первого партнёра. В РЭ.23 каталог `/super/master-catalog` показывает только мастера OkeyBook (текущего тенанта).

**D. Что в превью для двухстраничных (cover, spread).** ✅ **Решение: обе страницы рядом** в одной SVG, шириной 2× page_width + промежуток-сгиб (≈4mm). Партнёр видит весь разворот.

**E. Когда применяется фаза.** ✅ **Решение: сейчас** — превью нужны и для РЭ.24 (анонс пресета), и для РЭ.25.

## 8. Что НЕ входит в РЭ.23 (явно)

- Конструктор пресетов / галерея вариантов для партнёра — это РЭ.25.
- Готовые рекомендованные пресеты — это РЭ.24.
- Превью для готового **альбома** (со всеми вставленными фото) — это уже отдельный сценарий «предпросмотр перед сборкой», не каталог.
- Партнёрские template_set'ы — пока не существуют.
- Mass-API для дизайнера InDesign — пока не нужно (мастера загружаются скриптом из InDesign в БД уже сейчас).
- Группировка мастеров по «семьям» (например все student_grid в одну группу) — оставляем плоский список, фильтруем по page_role.

## 9. Связь с РЭ.24 и РЭ.25

**РЭ.24** (готовые пресеты для быстрого старта партнёра):
- Использует SVG-превью из РЭ.23 для анонса пресета (4-6 мини-превью ключевых разворотов).
- Использует `display_label` для отображения «Вариант 4» вместо `M-Grid-Page`.
- Готовые пресеты — стартовая линейка Сергея: мини / лайт / медиум / стандарт / универсал / максимум / индивидуальный.

**РЭ.25** (конструктор пресетов с галереей вариантов):
- Использует SVG-превью для галереи альтернативных layout'ов.
- Партнёр видит «Вариант 1 / Вариант 2 / Вариант 3» с превью под каждой ролью (cover, student_grid, teacher_left и т.д.).
- Каждый клик в галерее меняет выбранный мастер в пресете.

---

## 10. Принципы

- **Изолированность.** РЭ.23 не трогает engine, не трогает существующие пресеты, не трогает алгоритм сборки альбома. Никаких регрессий — только новая возможность.
- **Бэк-совместимость.** Колонка `display_label` опциональна. Все существующие записи без неё работают (UI fallback на `name`).
- **Простота.** SVG генерируется чистой функцией, без зависимости от Konva/React. Тестируется в vitest без браузера.
- **Готовность к РЭ.24/25.** После закрытия РЭ.23 эти фазы могут стартовать без подготовительной работы — каталог и превью готовы.
