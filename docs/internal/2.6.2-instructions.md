# Инструкция для подэтапа 2.6.2

**Цель:** заменить заглушку правой колонки редактора на полноценный
компонент `PhotoPalette` — сетка фото альбома, сгруппированная по типу
(портреты / группы / учителя / оригиналы), с поиском по filename'у и
переключателем «Показать оригиналы». Без drag-and-drop — миниатюры
просто отображаются. Drag-and-drop появится в 2.6.3.

**Зачем:** в редакторе партнёр работает с двумя сущностями: layout
(canvas слева) и палитрой фото (справа). До 2.6.2 правая колонка
показывает только текст «179 фото загружено». В 2.6.2 партнёр впервые
видит **сами фото** — миниатюры с привязкой к ученикам/учителям, с
информацией где это фото уже использовано в текущем layout. Это даёт
партнёру понимание «вот мой пул, вот что уже в макете, вот что
свободно».

**Контекст:** endpoint `GET /api/tenant?action=album_photos` (2.4)
готов и возвращает все нужные данные: `id`, `filename`, `thumb_url`,
`type`, `source`, `child_ids[]`, `teacher_ids[]`, `selection_types[]`.
Загрузка уже идёт в `LayoutEditorPage` (2.6.1) — `photos` лежит в state.
Нам нужно только сделать компонент который рендерит этот state красиво.

**Что меняется:**
- Новый файл `app/app/_components/PhotoPalette.tsx` (~250 строк)
- `app/app/album/[id]/layout/page.tsx` — две точки:
  1. Импорт `PhotoPalette`
  2. Замена заглушки `<aside>` на `<PhotoPalette layout={layout} photos={photos} />`
- `docs/internal/2.6.2-instructions.md` — этот файл

**Что НЕ трогаем:**
- `AlbumSpreadCanvas.tsx` — без изменений
- `LayoutPreviewStrip.tsx` — без изменений
- API endpoint'ы — все используются read-only
- Миграции БД — нет
- `package.json` — никаких новых зависимостей

## Принципиальные решения

### Дублирование типа `AlbumPhoto` в новом файле

Тип `AlbumPhoto` определён локально в `app/app/album/[id]/layout/page.tsx:25`.
Чтобы избежать циклических импортов и не выносить тип в общий module,
дублируем его в `PhotoPalette.tsx`. Это микро-дублирование в 14 строк
оправдано тем же что и для `LayoutData` — снижает связность.

Если в будущем понадобится использовать `AlbumPhoto` ещё в одном месте
(например, в 2.6.3 при drag handlers) — вынесем в общий `types.ts`.

### Группировка: `source` затем `type`

Логика секций:
- Если `showOriginals=false` (по умолчанию) → показываем только
  `source==='selections'`, в трёх секциях: «Портреты», «Группы»,
  «Учителя»
- Если `showOriginals=true` → дополнительно секция «Оригиналы»
  (только `source==='originals'`) в конце

Секции с пустым содержимым **не показываем**. Если в альбоме нет
group-фото — секция «Группы» вообще не появляется.

**Пустой массив originals**: если `originals.length === 0`, не показываем
переключатель «Показать оригиналы» — он бесполезный.

### Бейдж «✓ в развороте N»

Для каждого фото проверяем: используется ли оно в текущем layout?
Используется = в каком-то spread.data есть value === photo.url.

```typescript
function findUsage(photo: AlbumPhoto, spreads: SpreadInstance[]): number[] {
  const usedIn: number[] = []
  for (const spread of spreads) {
    if (Object.values(spread.data).some((v) => v === photo.url)) {
      usedIn.push(spread.spread_index)
    }
  }
  return usedIn
}
```

Сложность O(spreads × placeholders) для каждого фото. Для 500 фото ×
11 spread'ов × 30 placeholder'ов = ~165k operations при render.
**Считаем единожды через `useMemo`** при изменении photos+spreads,
кешируем в `Map<photoId, number[]>`.

```typescript
const usageMap = useMemo(() => {
  const m = new Map<string, number[]>()
  for (const photo of photos) {
    m.set(photo.id, findUsage(photo, spreads))
  }
  return m
}, [photos, spreads])
```

Бейдж — небольшая pill-метка в верхнем правом углу миниатюры:
- 1 использование: «✓ {N+1}» (например «✓ 3»)
- Несколько: «✓ {N+1}, {M+1}» или «✓ ×3» если 3+

**`spread_index + 1`**: помним что в БД `spread_index` начинается с 0,
для UI — с 1 (как мы делали в 2.3.1 для подписей миниатюр).

### Поиск по filename

Простой клиентский фильтр без debounce:

```typescript
const filtered = photos.filter((p) =>
  p.filename.toLowerCase().includes(query.toLowerCase())
)
```

Без debounce потому что для 500 фото фильтр занимает <1мс. Если
profiling в будущем покажет проблемы (5000+ фото) — добавим.

Поиск ищет только по filename, не по child_id / teacher_id (это
уровень UI который мы пока не показываем — у партнёра только filenames
типа «DSC_5570.jpg»).

### Виртуализация — НЕ делаем

Для 500 фото обычный grid без virtualization работает нормально
(~150ms render). Виртуализация (`react-window`) — это новая зависимость
+ сложность реализации drag'а в виртуальном списке. Откладываем
в backlog.

При жалобах на производительность (1000+ фото в реальных альбомах) —
вернёмся.

### Размер миниатюр

Сетка `grid-cols-3` (3 в ряд). При ширине палитры 30% × 1440px =
432px минус padding 32px = 400px / 3 = ~130px на миниатюру. Плюс
gap 8px = ~120px чистая ширина миниатюры. Высота пропорциональная
(aspect-ratio: 3/4 = вертикальный портрет).

```tsx
<div className="aspect-[3/4] bg-gray-100 rounded overflow-hidden ...">
  <img src={photo.thumb_url} alt={photo.filename} className="w-full h-full object-cover" />
</div>
```

`object-cover` даёт same-effect что `getCoverCrop` в Konva — фото
заполняет область, лишнее обрезается.

### Hover-state

При hover на миниатюру — лёгкая обводка `ring-2 ring-blue-300` +
overlay с filename в нижней части. В 2.6.2 это не функционал, просто
визуальный feedback. В 2.6.3 будет cursor-grab при hover.

### Загрузка изображений: lazy

У `<img>` атрибут `loading="lazy"` — браузер сам отложит загрузку
вне viewport'а. Бесплатно, без зависимостей.

### Названия секций — на русском, без emoji

Спека phase-2-spec §4.6 использует emoji «📷 Портреты». Я **отступаю
от spec** и убираю emoji. Причины:
- Emoji в title — заигрывание, не профессионально
- Партнёры — фотографы, не tech savvy; emoji в UI могут смотреться
  вычурно
- Без emoji проще копировать текст в support-сообщения

Просто: «Портреты», «Группы», «Учителя», «Оригиналы».

### Чекбокс «Показать оригиналы»: нативный input

Без кастомного UI — нативный `<input type="checkbox">` с label.
Tailwind классы `accent-blue-600` для тонировки галочки.

```tsx
<label className="flex items-center gap-2 text-xs text-gray-700 cursor-pointer">
  <input
    type="checkbox"
    checked={showOriginals}
    onChange={(e) => setShowOriginals(e.target.checked)}
    className="accent-blue-600"
  />
  Показать оригиналы ({originalsCount})
</label>
```

Счётчик `({originalsCount})` помогает партнёру понять «много ли там
вообще». Если 0 — переключатель скрыт целиком.

### Sticky header палитры

Поиск + переключатель должны оставаться видимыми при скролле списка
фото вниз. Делаем sticky через `position: sticky` + `top: 0`:

```tsx
<div className="sticky top-0 bg-white pb-2 z-10 border-b border-gray-100">
  <input ... placeholder="Поиск..." />
  <label>Показать оригиналы</label>
</div>
```

Sticky внутри `<aside>` с `overflow-y-auto` работает корректно — это
стандартный паттерн.

### Шаг 1. Создать компонент `PhotoPalette`

Файл: `app/app/_components/PhotoPalette.tsx`

```tsx
'use client'

import { useMemo, useState } from 'react'
import type { SpreadInstance } from '@/lib/album-builder/types'

// ─── Тип AlbumPhoto (соответствует /api/tenant?action=album_photos) ──────
// Дубликат из app/app/album/[id]/layout/page.tsx — оба файла должны
// держать тип в синхроне. Альтернатива: вынести в общий types.ts при
// появлении третьего consumer'а.
type AlbumPhoto = {
  id: string
  filename: string
  storage_path: string
  thumb_path: string | null
  type: 'portrait' | 'group' | 'teacher' | null
  source: 'selections' | 'originals'
  child_ids: string[]
  teacher_ids: string[]
  selection_types: string[]
  url: string
  thumb_url: string
  created_at: string
}

type Props = {
  spreads: SpreadInstance[]
  photos: AlbumPhoto[]
}

// ─── Найти все spread_index'ы где фото используется ──────────────────────
function findUsage(
  photo: AlbumPhoto,
  spreads: SpreadInstance[],
): number[] {
  const used: number[] = []
  for (const spread of spreads) {
    if (Object.values(spread.data).some((v) => v === photo.url)) {
      used.push(spread.spread_index)
    }
  }
  return used
}

// ─── Бейдж использования ──────────────────────────────────────────────────
function UsageBadge({ usage }: { usage: number[] }) {
  if (usage.length === 0) return null
  const text = usage.length === 1
    ? `✓ ${usage[0] + 1}`
    : usage.length === 2
    ? `✓ ${usage[0] + 1}, ${usage[1] + 1}`
    : `✓ ×${usage.length}`
  return (
    <div className="absolute top-1 right-1 px-1.5 py-0.5 rounded text-[10px] font-medium bg-green-600 text-white shadow">
      {text}
    </div>
  )
}

// ─── Миниатюра одного фото ────────────────────────────────────────────────
function PhotoTile({
  photo,
  usage,
}: {
  photo: AlbumPhoto
  usage: number[]
}) {
  return (
    <div
      className="relative aspect-[3/4] bg-gray-100 rounded overflow-hidden border border-gray-200 hover:ring-2 hover:ring-blue-300 transition cursor-default"
      title={photo.filename}
    >
      <img
        src={photo.thumb_url}
        alt={photo.filename}
        loading="lazy"
        className="w-full h-full object-cover"
        onError={(e) => {
          // Битая картинка → серая заливка с filename как fallback
          ;(e.target as HTMLImageElement).style.display = 'none'
        }}
      />
      <UsageBadge usage={usage} />
      {/* Filename overlay при hover */}
      <div className="absolute bottom-0 left-0 right-0 px-1.5 py-1 text-[10px] bg-black/60 text-white opacity-0 hover:opacity-100 transition truncate">
        {photo.filename}
      </div>
    </div>
  )
}

// ─── Секция (заголовок + grid) ────────────────────────────────────────────
function Section({
  title,
  photos,
  usageMap,
}: {
  title: string
  photos: AlbumPhoto[]
  usageMap: Map<string, number[]>
}) {
  if (photos.length === 0) return null
  return (
    <div className="mb-4">
      <h3 className="text-xs font-semibold text-gray-700 mb-2 sticky top-[88px] bg-white py-1 z-[5]">
        {title} ({photos.length})
      </h3>
      <div className="grid grid-cols-3 gap-2">
        {photos.map((p) => (
          <PhotoTile key={p.id} photo={p} usage={usageMap.get(p.id) ?? []} />
        ))}
      </div>
    </div>
  )
}

// ═════════════════════════════════════════════════════════════════════════
// PhotoPalette — правая панель редактора со всеми фото альбома (2.6.2).
//
// В подэтапе 2.6.2 — read-only отображение: миниатюры, поиск, фильтр
// по originals, бейджи использования. Drag-and-drop появится в 2.6.3.
// ═════════════════════════════════════════════════════════════════════════
export default function PhotoPalette({ spreads, photos }: Props) {
  const [query, setQuery] = useState('')
  const [showOriginals, setShowOriginals] = useState(false)

  // Карта использования для O(1) lookup'а в каждой миниатюре.
  const usageMap = useMemo(() => {
    const m = new Map<string, number[]>()
    for (const p of photos) {
      m.set(p.id, findUsage(p, spreads))
    }
    return m
  }, [photos, spreads])

  // Фильтр по запросу (clientside).
  const filtered = useMemo(() => {
    const q = query.toLowerCase().trim()
    if (!q) return photos
    return photos.filter((p) => p.filename.toLowerCase().includes(q))
  }, [photos, query])

  // Группировка по source/type.
  const portraits = filtered.filter(
    (p) => p.source === 'selections' && p.type === 'portrait',
  )
  const groups = filtered.filter(
    (p) => p.source === 'selections' && p.type === 'group',
  )
  const teachers = filtered.filter(
    (p) => p.source === 'selections' && p.type === 'teacher',
  )
  const originals = filtered.filter((p) => p.source === 'originals')

  // Счётчик originals по полному пулу (не filtered) — для лейбла checkbox'а.
  const totalOriginals = photos.filter((p) => p.source === 'originals').length

  return (
    <aside className="w-[30%] min-w-[300px] max-w-[440px] bg-white border-l border-gray-200 overflow-y-auto">
      <div className="p-3 sticky top-0 bg-white z-10 border-b border-gray-100">
        <h2 className="text-sm font-semibold text-gray-900 mb-2">
          Палитра фото
        </h2>
        <input
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Поиск по имени файла…"
          className="w-full px-2 py-1.5 text-xs border border-gray-300 rounded focus:outline-none focus:ring-2 focus:ring-blue-200"
        />
        {totalOriginals > 0 && (
          <label className="flex items-center gap-2 text-xs text-gray-700 cursor-pointer mt-2">
            <input
              type="checkbox"
              checked={showOriginals}
              onChange={(e) => setShowOriginals(e.target.checked)}
              className="accent-blue-600"
            />
            Показать оригиналы ({totalOriginals})
          </label>
        )}
      </div>

      <div className="p-3">
        {filtered.length === 0 ? (
          <p className="text-xs text-gray-500 text-center py-8">
            {query ? 'Ничего не найдено' : 'Нет фото в альбоме'}
          </p>
        ) : (
          <>
            <Section title="Портреты" photos={portraits} usageMap={usageMap} />
            <Section title="Группы" photos={groups} usageMap={usageMap} />
            <Section title="Учителя" photos={teachers} usageMap={usageMap} />
            {showOriginals && (
              <Section
                title="Оригиналы"
                photos={originals}
                usageMap={usageMap}
              />
            )}
          </>
        )}
      </div>
    </aside>
  )
}
```

**Замечания по реализации:**

1. **`'use client'`** — обязательно, использует useState + useMemo.

2. **`<img>` нативный, не Next.js `<Image>`** — Next.js `<Image>` требует
   `width`/`height` пропов и доменов в `next.config.js`. Native
   `<img>` работает out-of-the-box, для миниатюр в палитре достаточно.

3. **`onError` на `<img>`** — если фото отдаст 404 (как на smoke-альбоме),
   скрываем картинку через `style.display='none'`. Серая заливка
   `bg-gray-100` от родительского div'а останется видна — partner
   увидит «слот без фото». Filename в hover-overlay даст ему понимание.

4. **`sticky top-[88px]` для заголовков секций** — 88px это высота
   sticky-блока сверху (заголовок «Палитра фото» + поиск + checkbox).
   Магическое число, но альтернатива (CSS-калькуляция через
   variables) — overkill. Если в будущем header растёт — поправим.

5. **`aspect-[3/4]`** — Tailwind 3.x поддерживает arbitrary aspect
   ratios. Если в проекте Tailwind 2.x — заменить на `aspect-ratio:3/4`
   inline или фиксированной высотой.

## Применение и верификация в проде

### Порядок

Миграций нет. После push Vercel задеплоит за ~30-60 секунд.

### Проверка визуально

После деплоя открой `https://yearbook-v2.vercel.app/app/album/3fccf1dc-3e3d-402b-8c0c-7036e3496418/layout`:

**Чек-лист:**

1. **Правая панель** — теперь не «179 фото загружено», а:
   - Заголовок «Палитра фото»
   - Поле поиска «Поиск по имени файла…»
   - **Не должно быть** переключателя «Показать оригиналы» (originals=0
     для smoke-альбома)
   - Секции «Портреты (163)», «Учителя (16)» — с миниатюрами по 3 в ряд
   - Секции «Группы» **не должно быть** (group_photos=0)

2. **Миниатюры**: серые квадратики (битые URL для smoke-альбома).
   При hover — синяя обводка + overlay внизу с filename.

3. **Поиск**: введи в поле «5570» — должна отфильтроваться одна-две
   миниатюры. Очисти поле — все возвращаются.

4. **Бейджи использования** — на тех фото которые builder вставил в
   layout: маленький зелёный pill «✓ N» в верхнем правом углу
   миниатюры. Например, фото портрет ученика «Архалькина Вероника»
   используется в spread 2 → бейдж «✓ 2» (помним: spread_index в
   БД 1 → UI 2 после `+1`).

5. **Скролл вниз**: при скролле миниатюр заголовок «Палитра фото» +
   поиск остаются sticky сверху. Заголовки секций «Портреты»/«Учителя»
   тоже становятся sticky под основным header'ом.

### Что **не** должно сломаться

- Левая колонка с canvas работает как раньше
- Навигация ◀▶
- Кнопка «← К альбому»
- Пустые слоты на canvas рендерятся как раньше

### Что **может** не работать (если всплывёт — фиксим в 2.6.2-followup)

- **Бейдж использования не появляется на миниатюрах** — вероятная
  причина: builder сложил в `instance.data` URL'ы которые НЕ совпадают
  с теми что отдаёт `/api/tenant?action=album_photos`. Например, в
  builder'е через `getPhotoUrl(yc:path)` URL может быть другим (кеш-bust
  или нормализация). Если бейджи не появляются — посмотри в DevTools
  Console:

  ```js
  fetch('/api/layout?action=album_layout&album_id=3fccf1dc-3e3d-402b-8c0c-7036e3496418', {credentials:'include'}).then(r=>r.json()).then(d=>console.log('LAYOUT URL:', Object.values(d.layout.spreads[1].data).find(v=>typeof v==='string'&&v.startsWith('http'))))
  fetch('/api/tenant?action=album_photos&album_id=3fccf1dc-3e3d-402b-8c0c-7036e3496418', {credentials:'include'}).then(r=>r.json()).then(d=>console.log('PALETTE URL:', d.photos[0].url))
  ```

  Если оба вывода форматом совпадают (`https://storage.yandexcloud.net/yearbook-photos/.../portrait/...webp`),
  бейджи должны работать. Если разные — нужно нормализовать
  сравнение в `findUsage` (например по storage_path вместо url).

- **Sticky заголовки секций накладываются на кнопки**: на узких окнах
  возможно. Если заметишь — поправим стили в followup'е.

- **Производительность скролла дёрганая**: для 179 фото не должно
  быть — но если жалобы будут, добавим CSS `will-change: transform`
  или виртуализацию.
