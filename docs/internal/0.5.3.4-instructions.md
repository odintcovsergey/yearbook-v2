# Инструкция для подэтапа 0.5.3.4 (финальный для фазы 0.5.3)

**Цель:** Финальная замена и чистка. Старый `build.ts` и `scenarios.ts`
удаляются. `buildAlbumFromPreset` становится единственным `buildAlbum`.
Smoke-сцены адаптируются под Preset. Endpoint `/api/layout?action=build_album_test`
переходит на preset_slug. UI Build Test обновляется.

После 0.5.3.4 фаза 0.5.3 закрыта. Дальше фазы 0.5.6 (UI per-child) и
0.5.7 (контекст v40 → v41) — это отдельные подэтапы.

---

## Принципиальные решения

### Перенос `MasterFilter` в types.ts

Сейчас `MasterFilter` определён в `scenarios.ts`, импортируется в
`find-master.ts` и продублирован локально в `build-from-preset.ts`.
После удаления `scenarios.ts` тип нужно сохранить в одном месте.

**Решение:** перенести `MasterFilter` из `scenarios.ts` в `types.ts`
(публичный API builder'а). После этого:
- `find-master.ts` импортирует из `./types` (вместо `./scenarios`)
- `build-from-preset.ts` удаляет локальный дубликат, импортирует из `./types`
- `scenarios.ts` удаляется без проблем

### UI Build Test — оставляем 2 dropdown'а

Текущий UI имеет 2 поля (`config_type` + `print_type`). Объединять в один
dropdown с 14 пресетами не стоит — UX хуже. **Решение:** клиент собирает
`preset_slug = `${configType}-${printType}`` сам и отправляет на сервер
только slug. Серверу не надо знать про маппинг.

### Сохранение 58 сцен smoke

Расширение smoke до 65+ сцен с покрытием grid edge-cases отложим в
0.5.4 (отдельный подэтап спецификации фазы 0.5). В 0.5.3.4 — только
адаптация существующих 58 сцен под Preset, без новых.

Финал 0.5.3.4: smoke 58/58 на новом builder'е.

### Имя файла build-from-preset.ts

Оставляем как есть (НЕ переименовываем в `build.ts`). Историческая запись
эволюции builder'а. `index.ts` импортирует `buildAlbum` из этого файла —
имя для пользователей API чистое.

### Шаг 6. Добавить `loadPresetBySlug` в `load-template-set.ts`

В файле `lib/album-builder/load-template-set.ts` добавить новую функцию
после `loadTemplateSet`:

```typescript
import type { Preset, TemplateSet } from './types';

// ... существующая loadTemplateSet ...

/**
 * Загрузка config_preset из Supabase в формат, который ожидает buildAlbum().
 *
 * Используется:
 *  - В scripts/smoke-album-builder.ts для тестового прогона
 *  - В app/api/layout/route.ts (action=build_album_test)
 *
 * @param supabase — клиент с правами SELECT на config_presets
 * @param slug — slug пресета (например 'standard-layflat', 'mini-soft')
 */
export async function loadPresetBySlug(
  supabase: SupabaseClient,
  slug: string,
): Promise<Preset> {
  const { data, error } = await supabase
    .from('config_presets')
    .select('*')
    .eq('slug', slug)
    .single();
  if (error || !data) {
    throw new Error(
      `config_preset ${slug} not found: ${error?.message ?? 'no row'}`,
    );
  }
  return data as Preset;
}
```
