# Инструкция для подэтапа 2.6.5

**Цель:** активировать кнопку «Открыть редактор» в `LayoutPreviewStrip`
(в Обзоре альбома). Сейчас она `disabled` с tooltip «Редактор скоро
будет доступен» — после 2.6.5 будет рабочая, ведущая на
`/app/album/<id>/layout`. Это последний штрих UX'а — партнёр больше
не должен вводить UUID руками.

**Зачем:** редактор готов и работает (подэтапы 2.6.1-2.6.4 закрыты),
но единственный способ туда попасть сейчас — копировать UUID альбома
из URL/Supabase и вставлять в адресную строку. Кнопка-навигация —
финальный мост между Обзором и редактором.

**Контекст:** все участники готовы:
- `LayoutPreviewStrip` (2.3) принимает пропс `onOpenEditor: () => void`,
  кнопка `disabled` с заглушкой
- В `app/app/page.tsx:127` есть `const router = useRouter()`
- В `AlbumDetailModal` (где рендерится `LayoutPreviewStrip`)
  переменная `album.id` доступна в scope
- Call site в `page.tsx:1200` имеет TODO-комментарий «В 2.6: router.push(...)»
- Маршрут `/app/album/[id]/layout` существует и работает

**Что меняется:**
- `app/app/_components/LayoutPreviewStrip.tsx` — убрать `disabled`,
  обновить tooltip и стили
- `app/app/page.tsx` — заменить пустой `onOpenEditor` на `router.push`
- `docs/internal/2.6.5-instructions.md` — этот файл

**Что НЕ трогаем:**

## Принципиальные решения

### Стилизация кнопки — primary action

Сейчас кнопка серая «вторичная» (`text-gray-500`, `bg-white`,
`border-gray-300`). После активации она становится **главным CTA**
в Обзоре — клик ведёт на главный инструмент партнёра. Делаем
её primary blue:

```tsx
className="text-xs px-3 py-1.5 rounded bg-blue-600 text-white hover:bg-blue-700 transition"
```

Tooltip убираем — теперь не нужен.

### `router.push` — без беспокойства про базовый URL

`useRouter` из `next/navigation` использует относительные пути.
`router.push('/app/album/abc/layout')` работает на любом домене
(yearbook-v2.vercel.app, окeybook.ru, localhost). Никаких absolute
URL'ов.

### `target="_blank"` — НЕ делаем

Альтернатива: открывать редактор в новой вкладке. Подкупает удобством
(партнёр может держать Обзор и редактор рядом). Но:

1. Редактор использует beforeunload protection (2.6.4) — при unsaved
   changes показывается диалог. В новой вкладке партнёр может это не
   увидеть и закрыть с потерей данных, не сообразив что у него open
   editor где-то.
2. Кеш данных не разделяется между вкладками — если он переключится
   и сделает что-то в Обзоре (например пересборку), это не отразится
   в открытой вкладке редактора → конфликт.

Откладываем «open in new tab» в backlog. Сейчас — простая навигация
через `router.push`.

### Не очищаем layout state в page.tsx при навигации

При уходе из `/app` через `router.push` компонент unmount'ится,
state очищается автоматически. Возвращаясь обратно через «← К альбому»
из редактора, AlbumDetailModal получит fresh state и снова загрузит
layout. Это норма.

### Не передаём data в редактор через query params

Альтернатива: можно передать `?layout_loaded=true` чтобы редактор
не делал лишний fetch. Не делаем — редактор всегда грузит fresh
данные из БД. Если партнёр в Обзоре сделал автосборку, потом ушёл в
редактор — он получит свежий layout, не зависящий от cached state в
другом компоненте. Это правильнее архитектурно.

## Шаги

### Шаг 1. Активировать кнопку в `LayoutPreviewStrip`

Файл: `app/app/_components/LayoutPreviewStrip.tsx`

Найди блок с кнопкой (~строки 85-93):

**БЫЛО:**

```tsx
        <button
          type="button"
          onClick={onOpenEditor}
          disabled
          className="text-xs px-3 py-1.5 rounded border border-gray-300 bg-white text-gray-500 disabled:opacity-50 disabled:cursor-not-allowed"
          title="Редактор скоро будет доступен (подэтап 2.6)"
        >
          Открыть редактор
        </button>
```

**СТАЛО:**

```tsx
        <button
          type="button"
          onClick={onOpenEditor}
          className="text-xs px-3 py-1.5 rounded bg-blue-600 text-white hover:bg-blue-700 transition cursor-pointer"
        >
          Открыть редактор
        </button>
```

Изменения:
- Убран `disabled`
- Убран `title` с пояснением «скоро будет доступен»
- Стиль primary blue вместо серого secondary

## Применение и верификация в проде

### Порядок

Миграций нет. Vercel задеплоит за ~30-60 секунд.

### Проверка визуально

После деплоя:

1. Открой `https://yearbook-v2.vercel.app/app`
2. Открой тестовый альбом → вкладка **Обзор**
3. В блоке «Превью разворотов» справа должна быть **синяя кнопка**
   «Открыть редактор» (вместо серой disabled)
4. Hover на кнопку — курсор `pointer`, лёгкий darker-blue effect
5. **Клик** — должен произойти переход на `/app/album/<UUID>/layout`,
   редактор открывается
6. Header редактора показывает заголовок альбома, canvas грузится,
   палитра справа

7. Кнопка **«← К альбому»** в редакторе ведёт обратно на `/app`
   (модал альбома закрывается, видно общий список альбомов)

### Что **может** не работать

- **Кнопка кликабельна но переход не происходит** — возможно `router`
  не импортирован/не работает. Проверь в DevTools Console — есть ли
  ошибки. На корректном `router.push` Next.js должен мгновенно
  навигировать.

## После 2.6.5 — что разблокируется

После 2.6.5 редактор полностью интегрирован в UI кабинета. Партнёр
может:
1. Открыть альбом → Обзор
2. Нажать «Собрать автоматически» → builder создаёт layout
3. Видит миниатюры в `LayoutPreviewStrip`
4. Кликает «Открыть редактор» → попадает в редактор
5. Редактирует drag-and-drop'ом, всё сохраняется
6. Возвращается в Обзор через «← К альбому»

  (защита от перезатирания правок)
