# Инструкция для подэтапа 2.2

**Цель:** создать React-компонент `AlbumSpreadCanvas` — Konva-рендер
одного `SpreadInstance` с реальным контентом (фото и текст из
`instance.data`), а не просто bbox'ы плейсхолдеров. Это базовый
строительный блок фазы 2: его будут использовать `LayoutPreviewStrip`
(2.3) и fullscreen-редактор (2.6).

**Зачем:** в фазе 1.4 партнёр после автосборки видит только текстовый
result-блок («Layout собран · 11 разворотов · 2 предупреждений»).
Это не даёт визуального понимания того, что получилось. В фазе 2.2 мы
готовим компонент, который в 2.3 покажет миниатюры разворотов в Обзоре
с реально вставленными фото.

**Контекст:** В проекте уже есть `app/super/templates/_components/SpreadCanvas.tsx`
(фаза 0.8) — он рисует **только bbox'ы плейсхолдеров с лейблами**, для
просмотра шаблонов в `/super/templates/[id]`. Новый компонент —
**параллельный, не наследник**: рисует то же самое + поверх рисует
реальный контент из `instance.data`. Старый `SpreadCanvas` остаётся
без изменений.

**Что меняется:**
- Новая папка `app/app/_components/` (создаётся вместе с первым файлом)
- Новый файл `app/app/_components/AlbumSpreadCanvas.tsx`
- Новый файл `docs/internal/2.2-instructions.md` (этот файл)

**Что НЕ трогаем:**
- `app/super/templates/_components/SpreadCanvas.tsx` — остаётся для
  просмотрщика шаблонов
- `app/app/page.tsx` — call site для `AlbumSpreadCanvas` появится в
  подэтапе 2.3 (`LayoutPreviewStrip`)
- `lib/album-builder/types.ts` — берём типы как есть, без правок
- `package.json` — Konva 10.3 + react-konva 18.2 уже установлены, новых
  зависимостей не добавляем

## Принципиальные решения

### Где разместить файл

`app/app/_components/AlbumSpreadCanvas.tsx`

Папка `_components/` соответствует §3.2 phase-2-spec и аналогична существующей
`app/super/templates/_components/`. Префикс `_` — convention Next.js для
исключения папки из роутинга (в `/app/app/_components/...` не появляется
URL роут). В одном проекте уже используется такая структура.

### Импорт типов из lib/album-builder

Используем `SpreadInstance`, `SpreadTemplate`, `PhotoPlaceholder`,
`TextPlaceholder` из `@/lib/album-builder/types`. Тип `SpreadTemplate` там
содержит семантические поля (`applies_to_configs`, `page_role`, `slot_capacity`,
`is_fallback`, `mirror_for_soft`), но компонент их игнорирует — для рендера
важны только `width_mm`, `height_mm`, `placeholders`.

Альтернатива: завести узкий `RenderableTemplate = Pick<SpreadTemplate, ...>`.
Не делаем — в фазе 2.3+ компонент будет получать `SpreadTemplate` из ответа
`GET /api/layout?action=template_set_detail` (тип уже соответствует
проекту), накручивать узкий слой адаптации сейчас — ранняя оптимизация.

### Photo-плейсхолдер: object-fit cover через Konva crop

В IDML `fit: 'fill_proportional'` соответствует CSS `object-fit: cover`.
Konva `<Image>` нативно умеет атрибут `crop`, но без хелпера. Хелпер
`getCoverCrop(img, w, h)` вычисляет, какую область HTMLImageElement'а
обрезать, чтобы вписать в placeholder с заполнением и центрированием:

```typescript
function getCoverCrop(img: HTMLImageElement, targetW: number, targetH: number) {
  const targetRatio = targetW / targetH
  const imageRatio = img.naturalWidth / img.naturalHeight
  if (imageRatio > targetRatio) {
    // изображение шире → обрезаем по бокам
    const cropW = img.naturalHeight * targetRatio
    return {
      x: (img.naturalWidth - cropW) / 2,
      y: 0,
      width: cropW,
      height: img.naturalHeight,
    }
  }
  // изображение выше → обрезаем сверху/снизу
  const cropH = img.naturalWidth / targetRatio
  return {
    x: 0,
    y: (img.naturalHeight - cropH) / 2,
    width: img.naturalWidth,
    height: cropH,
  }
}
```

### Photo-плейсхолдер: пустой слот

Когда `instance.data[placeholder.label]` = `null` или `undefined` (или
URL не загрузился — image error), рисуем тонкую dashed-рамку. Это
визуальный сигнал «фото не назначено», который партнёр в 2.6 сможет
заполнить drag'ом из палитры.

```tsx
<Rect
  x={p.x_mm} y={p.y_mm}
  width={p.width_mm} height={p.height_mm}
  stroke="#cbd5e1"
  strokeWidth={0.3}
  dash={[1, 1]}
/>
```

### Photo-плейсхолдер: круглые портреты (`is_circle`)

Учительские портреты (`headerPhoto`, `teacherPhoto_*`) — круглые
(в БД `is_circle=true` для PhotoPlaceholder). Реализация через Konva
`<Group>` с `clipFunc` рисующим эллипс:

```tsx
if (p.is_circle) {
  const cx = p.x_mm + p.width_mm / 2
  const cy = p.y_mm + p.height_mm / 2
  const rx = p.width_mm / 2
  const ry = p.height_mm / 2
  return (
    <Group
      key={key}
      clipFunc={(ctx) => {
        ctx.beginPath()
        ctx.ellipse(cx, cy, rx, ry, 0, 0, Math.PI * 2)
        ctx.closePath()
      }}
    >
      <KonvaImage x={p.x_mm} y={p.y_mm} width={p.width_mm} height={p.height_mm} image={img} crop={crop} />
    </Group>
  )
}
```

Эллипс, не строго круг — placeholders могут быть не идеально квадратными
(хотя у `is_circle=true` обычно width≈height).

### Text-плейсхолдер: pt → mm

`placeholder.font_size_pt` — в типографических пунктах (IDML units).
Konva `<Text>` принимает `fontSize` в той же единице что Stage scaled to —
у нас Stage в мм. Коэффициент 1 pt = 0.3528 mm.

```typescript
const PT_TO_MM = 0.3528
const fontSizeMm = p.font_size_pt * PT_TO_MM
```

### Text-плейсхолдер: шрифт fallback

В фазе 2 не загружаем реальные шрифты (это фаза 3 вместе с PDF, см.
phase-2-spec §2). Все text-плейсхолдеры рендерятся через
`fontFamily="Arial, sans-serif"` независимо от того, что лежит в
`placeholder.font_family` (там может быть `Open Sans`, `Verdana Pro`,
кастомные семейства из IDML).

### Text-плейсхолдер: vertical_align

В реальных данных (idml-recon §4) встречается только `vertical_align: 'top'`.
Konva `<Text>` по умолчанию рисует от верха области — соответствует.
В props пишем `verticalAlign="top"` явно для будущего расширения.

### Text-плейсхолдер: auto_fit

Игнорируем в 2.2. `auto_fit: true` означает, что текст должен ужиматься
если не помещается в bbox. Реализация требует measureText итеративно —
это фаза 3 (вместе с реальными шрифтами).

### `mode='preview'` vs `mode='edit'`

В 2.2 пропс `mode` принимается, но влияет только на `listening` Stage'а:
- `'preview'` (default) → `listening={false}` (не подписываемся на события,
  экономия hit-graph и памяти, как в SpreadCanvas с `listening=false`)
- `'edit'` → `listening={true}` (события готовы для drag'а в 2.6)

`onDrop` и `onSwap` — типизированные пропсы, но в 2.2 не используются
(handler'ы прокинутся в 2.6).

### Без подписей плейсхолдеров

Старый `SpreadCanvas` показывает label поверх каждого bbox'а
(`<Label>` + `<Tag>` + `<Text>`). Новый `AlbumSpreadCanvas` подписей
**не показывает** — там вместо них реальный контент (фото или текст).
В режиме `'edit'` в 2.6 будет hover-highlight, но это уже другая фишка.

### Background image

У `SpreadTemplate` есть поле `background_url` (фоновое изображение
шаблона, см. `app/super/templates/_components/types.ts:73`). В 2.2
**игнорируем** — оно не используется ни одним из 39 мастеров
okeybook-default (всё через placeholders). Если в будущем появятся
шаблоны с фоном — добавим рендер первым слоем, до placeholder'ов.

### Тестирование

Файл создаётся, но нигде не импортируется. Это значит:
- `npx tsc --noEmit` проверит синтаксис и типы
- `npx next build` соберёт страницу, но dead code-shaking может выкинуть
  компонент → проверка минимальная
- **Реальная проверка** — в подэтапе 2.3, когда `LayoutPreviewStrip`
  начнёт рендерить компонент с боевыми данными

Это нормально — следуем плану коммитов. Если в 2.3 найдутся баги
рендера — фиксим там.

### Шаг 1. Создать папку и файл компонента

Файл: `app/app/_components/AlbumSpreadCanvas.tsx`

```tsx
'use client'

import { useEffect, useState } from 'react'
import { Stage, Layer, Rect, Image as KonvaImage, Text, Group } from 'react-konva'
import type {
  SpreadInstance,
  SpreadTemplate,
  Placeholder,
  PhotoPlaceholder,
  TextPlaceholder,
} from '@/lib/album-builder/types'

// ─────────────────────────────────────────────────────────────────────────
// AlbumSpreadCanvas — Konva-рендер одного SpreadInstance.
//
// В отличие от app/super/templates/_components/SpreadCanvas.tsx (рисует
// только bbox'ы плейсхолдеров для просмотрщика шаблонов), этот компонент
// рисует РЕАЛЬНЫЙ контент: фото из instance.data[label] и текстовые
// значения. Используется в LayoutPreviewStrip (фаза 2.3) и в редакторе
// /app/album/[id]/layout (фаза 2.6).
//
// Координаты плейсхолдеров — в миллиметрах (как в БД, см. idml-recon §3).
// Stage применяет scale=containerWidth/template.width_mm; всё внутри
// Layer'а рисуется в мм-единицах.
//
// Шрифты: fontFamily="Arial, sans-serif" fallback (реальные шрифты —
// фаза 3 вместе с PDF, см. phase-2-spec §2).
// ─────────────────────────────────────────────────────────────────────────

const PT_TO_MM = 0.3528  // 1 typographic point = 0.3528 mm

type Mode = 'preview' | 'edit'

type Props = {
  instance: SpreadInstance
  template: SpreadTemplate
  containerWidth: number  // pixel-ширина Stage'а; Stage сам отскейлит до template.width_mm
  mode?: Mode  // default 'preview'
  // Drag handlers (используются в фазе 2.6, в 2.2 типизированы но не вызываются)
  onDrop?: (label: string, photoId: string) => void
  onSwap?: (fromLabel: string, toLabel: string) => void
}

// ─── Хелпер: загрузка HTMLImageElement из URL ────────────────────────────
//
// Возвращает null пока URL не пустой и картинка не загрузилась (или
// произошла ошибка загрузки). При смене URL очищает прежнее изображение.
function useImage(url: string | null): HTMLImageElement | null {
  const [img, setImg] = useState<HTMLImageElement | null>(null)
  useEffect(() => {
    if (!url) {
      setImg(null)
      return
    }
    const i = new window.Image()
    i.crossOrigin = 'anonymous'
    i.src = url
    i.onload = () => setImg(i)
    i.onerror = () => setImg(null)
  }, [url])
  return img
}

// ─── Хелпер: cover crop (object-fit: cover в терминах Konva crop) ────────
function getCoverCrop(img: HTMLImageElement, targetW: number, targetH: number) {
  const targetRatio = targetW / targetH
  const imageRatio = img.naturalWidth / img.naturalHeight
  if (imageRatio > targetRatio) {
    const cropW = img.naturalHeight * targetRatio
    return {
      x: (img.naturalWidth - cropW) / 2,
      y: 0,
      width: cropW,
      height: img.naturalHeight,
    }
  }
  const cropH = img.naturalWidth / targetRatio
  return {
    x: 0,
    y: (img.naturalHeight - cropH) / 2,
    width: img.naturalWidth,
    height: cropH,
  }
}

// ─── Photo placeholder ────────────────────────────────────────────────────
function PhotoSlot({
  placeholder,
  url,
}: {
  placeholder: PhotoPlaceholder
  url: string | null
}) {
  const img = useImage(url)

  // Пустой слот → dashed frame
  if (!url || !img) {
    return (
      <Rect
        x={placeholder.x_mm}
        y={placeholder.y_mm}
        width={placeholder.width_mm}
        height={placeholder.height_mm}
        stroke="#cbd5e1"
        strokeWidth={0.3}
        dash={[1, 1]}
      />
    )
  }

  const crop = getCoverCrop(img, placeholder.width_mm, placeholder.height_mm)

  // Круглые портреты (учительские) → клиппинг по эллипсу
  if (placeholder.is_circle) {
    const cx = placeholder.x_mm + placeholder.width_mm / 2
    const cy = placeholder.y_mm + placeholder.height_mm / 2
    const rx = placeholder.width_mm / 2
    const ry = placeholder.height_mm / 2
    return (
      <Group
        clipFunc={(ctx) => {
          ctx.beginPath()
          ctx.ellipse(cx, cy, rx, ry, 0, 0, Math.PI * 2)
          ctx.closePath()
        }}
      >
        <KonvaImage
          x={placeholder.x_mm}
          y={placeholder.y_mm}
          width={placeholder.width_mm}
          height={placeholder.height_mm}
          image={img}
          crop={crop}
        />
      </Group>
    )
  }

  return (
    <KonvaImage
      x={placeholder.x_mm}
      y={placeholder.y_mm}
      width={placeholder.width_mm}
      height={placeholder.height_mm}
      image={img}
      crop={crop}
    />
  )
}

// ─── Text placeholder ─────────────────────────────────────────────────────
function TextSlot({
  placeholder,
  value,
}: {
  placeholder: TextPlaceholder
  value: string | null
}) {
  if (!value) return null
  return (
    <Text
      x={placeholder.x_mm}
      y={placeholder.y_mm}
      width={placeholder.width_mm}
      height={placeholder.height_mm}
      text={value}
      fontSize={placeholder.font_size_pt * PT_TO_MM}
      fontFamily="Arial, sans-serif"
      fill={placeholder.color || '#000000'}
      align={placeholder.align}
      verticalAlign="top"
    />
  )
}

// ─── Главный компонент ────────────────────────────────────────────────────
export default function AlbumSpreadCanvas({
  instance,
  template,
  containerWidth,
  mode = 'preview',
}: Props) {
  const scale = containerWidth / template.width_mm
  const stageWidth = template.width_mm * scale
  const stageHeight = template.height_mm * scale

  return (
    <Stage
      width={stageWidth}
      height={stageHeight}
      scaleX={scale}
      scaleY={scale}
      listening={mode === 'edit'}
    >
      <Layer>
        {/* Фон spread'а — серый прямоугольник с тонкой границей */}
        <Rect
          x={0}
          y={0}
          width={template.width_mm}
          height={template.height_mm}
          fill="#fafafa"
          stroke="#e5e7eb"
          strokeWidth={0.3}
        />

        {/* Контент из instance.data */}
        {template.placeholders.map((p: Placeholder, i) => {
          const value = instance.data[p.label] ?? null
          const key = `${p.label}-${i}`
          if (p.type === 'photo') {
            return <PhotoSlot key={key} placeholder={p} url={value} />
          }
          if (p.type === 'text') {
            return <TextSlot key={key} placeholder={p} value={value} />
          }
          return null
        })}
      </Layer>
    </Stage>
  )
}
```

**Замечания по реализации:**

1. **`window.Image` вместо просто `Image`** — чтобы tsc не путался с
   React-импортом `Image`. Уже в this file используется
   `Image as KonvaImage`, но безопаснее писать `new window.Image()`.

2. **Пропсы `onDrop` / `onSwap` объявлены, но не использованы** — это
   нормально для 2.2. tsc не ругается на неиспользованные параметры
   функции (только на неиспользованные **импорты** и **переменные**),
   а в react props для функционального компонента они вообще не считаются
   неиспользованными если не деструктуризированы. **Не деструктуризируем
   их** — оставляем в типе Props для документации API, в реализации
   пропсы достаются позже в 2.6.

3. **`window.Image` доступен только на клиенте** — поэтому компонент
   обязательно с `'use client'`. На call site'ах в 2.3+ обернём через
   `dynamic(() => import(...), { ssr: false })` (паттерн из
   `app/super/templates/[id]/page.tsx:48`).

## Замечания (на потом, если что-то всплывёт)

### Производительность миниатюр

Спека phase-2-spec §6 (критерии приёмки) предполагает миниатюры по 250px.
Для альбома на 11 разворотов это 11 Stage'ей с реальными фото. Если
в DevTools будет видно тормоза — можно:
- Добавить `pixelRatio={1}` в Stage (как в SpreadCanvas:39)
- Сжать фото до thumb-версии перед загрузкой (использовать `getThumbUrl`
  вместо `getPhotoUrl` — это уже паттерн проекта, см. `lib/supabase.ts`)

В 2.2 не оптимизируем — реальная картина видна только в 2.3 с боевыми
данными.

### CORS на storage.yandexcloud.net

`<Image>` загружается с `crossOrigin='anonymous'`. YC бакет
`yearbook-photos` публичный (см. yearbook-context-v43:111), CORS должен
быть открыт. Если что-то пойдёт не так в 2.3 (картинки не отображаются,
ошибка в console про CORS) — проверить настройки бакета в YC.
