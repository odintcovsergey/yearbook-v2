# Инструкция для подэтапа 0.5.6.1

**Цель:** UI выбора пресета у альбома в `/app`. Два dropdown'а
(«Комплектация» + «Тип печати») в AlbumFormModal, отображение в
AlbumDetailModal → Обзор. API `presets_list` + расширение
`create_album` / `update_album` + auto-resolve `template_set_id`.

**Зачем нужно:** все 9 существующих альбомов имеют
`config_preset_id = NULL`. Для запуска фазы 1 (Smart-fill) и UI
per-child override (0.5.6.2) у альбома должен быть пресет.

---

## Принципиальные решения

### 2 dropdown'а вместо одного с 14 опциями

UX-эталон — Build Test UI (`/super/templates/[id]`): два отдельных
dropdown'а — комплектация (7 значений) и тип печати (2 значения).
В AlbumFormModal делаем по тому же шаблону. Клиент собирает
`preset_slug = `${config_type}-${print_type}`` перед отправкой.

### Сервер принимает `preset_slug`

API endpoints (`create_album`, `update_album`) принимают опциональное
поле `preset_slug`. Сервер находит пресет по slug и записывает его id
в `albums.config_preset_id`. Клиент не знает про preset.id — только
slug. Симметрично с `/api/layout?action=build_album_test`.

### Auto-resolve `template_set_id`

В системе сейчас один template_set (`okeybook-default`). При сохранении
альбома, если `template_set_id` всё ещё NULL — сервер автоматически
ставит ID единственного template_set. Когда появятся другие (фаза 4) —
добавим UI выбора.

### Backward compat для `albums.print_type`

Поле `albums.print_type` (legacy с фазы 1.0) остаётся синхронизированным
с print_type выбранного пресета. UI продолжает читать это поле
(в AlbumCard на строке 976). При сохранении из формы — сервер записывает
обе колонки.

### Существующие 9 альбомов

Они продолжат работать как раньше: пока пользователь не отредактирует
альбом и не выберет пресет, `config_preset_id` останется NULL. Builder
будет ругаться `master_not_found` если попытаться запустить — это
ожидаемо. UI должен явно показывать когда пресет не задан и предлагать
выбрать.

### Шаг 2. API: helper `resolvePresetSlug`

В начале `app/api/tenant/route.ts` (рядом с другими helpers, до
блоков `if (body.action === ...)`) добавить:

```typescript
/**
 * Резолвит preset_slug → preset record. Используется в create_album и
 * update_album для маппинга {config_type, print_type} формы на FK.
 */
async function resolvePresetBySlug(slug: string): Promise<{
  id: string
  slug: string
  print_type: string
} | null> {
  const { data, error } = await supabaseAdmin
    .from('config_presets')
    .select('id, slug, print_type')
    .eq('slug', slug)
    .is('tenant_id', null)
    .single()

  if (error || !data) return null
  return data
}

/**
 * Возвращает ID единственного глобального template_set (okeybook-default).
 * Используется при создании/обновлении альбома если template_set_id ещё NULL.
 * В фазе 4 (расширение библиотеки) этот хардкод заменится на UI выбор.
 */
async function getDefaultTemplateSetId(): Promise<string | null> {
  const { data, error } = await supabaseAdmin
    .from('template_sets')
    .select('id')
    .eq('slug', 'okeybook-default')
    .is('tenant_id', null)
    .single()

  if (error || !data) return null
  return data.id
}
```

### Шаг 3. API: расширить `create_album`

```typescript
// Резолв preset_slug → config_preset_id + print_type
let configPresetId: string | null = null
let presetPrintType: string | null = null
if (typeof body.preset_slug === 'string' && body.preset_slug.length > 0) {
  const preset = await resolvePresetBySlug(body.preset_slug)
```

```typescript
// Если клиент прислал preset_slug — резолвим в id и print_type
if (typeof body.preset_slug === 'string' && body.preset_slug.length > 0) {
  const preset = await resolvePresetBySlug(body.preset_slug)
  if (!preset) {
    return NextResponse.json(
      { error: `preset_slug ${body.preset_slug} not found` },
      { status: 400 },
    )
  }
  body.config_preset_id = preset.id
  body.print_type = preset.print_type
}

// Auto-resolve template_set_id если не задан
if (body.template_set_id === undefined) {
  // Проверяем что у альбома уже есть template_set_id; если нет — ставим default
  const { data: existing } = await supabaseAdmin
    .from('albums')
    .select('template_set_id')
    .eq('id', album_id)
    .single()
  if (!existing?.template_set_id) {
    const tsId = await getDefaultTemplateSetId()
    if (tsId) body.template_set_id = tsId
  }
}
```
