# Инструкция для подэтапа 2.6.5.1

**Цель:** возврат из редактора через «← К альбому» должен **открыть
тот же альбом** (модал AlbumDetailModal), а не сваливать на общий
список альбомов. Реализуем через query-параметр `?album=<UUID>`.

**Зачем:** в 2.6.1 я заложил простой `router.push('/app')` без UUID
с пометкой «улучшим в 2.6.5 если будет неудобно». Партнёр на проверке
2.6.5 подтвердил — каждый возврат из редактора заставляет искать
альбом среди 9+ карточек на главной. Это плохой UX, особенно когда
партнёр будет несколько раз ходить туда-сюда (правка → проверка
в Обзоре → ещё правка).

**Контекст:** все необходимые функции есть:
- В `app/app/page.tsx:127` есть `useRouter` (главный компонент)
- В `useSearchParams` из `next/navigation` — стандартный hook для
  чтения query
- `albums` загружаются через `loadDashboard()` после авторизации
- `setSelectedAlbum(album)` открывает `AlbumDetailModal`
- В редакторе кнопка «← К альбому» уже использует `router.push`

**Что меняется:**
- `app/app/album/[id]/layout/page.tsx` — `router.push('/app')` →
  `router.push(\`/app?album=${albumId}\`)`
- `app/app/page.tsx` — добавить `useSearchParams`, useEffect который
  читает query и автоматически вызывает `setSelectedAlbum` если
  `?album=<UUID>` есть в URL и albums загружены
- `docs/internal/2.6.5.1-instructions.md` — этот файл

**Что НЕ трогаем:**
- API endpoint'ы — нет
- Маршрутизация — без изменений (используем существующий /app)
- Логика модала AlbumDetailModal — без изменений

## Принципиальные решения

### Query-параметр `?album=<UUID>` — стандартный паттерн deep link

Альтернативы:
- **Hash-router (`#album=<UUID>`)** — старый паттерн, нет реального
  отличия от query, но менее стандартный для Next.js
- **Path-параметр (`/app/<UUID>`)** — потребует ребилд маршрута,
  ломает существующие ссылки
- **localStorage** — работает между вкладками, но не между сессиями
  и не делится через ссылку

Query — правильный выбор. Партнёр может скопировать URL и поделиться
с менеджером OkeyBook («посмотри этот альбом»), это бонус.

### `useSearchParams` через `'use client'`

`useSearchParams` от `next/navigation` работает только в client-side
компонентах. `/app/page.tsx` уже `'use client'` — паттерн совместим.

### useEffect: srartup-only, без race condition

Логика читает query при загрузке страницы и при изменении `albums`.
Если URL `?album=<UUID>` и в `albums` есть такой `id` — вызываем
`setSelectedAlbum(album)`. Если ID не найден — игнорируем (вероятно
устаревшая ссылка).

```typescript
const searchParams = useSearchParams()

useEffect(() => {
  const albumId = searchParams.get('album')
  if (!albumId || albums.length === 0) return
  const album = albums.find((a) => a.id === albumId)
  if (album) setSelectedAlbum(album)
}, [searchParams, albums])
```

**Защита от циклов:** при вызове `setSelectedAlbum` модал откроется,
но useEffect снова не сработает (deps те же). При закрытии модала
(`onClose`) state в URL остаётся `?album=<UUID>` — это **намеренно**:
если партнёр ещё раз нажмёт назад в редактор, URL'у уже не нужно
ничего менять.

**Если** партнёр кликнет на другой альбом в списке (пока модал открыт
от предыдущего альбома — теоретически, на практике закрывается)
— state перезапишется через `setSelectedAlbum`, URL сам не
обновится, но это незначительно: при следующем обновлении страницы
откроется альбом из URL, а не последний кликнутый. Лечится одной
строкой при желании, но не критично для MVP.

### Не очищаем query при закрытии модала

Альтернатива: `onClose={() => { setSelectedAlbum(null);
router.replace('/app') }}` — убирает `?album=<UUID>` при крестике.
Не делаем потому что:
- Лишний side-effect
- Если партнёр случайно закрыл модал, F5 вернёт его обратно
  (URL — единственный источник правды)
- Нет риска: deep link «нерасторжим», что в нашем случае хорошо

### Suspense Boundary — нужен или нет?

Next.js 14+ требует оборачивать `useSearchParams` в `<Suspense>`
boundary при использовании в App Router pages. Иначе билд падает.

В нашем случае `/app/page.tsx` это client component (`'use client'`)
с большим деревом — оборачивать всё в Suspense нерационально.
Стандартный обходной путь: использовать **внутренний компонент**
который читает `useSearchParams`, обернутый в Suspense только этот
кусок.

**Но** в этом проекте Next 15 (см. package.json) есть нюансы. Возможно
билд успешно пройдёт без Suspense — мы уже используем `useRouter` без
проблем. **Если build упадёт** на этом фиксе — добавим Suspense
обёртку для конкретного useEffect-блока.

Пока пробуем без Suspense — самая простая реализация.

## Шаги

### Шаг 1. В редакторе — pass UUID в URL

Файл: `app/app/album/[id]/layout/page.tsx`

Найди ВСЕ вхождения `router.push('/app')` (их два — в header'е и
в error блоке). Точнее:

```bash
grep -n "router.push('/app')" app/app/album/\[id\]/layout/page.tsx
```

Должно быть 2 вхождения:
1. В error блоке (~167)
2. В onClick header'а кнопки «← К альбому» (~315)

В обоих заменить:

**БЫЛО:**

```typescript
onClick={() => router.push('/app')}
```

**СТАЛО:**

```typescript
onClick={() => router.push(`/app?album=${albumId}`)}
```

### Шаг 2. Главная — useSearchParams + useEffect

Файл: `app/app/page.tsx`

#### 2а. Расширить импорт

В верхней части файла (~строка 4-6) найди существующий импорт:

```typescript
import { useRouter } from 'next/navigation'
```

Замени на:

```typescript
import { useRouter, useSearchParams } from 'next/navigation'
```

#### 2б. Использовать hook в AppPage

Найди главный компонент `export default function AppPage()` (~строка 126).
**Сразу после** `const router = useRouter()` добавь:

```typescript
  const searchParams = useSearchParams()
```

#### 2в. useEffect для deep link

Найди useEffect загрузки albums (это `useEffect(() => { if (auth)
loadDashboard() }, [auth])` ~строка 186-188). **Сразу после** него
добавь:

```typescript
  // Deep link: если URL содержит ?album=<UUID> и эти альбом есть в
  // списке — автоматически открыть его модал. Используется при
  // возврате из редактора `/app/album/<id>/layout`.
  useEffect(() => {
    const queryAlbumId = searchParams.get('album')
    if (!queryAlbumId || albums.length === 0) return
    const found = albums.find((a) => a.id === queryAlbumId)
    if (found) setSelectedAlbum(found)
  }, [searchParams, albums])
```

## Применение и верификация в проде

### Порядок

Миграций нет. Vercel задеплоит за ~30-60 секунд.

### Проверка визуально

1. Открой `https://yearbook-v2.vercel.app/app` → **тестовый альбом**
   → Обзор
2. Кликни **«Открыть редактор»** — должно перейти на
   `/app/album/<UUID>/layout`
3. В редакторе кликни **«← К альбому»**
4. Должно перейти на `/app?album=<UUID>` (видно в адресной строке)
5. **Автоматически открывается** модал именно тестового альбома, на
   вкладке Обзор. Без поиска вручную ✅

### Дополнительные проверки

- **Прямая ссылка с UUID работает**: открой `https://yearbook-v2.vercel.app/app?album=54bf48ee-5501-4c7f-a66a-1e8f8d2fc20e`
  в новой вкладке → должен открыться тестовый альбом сразу
- **Несуществующий UUID** в URL: `?album=00000000-0000-0000-0000-000000000000`
  — модал не открывается, видна обычная главная (album.find вернул
  undefined, ничего не происходит)
- **Закрытие модала через крестик**: URL остаётся `?album=<UUID>`,
  при F5 модал снова откроется (намеренное поведение)

## Что НЕ делаем в 2.6.5.1

- **Восстановление активной вкладки модала** (открыт на «Ученики»,
  не «Обзор») — в URL передавать ещё `?tab=...` — backlog
- **Open in new tab** — backlog
- **Контекст-файл v43 → v44** — обновляется в 2.8
