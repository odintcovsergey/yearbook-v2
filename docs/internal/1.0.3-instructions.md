# Инструкция для подэтапа 1.0.3

**Цель:** в кабинете партнёра (`/app` → вкладка «Учителя») увеличить
миниатюры фотографий учителей с 64×64 до 96×96 и под каждой добавить
имя файла (номер фото от камеры — `DSC08521`, `IMG_20250515_142933`).

**Зачем:** партнёр-фотограф работает в терминах номеров файлов с камеры
— у него на диске лежит та же серия исходников, и ему нужно сопоставить
выбранное фото с конкретным RAW/JPG для печати. Подпись под миниатюрой
даёт это сопоставление с одного взгляда без открытия CSV.

**Что меняется:**
- `app/app/page.tsx` → `TeachersTab` readonly-блок:
  - размер миниатюры `w-16 h-16` → `w-24 h-24`
  - блок миниатюры обёрнут во внешний `<div>` фиксированной ширины
    (для подписи без поломки выравнивания)
  - снизу под миниатюрой — `<div>` с именем файла без расширения,
    truncate + title с полным именем

**Что НЕ трогаем:**
- API не меняется — `photo_filename` уже отдаётся (1.0.2)
- Тип `Teacher` не меняется
- Edit-режим не трогаем
- `/teacher/[token]` страница родителя

## Принципиальные решения

### Размер 96×96 (`w-24 h-24`)

Между 64 (текущее, мелковато) и 128 (слишком гигантский на 8+ учителях).
96 — стандартный шаг Tailwind `w-24`, выглядит сбалансированно.

### Имя файла без расширения

Хранится как `DSC08521.jpg` (см. `app/api/upload/route.ts:68` —
`filename = originalName`). Расширение (`.jpg`/`.webp`) не несёт смысла
для партнёра, занимает место в подписи. Срезаем через
`.replace(/\.[^.]+$/, '')`.

Никакой умной обработки (вырезание `DSC`/`IMG` префиксов, выделение
только цифровой части) — хрупко и не нужно. Партнёр сам понимает свою
систему именования.

### Truncate + title

Имена бывают длинные (`IMG_20250515_142933.jpg`). Под миниатюру 96px
такое не влезет. Решение:
- `text-xs` — мелкий шрифт чтобы поместилось больше
- `truncate` — обрезка с `…` если не влезает
- `title={t.photo_filename ?? ''}` — full имя по hover-tooltip
  (включая расширение, потому что user может хотеть скопировать)
- Контейнер фиксированной ширины 96px (`w-24`) — иначе truncate не
  сработает, flex растянет

### Без подписи у placeholder'а

Если `photo_storage_path === null`, показываем серый «?» как сейчас, без
подписи. Подпись «Фото не выбрано» дублировала бы то что уже видно по
визуалу + лишняя вертикальная высота.

### Не показываем подпись при undefined `photo_filename`

В `Teacher` тип `photo_filename: string | null`. Если фото есть
(`photo_storage_path` не null) то и `photo_filename` тоже не null
(сервер заполняет оба вместе). Но на всякий случай — рендер подписи
условный по `t.photo_filename` (не падаем если null).

## Шаги

### Шаг 1. UI: правки readonly-блока

Файл: `app/app/page.tsx`, readonly-ветка `TeachersTab` (строки 2848-2895
после правок 1.0.2).

**Текущий код** (показан в инструкции 1.0.2):

```jsx
<div className="flex items-start gap-3 min-w-0 flex-1">
  {t.photo_storage_path ? (
    <a
      href={photoUrl(t.photo_storage_path)}
      target="_blank"
      rel="noopener noreferrer"
      className="block w-16 h-16 rounded-xl overflow-hidden flex-shrink-0 bg-gray-100"
      title={t.photo_filename ?? 'Открыть оригинал'}
    >
      <img
        src={photoUrl(t.photo_storage_path)}
        alt=""
        className="w-full h-full object-cover"
        loading="lazy"
      />
    </a>
  ) : (
    <div
      className="w-16 h-16 rounded-xl bg-gray-100 flex items-center justify-center flex-shrink-0 text-gray-300 text-2xl"
      title="Фото не выбрано"
    >
      ?
    </div>
  )}
  <div className="min-w-0 flex-1">
    {/* ФИО, бейджи, position, description */}
  </div>
</div>
```

**Заменить** блок с миниатюрой/placeholder'ом на (текстовый блок с
ФИО/бейджами/position/description **не трогать**):

```jsx
<div className="flex items-start gap-3 min-w-0 flex-1">
  {t.photo_storage_path ? (
    <div className="flex flex-col items-center w-24 flex-shrink-0">
      <a
        href={photoUrl(t.photo_storage_path)}
        target="_blank"
        rel="noopener noreferrer"
        className="block w-24 h-24 rounded-xl overflow-hidden bg-gray-100"
        title={t.photo_filename ?? 'Открыть оригинал'}
      >
        <img
          src={photoUrl(t.photo_storage_path)}
          alt=""
          className="w-full h-full object-cover"
          loading="lazy"
        />
      </a>
      {t.photo_filename && (
        <div
          className="text-xs text-gray-500 mt-1 truncate w-full text-center"
          title={t.photo_filename}
        >
          {t.photo_filename.replace(/\.[^.]+$/, '')}
        </div>
      )}
    </div>
  ) : (
    <div
      className="w-24 h-24 rounded-xl bg-gray-100 flex items-center justify-center flex-shrink-0 text-gray-300 text-3xl"
      title="Фото не выбрано"
    >
      ?
    </div>
  )}
  <div className="min-w-0 flex-1">
    {/* ФИО, бейджи, position, description — БЕЗ ИЗМЕНЕНИЙ */}
  </div>
</div>
```

Ключевые изменения:
- Превью с фото обёрнута во внешний `<div className="flex flex-col
  items-center w-24 flex-shrink-0">` — теперь это вертикальная стопка
  «миниатюра + подпись» фиксированной ширины 96px
- `<a>` сама стала `w-24 h-24` (без `flex-shrink-0` — он на родительском
  div, чтобы стопка не сжималась рядом с длинным текстом)
- Placeholder тоже стал `w-24 h-24` (`text-3xl` чтобы «?» был
  пропорциональным новому размеру)
- Подпись: `text-xs text-gray-500 mt-1 truncate w-full text-center` +
  `title` для full-tooltip; рендерится только если `t.photo_filename`
  не null
- Имя файла без расширения через `.replace(/\.[^.]+$/, '')`
