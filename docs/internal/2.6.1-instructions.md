# Инструкция для подэтапа 2.6.1

**Цель:** создать скелет fullscreen-редактора `/app/album/[id]/layout`.
В этом подэтапе — только базовая структура: загрузка данных, header,
левая колонка с одним spread в edit-режиме, правая колонка-заглушка
для палитры, навигация ◀▶ между разворотами.

**Зачем:** редактор — главный UI фазы 2, в нём партнёр будет
drag-and-drop'ом править layout. Подэтап 2.6 целиком (~6 часов работы)
дробится на пять под-подэтапов 2.6.1...2.6.5, чтобы каждый коммит
был тестируемым и проверяемым отдельно. В 2.6.1 проверяем что страница
открывается, данные грузятся, canvas рендерит spread в edit-режиме,
навигация работает — без drag-and-drop, без палитры, без auto-save.

**Контекст:** все необходимые компоненты и endpoint'ы уже готовы:
- `AlbumSpreadCanvas` (2.2) с props `mode='edit'` ← используем здесь
- `GET /api/layout?action=album_layout` (1.4) ← загрузка layout
- `GET /api/layout?action=template_set_detail&id=X` (0.5) ← placeholder'ы
- `GET /api/tenant?action=album_photos` (2.4) ← фото для палитры
- `useRouter` из `next/navigation` (паттерн уже в `app/app/page.tsx:127`)

**Что меняется:**
- Новая папка `app/app/album/[id]/layout/` (создаётся вместе с первым файлом)
- Новый файл `app/app/album/[id]/layout/page.tsx` (~280 строк)
- Новый файл `docs/internal/2.6.1-instructions.md` (этот файл)

**Что НЕ трогаем:**
- `LayoutPreviewStrip.tsx` — кнопка «Открыть редактор» остаётся disabled.
  Активируется в подэтапе 2.6.5 (отдельный коммит).
- `AlbumSpreadCanvas.tsx` — без правок (mode='edit' уже работает,
  просто включает `listening={true}` на Stage'е)
- `app/app/page.tsx` — без правок
- API endpoint'ы — все используются read-only, изменений нет
- Миграции БД — нет
- `package.json` — `@dnd-kit/core` и `@dnd-kit/sortable` уже стоят
  (для CRM использовались), но импортировать их будем в 2.6.3+

## Принципиальные решения

### Где именно создавать папку

`app/app/album/[id]/layout/page.tsx`

Это Next.js App Router конвенция:
- `app/app/` — namespace кабинета (тенант-aware, авторизация owner/manager/viewer)
- `[id]` — динамический сегмент UUID альбома
- `layout/` — роут редактора. **NB**: имя папки `layout` это просто
  URL-сегмент (`/app/album/<uuid>/layout`), а не зарезервированное
  Next.js имя (которое было бы `layout.tsx` в корне сегмента).
  Файл `page.tsx` внутри сделает страницу с URL.

Альтернатива `app/app/layout-editor/[id]/page.tsx` — короче в коде,
но менее консистентно с моделью «redactор это часть альбома».
Принимаем `album/[id]/layout` как из spec phase-2-spec §3.2.

### Локальный тип `LayoutData` вместо импорта `SmartFillLayout`

Тип `SmartFillLayout` определён в `app/app/page.tsx:708` (локально для
того файла). Чтобы не выносить его в общий module и не тянуть
связанные типы (`EnrichedWarning`, `WarningLevel`, `SmartFillSummary`),
дублируем минимально нужный shape в редакторе:

```typescript
type LayoutData = {
  layout_id: string
  template_set_id: string
  spreads: SpreadInstance[]
  // warnings и summary — не нужны редактору
}
```

Микро-дублирование в 4 строки оправдано — снижает связность файлов,
упрощает грэп. Если в 2.6.4 при auto-save потребуется передавать
больше полей — расширим.

### Параллельная загрузка трёх endpoint'ов через `Promise.all`

При открытии редактора нужны три запроса:
1. `GET /api/layout?action=album_layout` — текущий layout альбома
2. `GET /api/layout?action=template_set_detail&id=X` — placeholder'ы
   мастеров для рендера
3. `GET /api/tenant?action=album_photos` — фото для палитры (в 2.6.1
   ещё не используются, но загружаем сразу — палитра в 2.6.2 будет
   ждать только их)

Все три read-only, независимые. Запускаем через `Promise.all`. Время
загрузки = max(3 запросов), а не sum.

**Tricky момент:** для `template_set_detail` нужен `template_set_id`,
а он берётся из `album_layout` response. То есть последовательность:
1. Грузим `album_layout` (получаем `template_set_id`)
2. Параллельно грузим `template_set_detail` + `album_photos`

Реализация — через два await:

```typescript
const layoutRes = await api(`/api/layout?action=album_layout&album_id=${albumId}`)
// ... проверки ошибок ...
const layoutData = (await layoutRes.json()).layout
// Теперь у нас template_set_id
const [templateRes, photosRes] = await Promise.all([
  api(`/api/layout?action=template_set_detail&id=${layoutData.template_set_id}`),
  api(`/api/tenant?action=album_photos&album_id=${albumId}`),
])
```

Альтернатива «загрузить всё одним вызовом через новый endpoint»
не делаем — endpoint'ы уже есть, не плодим дубликатов.

### Один canvas в режиме edit, не строка миниатюр

В 2.6.1 показываем **только текущий spread** — большой, по центру
левой колонки. Между ними — навигация ◀▶ + индикатор «3 из 11».
Это отличается от `LayoutPreviewStrip` (миниатюры всех 11 в полосе),
и по spec phase-2-spec §4.5 редактор именно так выглядит.

**Размер canvas'а** считаем динамически:
- Левая колонка занимает 70% ширины окна (палитра — 30%)
- Минус padding'и ~40px
- Минус навигация снизу ~60px
- Получаем доступную область, в неё вписываем spread по высоте
  максимум 80vh

Простая формула в коде:
```typescript
const containerWidth = Math.min(
  windowWidth * 0.7 - 80,
  (windowHeight * 0.8) * (template.width_mm / template.height_mm),
)
```

Динамический resize при изменении окна — пока не делаем (не нужен
для 2.6.1, добавим в 2.6.5 если будет время).

### `mode='edit'` без drag handlers

`AlbumSpreadCanvas` принимает `mode: 'preview' | 'edit'`, влияет только
на `listening={mode === 'edit'}` в Stage. В 2.6.1 ставим `mode='edit'`
для будущего drag'а, но handler'ы (`onDrop`, `onSwap`) **не передаём** —
их добавим в 2.6.3 / 2.6.4.

Для пользователя в 2.6.1 это выглядит как «редактор открылся, можно
кликать-возиться, но ничего не происходит». Это ожидаемо.

### `useRouter` для возврата к альбому

Header содержит ссылку «← К альбому» которая возвращает на
`/app` с открытым модалом этого альбома. **Простой вариант:**
просто `router.push('/app')` — модал не откроется, партнёр сам
кликнет на нужный альбом. **Сложный вариант:** глубокая ссылка
типа `/app?album=<id>&tab=overview` — требует правок в `app/app/page.tsx`
для парсинга query params.

В 2.6.1 — простой вариант. Если будет неудобно (партнёр часто гоняет
туда-сюда) — улучшим в 2.6.5 или в фазе 3.

### `'use client'` обязательно

Страница использует:
- `useState` / `useEffect` — клиентские хуки
- `useRouter` из `next/navigation` — клиентский
- `AlbumSpreadCanvas` — Konva, требует `window.Image`

Поэтому строго `'use client'` сверху. Параллельно — динамический
импорт `AlbumSpreadCanvas` через `dynamic({ ssr: false })` уже есть
в `LayoutPreviewStrip.tsx` как паттерн, копируем.

### Загрузка как state, не как Server Component

Есть искушение использовать React Server Components / Next.js data
fetching для загрузки на сервере. **Не делаем** по двум причинам:
1. Авторизация в проекте через cookie + `/api/auth/me`. RSC потребует
   отдельной серверной авторизации.
2. Динамика данных (auto-save в 2.6.4) проще когда state на клиенте.

Используем тот же паттерн что в `app/app/page.tsx`: `'use client'` +
`useEffect` + `fetch` с `credentials: 'include'`.

### Заглушка палитры — текстовая

В 2.6.1 правая колонка должна **существовать** (для будущего pattern'а
flex layout), но содержимое — заглушка:

```tsx
<aside className="w-[30%] border-l border-gray-200 p-4 overflow-y-auto">
  <h2 className="text-sm font-semibold text-gray-700 mb-2">Палитра фото</h2>
  <p className="text-xs text-gray-500">
    {photos.length} фото загружено. Drag-and-drop появится в 2.6.3.
  </p>
</aside>
```

В 2.6.2 заменим на `PhotoPalette` компонент.

### Header кнопка-аутлайн «Назад»

Минимальный header:
```
← К альбому | Школа 89 — Layout редактор | (палитра справа)
```

Без `SaveIndicator` (он появится в 2.6.4) — пока «правки» сохранять
негде, индикатор бесполезный.

## Шаги

### Шаг 1. Создать страницу

Файл: `app/app/album/[id]/layout/page.tsx`

```tsx
'use client'

import { useEffect, useState } from 'react'
import { useRouter } from 'next/navigation'
import dynamic from 'next/dynamic'
import type {
  SpreadInstance,
  SpreadTemplate,
} from '@/lib/album-builder/types'

// Konva-компонент: SSR-incompatible (использует window.Image).
const AlbumSpreadCanvas = dynamic(
  () => import('../../../_components/AlbumSpreadCanvas'),
  { ssr: false, loading: () => null },
)

// ─── Тип loadable layout (минимальный shape для редактора) ───────────────
type LayoutData = {
  layout_id: string
  template_set_id: string
  spreads: SpreadInstance[]
}

// ─── Тип фото для палитры (соответствует 2.4 endpoint response) ──────────
type AlbumPhoto = {
  id: string
  filename: string
  storage_path: string
  thumb_path: string | null
  type: 'portrait' | 'group' | 'teacher' | null
  source: 'selections' | 'originals'
  child_ids: string[]
  teacher_ids: string[]
  selection_types: string[]
  url: string
  thumb_url: string
  created_at: string
}

// ─── Хелпер api() — копия из app/app/page.tsx, но без refresh-loop'а ─────
// (refresh-логика тут не нужна: при истёкшем токене страница редактора
// просто покажет ошибку и партнёр откроет её заново из /app)
async function api(path: string, opts?: RequestInit): Promise<Response> {
  return fetch(path, {
    ...opts,
    credentials: 'include',
    headers: { 'Content-Type': 'application/json', ...(opts?.headers ?? {}) },
  })
}

// ═════════════════════════════════════════════════════════════════════════
// LayoutEditorPage — fullscreen редактор layout'а альбома (фаза 2.6).
//
// В подэтапе 2.6.1 — только скелет: header, левая колонка с одним
// spread в edit-режиме, правая колонка-заглушка, навигация ◀▶ между
// разворотами. Drag-and-drop, палитра, auto-save — в подэтапах 2.6.2-4.
// ═════════════════════════════════════════════════════════════════════════
export default function LayoutEditorPage({
  params,
}: {
  params: { id: string }
}) {
  const router = useRouter()
  const albumId = params.id

  const [layout, setLayout] = useState<LayoutData | null>(null)
  const [templates, setTemplates] = useState<SpreadTemplate[]>([])
  const [photos, setPhotos] = useState<AlbumPhoto[]>([])
  const [albumTitle, setAlbumTitle] = useState<string>('')
  const [currentIdx, setCurrentIdx] = useState(0)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState<string | null>(null)

  // ─── Загрузка данных при монтировании ──────────────────────────────────
  useEffect(() => {
    let cancelled = false

    async function load() {
      try {
        // 1. Загружаем layout (получаем template_set_id)
        const layoutRes = await api(
          `/api/layout?action=album_layout&album_id=${albumId}`,
        )
        if (!layoutRes.ok) {
          throw new Error(`layout load failed: ${layoutRes.status}`)
        }
        const layoutJson = await layoutRes.json()
        if (!layoutJson.layout) {
          throw new Error(
            'Layout ещё не построен. Откройте альбом в кабинете и нажмите «Собрать автоматически».',
          )
        }
        const loadedLayout: LayoutData = {
          layout_id: layoutJson.layout.layout_id,
          template_set_id: layoutJson.layout.template_set_id,
          spreads: layoutJson.layout.spreads as SpreadInstance[],
        }

        // 2. Параллельно: template_set_detail + album_photos + album title
        const [templateRes, photosRes, albumRes] = await Promise.all([
          api(
            `/api/layout?action=template_set_detail&id=${loadedLayout.template_set_id}`,
          ),
          api(`/api/tenant?action=album_photos&album_id=${albumId}`),
          api(`/api/tenant?action=albums&album_id=${albumId}`),
        ])

        if (!templateRes.ok) {
          throw new Error(`template load failed: ${templateRes.status}`)
        }
        if (!photosRes.ok) {
          throw new Error(`photos load failed: ${photosRes.status}`)
        }

        const templateJson = await templateRes.json()
        const photosJson = await photosRes.json()
        // album endpoint можно проигнорировать если упал — это не блокер
        let title = ''
        if (albumRes.ok) {
          const albumJson = await albumRes.json()
          title = (Array.isArray(albumJson) ? albumJson[0]?.title : albumJson?.title) ?? ''
        }

        if (cancelled) return
        setLayout(loadedLayout)
        setTemplates(templateJson.spread_templates ?? [])
        setPhotos(photosJson.photos ?? [])
        setAlbumTitle(title)
        setLoading(false)
      } catch (e) {
        if (!cancelled) {
          setError((e as Error).message)
          setLoading(false)
        }
      }
    }

    load()
    return () => {
      cancelled = true
    }
  }, [albumId])

  // ─── Loading / error состояния ──────────────────────────────────────────
  if (loading) {
    return (
      <div className="min-h-screen flex items-center justify-center text-gray-500">
        Загружаем редактор…
      </div>
    )
  }

  if (error) {
    return (
      <div className="min-h-screen flex flex-col items-center justify-center gap-3 p-6">
        <p className="text-red-600 max-w-md text-center">{error}</p>
        <button
          type="button"
          onClick={() => router.push('/app')}
          className="text-sm px-4 py-2 rounded border border-gray-300 hover:bg-gray-50"
        >
          ← В кабинет
        </button>
      </div>
    )
  }

  if (!layout) {
    // Не должно случиться (loading=false + error=null = layout есть),
    // но TypeScript narrowing требует явной проверки.
    return null
  }

  // ─── Текущий spread + его template ──────────────────────────────────────
  const spreads = layout.spreads
  const currentSpread = spreads[currentIdx]
  const currentTemplate = templates.find(
    (t) => t.id === currentSpread?.template_id,
  )

  // ─── Размер canvas'а — фиксированный для 2.6.1, динамика в 2.6.5 ───────
  // 70% ширины окна минус палитра (30%) и padding'и. Берём 800px как
  // комфортный default — на ноутбуке 1440px это даст ~ окно редактора.
  const canvasContainerWidth = 800

  return (
    <div className="min-h-screen flex flex-col bg-gray-50">
      {/* ═══ Header ═══ */}
      <header className="bg-white border-b border-gray-200 px-6 py-3 flex items-center justify-between">
        <div className="flex items-center gap-4">
          <button
            type="button"
            onClick={() => router.push('/app')}
            className="text-sm text-gray-600 hover:text-gray-900"
          >
            ← К альбому
          </button>
          <h1 className="text-base font-semibold text-gray-900">
            {albumTitle || 'Альбом'}
          </h1>
          <span className="text-xs text-gray-400">— Layout редактор</span>
        </div>
        {/* SaveIndicator появится в 2.6.4 */}
      </header>

      {/* ═══ Main: левая колонка (canvas) + правая (палитра) ═══ */}
      <div className="flex-1 flex overflow-hidden">
        {/* ─── Левая колонка: canvas + навигация ─── */}
        <main className="flex-1 flex flex-col items-center justify-center p-6 overflow-auto">
          {currentSpread && currentTemplate ? (
            <>
              <div className="bg-white rounded shadow-sm border border-gray-200">
                <AlbumSpreadCanvas
                  instance={currentSpread}
                  template={currentTemplate}
                  containerWidth={canvasContainerWidth}
                  mode="edit"
                />
              </div>

              {/* Навигация */}
              <div className="mt-4 flex items-center gap-3">
                <button
                  type="button"
                  onClick={() => setCurrentIdx((i) => Math.max(0, i - 1))}
                  disabled={currentIdx === 0}
                  className="px-3 py-1.5 text-sm rounded border border-gray-300 bg-white hover:bg-gray-50 disabled:opacity-50 disabled:cursor-not-allowed"
                >
                  ◀ Назад
                </button>
                <span className="text-sm text-gray-600">
                  Разворот {currentIdx + 1} из {spreads.length}
                </span>
                <button
                  type="button"
                  onClick={() =>
                    setCurrentIdx((i) => Math.min(spreads.length - 1, i + 1))
                  }
                  disabled={currentIdx >= spreads.length - 1}
                  className="px-3 py-1.5 text-sm rounded border border-gray-300 bg-white hover:bg-gray-50 disabled:opacity-50 disabled:cursor-not-allowed"
                >
                  Вперёд ▶
                </button>
              </div>
            </>
          ) : (
            <p className="text-gray-500">Шаблон не найден для текущего разворота</p>
          )}
        </main>

        {/* ─── Правая колонка: палитра (заглушка в 2.6.1) ─── */}
        <aside className="w-[30%] min-w-[280px] max-w-[400px] bg-white border-l border-gray-200 p-4 overflow-y-auto">
          <h2 className="text-sm font-semibold text-gray-700 mb-2">
            Палитра фото
          </h2>
          <p className="text-xs text-gray-500">
            {photos.length} фото загружено. Drag-and-drop появится в 2.6.3.
          </p>
        </aside>
      </div>
    </div>
  )
}
```

**Замечания по реализации:**

1. **`api(/api/tenant?action=albums&album_id=...)`** — для загрузки названия альбома.
   Этот action существует в `app/api/tenant/route.ts` (используется главной
   страницей `/app`). Если он там отдаёт массив — берём `[0].title`. Если
   объект — `albumJson.title`. Хелпер `Array.isArray` обрабатывает оба случая.

2. **`api()` хелпер локальный, без refresh-loop'а** — главная `/app/page.tsx`
   импортирует refresh-token логику; на странице редактора такая глубокая
   обработка избыточна. Если токен истёк — пользователь увидит ошибку и
   откроет редактор заново из `/app` (где уже refresh обработается).

3. **`useState<LayoutData | null>(null)`** + явный narrowing
   `if (!layout) return null` — стандартный TS-pattern для async-loaded state.

4. **`canvasContainerWidth = 800`** — захардкожено в 2.6.1. Динамика
   через `window.innerWidth + ResizeObserver` в 2.6.5 если будет нужно.

5. **`useEffect cancellation flag`** — стандартная защита от race condition'ов
   при unmount во время fetch (паттерн уже в `LayoutPreviewStrip.tsx`).

6. **404 NoSuchKey битых фото** — на canvas в edit-режиме они будут так же,
   как и в LayoutPreviewStrip (2.3): пустой серый прямоугольник. Это
   ожидаемое поведение для smoke-альбомов.

## Применение и верификация в проде

### Порядок

Миграций нет. После push Vercel задеплоит за ~30-60 секунд.

### Проверка вручную

После деплоя:

1. Открой в браузере **`https://yearbook-v2.vercel.app/app/album/3fccf1dc-3e3d-402b-8c0c-7036e3496418/layout`**
   (это URL Красночетайки + `/layout`).
2. Должно появиться:
   - Header «← К альбому | Красночетайская СОШ — Layout редактор»
   - Большой canvas в центре экрана с одним spread'ом
   - Навигация снизу: «◀ Назад | Разворот 1 из 11 | Вперёд ▶»
   - Правая панель «Палитра фото — 179 фото загружено. Drag-and-drop появится в 2.6.3.»
3. Нажми «Вперёд ▶» — canvas должен переключиться на следующий spread,
   подпись «Разворот 2 из 11».
4. Нажми «◀ Назад» — обратно к первому. Кнопка «Назад» должна быть
   disabled когда разворот = 1, кнопка «Вперёд» — disabled когда
   разворот = последний.
5. Нажми «← К альбому» — возврат на `/app`.

### Что **не** проверяем в 2.6.1

- Drag-and-drop — нет, это 2.6.3
- Палитра с миниатюрами — нет, это 2.6.2
- Save indicator — нет, это 2.6.4
- Правый клик / сохранение состояния — нет
- Кнопка «Открыть редактор» в Обзоре альбома — пока **disabled**,
  активируем в 2.6.5

### Что **может** не работать (если всплывёт — фиксим в 2.6.1-followup)

- **Cookie не передаётся** на новой странице — должен передаваться
  автоматически через `credentials: 'include'`. Если 401 на любом
  endpoint'е — открой DevTools Network, проверь Cookie header.
- **Action `albums` в `/api/tenant`** возвращает не то что ожидаем —
  заголовок альбома может быть пустым строкой (мы это игнорируем).
  Если поле title пустое — скорее всего endpoint возвращает другой
  формат, не блокер для 2.6.1.
- **`GET ?action=album_layout` 404 для альбома без layout** — обработали
  через сообщение «Layout ещё не построен». Партнёр должен сначала
  «Собрать автоматически» в Обзоре.

## После 2.6.1 — что разблокируется

После 2.6.1 у нас:
- Страница `/app/album/[id]/layout` существует и открывается
- Загружает три источника данных параллельно
- Показывает один spread в edit-режиме
- Навигация ◀▶ между разворотами работает

Это разблокирует подэтапы 2.6.2 (палитра компонент) и 2.6.3 (drag-and-drop).
Они могут идти последовательно или параллельно — палитра не зависит от
DnD, и наоборот canvas уже готов принимать `mode='edit'`.

После 2.6.5 редактор полностью функционален. Идут 2.7 (confirm) и 2.8 (smoke).
