# Инструкция для подэтапа 0.11.1

> Это временный файл-инструкция. После выполнения подэтапа удалить и закоммитить вместе с основными изменениями.

## Цель подэтапа

Реализовать комплектации **Лайт** и **Мини** для классов ≤24 учеников. Без overflow-логики (она в 0.11.2). Включить в реализацию **adaptive grid** — алгоритм который выбирает мастер с минимальной достаточной ёмкостью среди доступных в БД. Сейчас в БД только L-6 и N-12 (адаптивные L-2/L-3/L-4 и N-4/N-6/N-9 — задача дизайнеру по master-cleanup-tz §A1/A2). Алгоритм должен корректно работать на текущей БД (degraded — все классы ≤24 идут через L-6/N-12 с пустыми слотами + warning), и автоматически подхватывать адаптивные мастера когда они появятся в БД.

Один коммит:
```
feat(layout/0.11.1): Лайт и Мини ≤24 учеников + adaptive grid
```

## Архитектурная суть

Лайт и Мини отличаются от Медиума двумя вещами:
1. **Фиксированное число базовых страниц** — 4 для Лайт, 2 для Мини (независимо от числа учеников ≤24)
2. **Адаптивная сетка** — `slotsPerPage` подбирается так чтобы все ученики уместились на эти базовые страницы

Из build_album.jsx (строки 2052-2065):
```javascript
function getLightGrid(n) {
    if (n <= 8)  return { slotsPerPage: 2, ... };  // 4 × 2 = 8
    if (n <= 12) return { slotsPerPage: 3, ... };  // 4 × 3 = 12
    if (n <= 16) return { slotsPerPage: 4, ... };  // 4 × 4 = 16
    return       { slotsPerPage: 6, ... };          // 4 × 6 = 24
}
function getMiniGrid(n) {
    if (n <= 9)  return { slotsPerPage: 4, ... };   // 2 × 4 = 8 — не хватает для 9
    if (n <= 12) return { slotsPerPage: 6, ... };   // 2 × 6 = 12
    if (n <= 18) return { slotsPerPage: 9, ... };   // 2 × 9 = 18
    return       { slotsPerPage: 12, ... };          // 2 × 12 = 24
}
```

Заметь нюанс в Мини: при 9 учениках slotsPerPage=4 даёт всего 8 слотов на 2 страницы. 9-й ученик НЕ помещается на 2 базовые страницы. Реальная логика jsx — последний ученик на overflow-странице. **Это уже часть overflow-логики (0.11.2).** В 0.11.1 для Мини ≤8 → slotsPerPage=4, 10-12 → 6, 13-18 → 9, 19-24 → 12. Класс из 9 учеников (overflow case) — пропускаем, в 0.11.2 разберём.

В **adaptive grid алгоритме** (наш TS-вариант) логика обобщается:
```typescript

  const requiredCapacity = Math.ceil(totalStudents / basePages);

  // if none — pick last (max) + warning 'adaptive_grid_fallback'
```

Этот алгоритм эквивалентен `getLightGrid`/`getMiniGrid` при наличии нужных мастеров в БД. При отсутствии (только L-6/N-12) — fallback на максимум.

### 3.3 Новая функция `pickAdaptiveGrid`

Добавить ПЕРЕД `buildAdaptiveGridStudents`:

```typescript
/**
 * Выбирает grid-мастера так, чтобы base_pages вмещали total учеников.
 *
 * Считает required_capacity = ceil(total / base_pages). Среди кандидатов
 * (мастера с заданным page_role и applies_to_config) выбирает с минимальной
 * slot_capacity.students >= required_capacity.
 *
 * Если ни один не подходит (например, total=24, base_pages=4 → required=6,
 * но в БД только L-6 → подходит) — стандартный путь. Если total=24, base_pages=4,
 * но в БД только мастер students=4 → ни один не >= 6 → возвращаем последний
 * (максимальный) + warning 'adaptive_grid_fallback'.
 *
 * Если кандидатов вообще нет — возвращает null (вызывающая сторона пишет
 * 'master_not_found').
 */
function pickAdaptiveGrid(
  ctx: BuildContext,
  baseFilter: MasterFilter,
  totalStudents: number,
  basePages: number,
): SpreadTemplate | null {
  const filter = { ...baseFilter, applies_to_config: ctx.config.config_type };
  const requiredCapacity = Math.ceil(totalStudents / basePages);

  // Все кандидаты с заданным page_role и applies_to_config (без фильтра по
  // slot_capacity_min — мы сами найдём минимальный достаточный).
  const candidates = ctx.config.template_set.spreads.filter((s) => {
    if (s.page_role !== filter.page_role) return false;
    if (s.applies_to_configs.indexOf(filter.applies_to_config) < 0) return false;
    return s.slot_capacity !== null && typeof s.slot_capacity.students === 'number';
  });

  if (candidates.length === 0) return null;

  // Сортируем по slot_capacity.students по возрастанию
  candidates.sort((a, b) => {
    const aCap = a.slot_capacity?.students ?? 0;
    const bCap = b.slot_capacity?.students ?? 0;
    return aCap - bCap;
  });

  // Минимальный достаточный
  for (let i = 0; i < candidates.length; i++) {
    const cap = candidates[i].slot_capacity?.students ?? 0;
    if (cap >= requiredCapacity) {
      return candidates[i];
    }
  }

  // Никто не достаточно — берём максимальный + warning
  const fallback = candidates[candidates.length - 1];
  pushWarning(ctx, {
    code: 'adaptive_grid_fallback',
    detail: `${ctx.config.config_type}: required capacity=${requiredCapacity} (для ${totalStudents} учеников на ${basePages} страниц), max available=${fallback.slot_capacity?.students ?? 0}, выбран ${fallback.name}`,
  });
  return fallback;
}
```

### 3.4 Новая функция `buildAdaptiveGridStudents`

После `pickAdaptiveGrid`:

```typescript
/**
 * Лайт/Мини — фиксированное base_pages, adaptive grid выбор мастеров.
 *
 * 1. Находим LEFT и RIGHT мастеров через pickAdaptiveGrid (по обоим базовый
 *    фильтр из scenarios + applies_to_config + page_role).
 * 2. Делим учеников на base_pages кусков (chunk).
 * 3. Чередуем Left/Right по чётности индекса страницы.
 *
 * Overflow (>24 учеников) — НЕ обрабатывается в этой функции, пишется warning
 * 'students_overflow_grid' и обрезка до base_pages*slotsPerPage. Полноценный
 * overflow в 0.11.2.
 */
function buildAdaptiveGridStudents(
  ctx: BuildContext,
  section: StudentSection,
): void {
  if (!section.base_pages || section.base_pages < 1) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: 'adaptive_grid scenario requires base_pages >= 1',
    });
    return;
  }
  if (!section.grid_filter_left || !section.grid_filter_right) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: 'adaptive_grid scenario requires grid_filter_left and grid_filter_right',
    });
    return;
  }

  const total = ctx.input.students.length;
  const basePages = section.base_pages;

  const leftMaster = pickAdaptiveGrid(ctx, section.grid_filter_left, total, basePages);
  const rightMaster = pickAdaptiveGrid(ctx, section.grid_filter_right, total, basePages);

  if (!leftMaster) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: `adaptive_grid LEFT: no candidates for page_role=${section.grid_filter_left.page_role} applies_to=${ctx.config.config_type}`,
    });
    return;
  }
  if (!rightMaster) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: `adaptive_grid RIGHT: no candidates for page_role=${section.grid_filter_right.page_role} applies_to=${ctx.config.config_type}`,
    });
    return;
  }

  const slotsPerPage = leftMaster.slot_capacity?.students ?? 0;
  if (slotsPerPage < 1) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: `adaptive_grid LEFT: ${leftMaster.name} has slot_capacity.students < 1`,
    });
    return;
  }

  // Overflow check (0.11.1 — degraded, обрезка до base_pages * slotsPerPage)
  const maxOnBase = basePages * slotsPerPage;
  if (total > maxOnBase) {
    pushWarning(ctx, {
      code: 'students_overflow',
      detail: `${ctx.config.config_type}: total=${total} > base_pages*slotsPerPage=${maxOnBase}, overflow обрезан (полная overflow-логика в 0.11.2)`,
    });
  }

  const studentsToPlace = total > maxOnBase
    ? ctx.input.students.slice(0, maxOnBase)
    : ctx.input.students;

  // Раскладываем по страницам
  const hasQuote = section.has_quote === true;
  for (let i = 0; i < basePages; i++) {
    const slice = studentsToPlace.slice(i * slotsPerPage, (i + 1) * slotsPerPage);
    if (slice.length === 0) break; // нет больше учеников
    const isLeft = i % 2 === 0;
    const master = isLeft ? leftMaster : rightMaster;

    ctx.spreads.push({
      spread_index: ctx.spreadCounter.value++,
      template_id: master.id,
      template_name: master.name,
      data: buildGridStudentData(slice, slotsPerPage, hasQuote),
    });
  }
}
```
