# Инструкция для подэтапа 0.11.3 — Индивидуальный

> Это временный файл-инструкция. После выполнения подэтапа удалить и закоммитить вместе с основными изменениями.

## Цель подэтапа

Реализовать комплектацию **Индивидуальный** (`config_type: 'individual'`) — седьмую и предпоследнюю комплектацию.

Структура:
1. **Личный раздел** — для каждого ученика пара (E-Max-Left + E-Ind-Right-3 ИЛИ E-Max-Right). Per-student выбор правого мастера зависит от `friend_photos.length`:
   - `≤ 3` фото → **E-Ind-Right-3** (default_for_configs=['individual'], 3 слота)
   - `= 4` фото → **E-Max-Right** (4 слота, fallback из Максимума)
2. **Сетка-миниатюр** — после всех личных разворотов, как в Мини: adaptive_grid с N-* мастерами, базовых страниц 2, overflow по уже реализованной логике.

Один коммит:
```
feat(layout/0.11.3): Индивидуальный (E-Max+E-Ind личные + N-* сетка миниатюр)
```

## Состояние БД (проверено через SELECT в 0.11.1.5)

```
E-Max-Left:    page_role=student_left,  default_for_configs=['maximum'],   slot_capacity={students:1}
E-Max-Right:   page_role=student_right, default_for_configs=['maximum'],   slot_capacity={students:1, photos_friend:4}
E-Ind-Right-3: page_role=student_right, default_for_configs=['individual'], slot_capacity={students:1, photos_friend:3}
N-12-Left:     page_role=student_grid_left,  default_for_configs=['mini','individual'], slot_capacity={students:12}
N-12-Right:    page_role=student_grid_right, default_for_configs=['mini','individual'], slot_capacity={students:12}
N-Overflow-Row: page_role=student_overflow,  default_for_configs=['mini','individual'], slot_capacity={students:4, photos_full:1}
```

**ВАЖНО:** E-Max-Left имеет `default_for_configs=['maximum']`, а нам нужно использовать его в Индивидуальном тоже. Решение: нужна **миграция БД** — расширить default_for_configs E-Max-Left до `['maximum','individual']`.

Аналогично, E-Max-Right — у него default только `['maximum']`, но в Индивидуальном он используется как fallback для учеников с 4 фото. Нужно добавить 'individual'.

И ещё: у `E-Max-Left/Right` поле `applies_to_configs = ['maximum', 'individual']` уже широкое — это останется. Меняем только `default_for_configs`.

Никаких миграций для остальных мастеров не нужно. E-Ind-Right-3 уже `['individual']`. N-* уже `['mini','individual']`.

## 1. `migrations/2026-05-07-default-individual-e-max.sql`

```sql
-- Миграция 0.11.3 — расширение default_for_configs для E-Max-Left/Right
-- чтобы они автоматически выбирались в Индивидуальном.
--
-- В Индивидуальном:
--   E-Max-Left  — левая страница (портрет + имя)
--   E-Max-Right — правая страница для учеников с 4 фото-друзьями
--   E-Ind-Right-3 — правая страница для учеников с ≤3 фото-друзьями
--
-- E-Ind-Right-3 уже имеет default_for_configs=['individual'].
-- Расширяем E-Max-Left/Right с ['maximum'] до ['maximum','individual'].

UPDATE spread_templates
SET default_for_configs = ARRAY['maximum', 'individual']
WHERE template_set_id = (SELECT id FROM template_sets WHERE slug = 'okeybook-default')
  AND name IN ('E-Max-Left', 'E-Max-Right');

-- Проверка
SELECT name, default_for_configs
FROM spread_templates
WHERE template_set_id = (SELECT id FROM template_sets WHERE slug = 'okeybook-default')
  AND name LIKE 'E-Max-%' OR name = 'E-Ind-Right-3'
ORDER BY name;
```

### 4.3 Новая функция `buildIndividualStudents`

Добавить ПОСЛЕ `buildMaximumStudents`:

```typescript
/**
 * Индивидуальный — для каждого ученика пара (E-Max-Left + правый мастер).
 *
 * Правый мастер выбирается per-student по friend_photos.length:
 *   ≤ 3 → E-Ind-Right-3 (3 слота фото-друзей)
 *   = 4 → E-Max-Right (4 слота)
 *
 * Если для конкретного ученика нужный мастер не нашёлся — warning
 * master_not_found + ученик пропускается (не падаем целиком).
 */
function buildIndividualStudents(
  ctx: BuildContext,
  section: IndividualStudentSection,
): void {
  const leftR = findMaster(ctx.config.template_set.spreads, {
    ...section.left_filter,
    applies_to_config: ctx.config.config_type,
  });
  const right3R = findMaster(ctx.config.template_set.spreads, {
    ...section.right_filter_3,
    applies_to_config: ctx.config.config_type,
  });
  const right4R = findMaster(ctx.config.template_set.spreads, {
    ...section.right_filter_4,
    applies_to_config: ctx.config.config_type,
  });

  if (!leftR.ok) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: `individual left: ${section.left_filter.expected_name_hint ?? '?'}`,
    });
    return;
  }
  if (leftR.warning) pushWarning(ctx, leftR.warning);
  if (right3R.ok && right3R.warning) pushWarning(ctx, right3R.warning);
  if (right4R.ok && right4R.warning) pushWarning(ctx, right4R.warning);

  // Если оба правых мастера не найдены — секция бессмысленна
  if (!right3R.ok && !right4R.ok) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: 'individual: ни right_filter_3, ни right_filter_4 не дают мастера',
    });
    return;
  }

  for (let i = 0; i < ctx.input.students.length; i++) {
    const s = ctx.input.students[i];

    // Левая страница — всегда E-Max-Left
    ctx.spreads.push({
      spread_index: ctx.spreadCounter.value++,
      template_id: leftR.master.id,
      template_name: leftR.master.name,
      data: {
        studentportrait: s.portrait,
        studentname: s.full_name,
      },
    });

    // Правая страница — выбор по friend_photos.length
    const friendCount = s.friend_photos.length;
    let rightMaster: SpreadTemplate | null = null;
    let friendSlots = 0;

    if (friendCount <= 3 && right3R.ok) {
      rightMaster = right3R.master;
      friendSlots = 3;
    } else if (friendCount === 4 && right4R.ok) {
      rightMaster = right4R.master;
      friendSlots = 4;
    } else if (friendCount === 4 && !right4R.ok && right3R.ok) {
      // Fallback: 4 фото есть, но E-Max-Right не нашёлся → используем E-Ind-Right-3 (теряем 1 фото) + warning
      rightMaster = right3R.master;
      friendSlots = 3;
      pushWarning(ctx, {
        code: 'master_not_found',
        detail: `individual right: student ${s.full_name} has 4 friend_photos, но E-Max-Right не найден; используется E-Ind-Right-3 (4-е фото отброшено)`,
      });
    } else if (friendCount > 4) {
      // Никогда не должно быть — но защита
      rightMaster = right4R.ok ? right4R.master : right3R.master;
      friendSlots = right4R.ok ? 4 : 3;
      pushWarning(ctx, {
        code: 'students_overflow',
        detail: `individual: student ${s.full_name} has ${friendCount} friend_photos, обрезано до ${friendSlots}`,
      });
    }

    if (!rightMaster) {
      pushWarning(ctx, {
        code: 'master_not_found',
        detail: `individual right: student ${s.full_name} (friend_photos=${friendCount}) — нет подходящего мастера`,
      });
      continue;
    }

    ctx.spreads.push({
      spread_index: ctx.spreadCounter.value++,
      template_id: rightMaster.id,
      template_name: rightMaster.name,
      data: studentSinglePageData(s, friendSlots),
    });
  }
}
```
