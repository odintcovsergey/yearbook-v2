## Принципиальные решения

### Финальный smoke — на одном тест-альбоме

Альтернатива: smoke на нескольких альбомах с разными конфигурациями
(standard-layflat, universal-soft, individual-layflat). Это ~30-60
минут ручного тестирования.

**Делаем точечный smoke** на одном тест-альбоме с пресетом
`universal-layflat`. Цель — убедиться что workflow «end-to-end»
работает после всех 17 коммитов вместе. Глубокое тестирование разных
конфигураций — ответственность фазы 3+ при реальных партнёрах.

Сценарий smoke:
1. Открыть `/app` → тестовый альбом → Обзор
2. Нажать «Пересобрать» (если уже собран)
3. Увидеть LayoutPreviewStrip с миниатюрами
4. Нажать «Открыть редактор» → переход в редактор
5. Сделать минимум 2 действия:
   - Drag фото из палитры → placeholder (auto-save)
   - Swap между placeholder'ами (auto-save)
6. Дождаться «✓ Сохранено»
7. Кликнуть «← К альбому» → обратно в модал альбома (deep link)
8. Видеть свои изменения в LayoutPreviewStrip миниатюрах
9. Нажать «Пересобрать» → confirm-диалог
10. Cancel → правки сохранены
11. Снова «Пересобрать» → OK → пересборка, флаг сброшен
12. SQL: `has_user_edits = false`

Не делаем итеративно с инструкциями — Сергей сам прогонит этот
сценарий после применения 2.8.

### v44 — копия v43 с целевыми правками, не переписать с нуля

v43 (~989 строк) содержит много исторической информации (mémoire-файлы
Claude Code, паттерны кода, миграции). Не имеет смысла переписывать
заново. Копируем v43 целиком и обновляем только конкретные секции.

Конкретные правки:
1. **Шапка** (строки 1-46): версия v44, дата 09.05.2026, статус
   обновлён, описание «что нового в v44 относительно v43»
2. **Раздел «ПРОДУКТ B»**: добавить секцию «Фаза 2 закрыта» с
   полным описанием 17 коммитов, списком новых файлов, артефактов
3. **Раздел «ТАБЛИЦЫ БД»**: добавить упоминание что
   `album_layouts.has_user_edits` колонка появилась в фазе 2
4. **Раздел «ЧТО ДАЛЬШЕ»**: убрать упоминание фазы 2, поставить
   фокус на фазе 3 (PDF) или альтернативных приоритетах
5. **Раздел «ВАЖНЫЕ ФАЙЛЫ»**: добавить новые файлы фазы 2
   (`AlbumSpreadCanvas.tsx`, `LayoutPreviewStrip.tsx`,
   `PhotoPalette.tsx`, `SaveIndicator.tsx`,
   `album/[id]/layout/page.tsx`)
6. **Раздел про текущую секцию подэтапов**: пометить фазу 2 как
   полностью закрытую

### Инструкция точечная, без полного текста v44

Полный текст v44 — ~1000 строк, прорастёт инструкцию до 1500+ строк.
Вместо этого — **точные diff'ы по секциям**: «найти в v43 такой-то
блок, заменить на такой-то в v44».

Альтернативно: я мог бы сгенерировать v44 целиком прямо в инструкцию.
Но это в основном перезапись v43, которая не несёт ценности
(копирование). Точечные правки яснее показывают **что** изменилось.

## Применение и верификация

### Финальный smoke на проде

После push (Vercel деплой не нужен — кода не меняли) Сергей сам
прогоняет полный smoke на тест-альбоме:

1. **Открыть `https://yearbook-v2.vercel.app/app`** → тестовый альбом «тест»
2. **Обзор** → видны миниатюры разворотов в LayoutPreviewStrip
3. **Кликнуть «Открыть редактор»** → переход на `/app/album/<UUID>/layout`
4. **Header показывает «✓ Сохранено»** при первой загрузке
5. **Drag любого фото** из палитры в любой пустой placeholder
6. **Header → «● Не сохранено»** через секунду
7. **Подождать 2с** → «Сохраняется…» → «✓ Сохранено»
8. **Перейти ◀▶ на разворот с двумя фото**
9. **Hover на placeholder с фото** → курсор `move`
10. **Drag фото в другой placeholder** → swap, фото меняются местами
11. **Дождаться save**
12. **Кликнуть «← К альбому»** → URL `?album=<UUID>`, модал альбома открыт
13. **В Обзоре видны актуальные миниатюры** с правками
14. **Нажать «Пересобрать»** → диалог браузера «У вас есть ручные правки..»
15. **Cancel** → правки на месте
16. **Снова «Пересобрать» → OK** → пересборка, правки потеряны
17. **SQL** в Supabase: `has_user_edits = false`

Если все 17 шагов прошли — **фаза 2 closed** ✅.
