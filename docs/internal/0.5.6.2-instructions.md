# Инструкция для подэтапа 0.5.6.2

**Цель:** UI override пресета per-child в expanded row карточки ученика.
Закрывает фазу 0.5.6.

**Зачем:** Поле `children.config_preset_id` существует с 0.5.1 (FK NULL=
использовать альбомный пресет, иначе override). UI пока нет. Use case —
например, ученик доплатил за «Максимум» в классе с базовой комплектацией
«Стандарт».

**Что меняется:**
- `app/api/tenant/route.ts` — новый action `update_child_preset` +
  расширение SELECT в action=children
- `app/app/page.tsx` — Child type, state презетов в AlbumDetailModal,
  dropdown в expanded row, чип в основной строке

---

## Принципиальные решения

### Default = «Использовать пресет альбома»

Sentinel-опция первой в dropdown'е: `<option value="">Использовать пресет альбома</option>`.
Это и есть NULL в БД. Большинство учеников будут именно так.

### Один dropdown на 15 опций (sentinel + 14 пресетов)

В отличие от формы альбома (где 2 dropdown'а), в expanded row места
меньше — один dropdown компактнее. Опции в формате
`«Стандарт (твёрдые листы)»` и т.д.

### Кнопка «Применить»

Не auto-save при изменении dropdown'а (защита от случайных кликов).
Пользователь меняет dropdown → видит подсветку «Изменения не сохранены»
→ нажимает «Применить» → сохраняется.

### Чип в основной строке таблицы

Если override установлен — показать рядом с именем ученика мелкий бейдж
`Максимум` (только имя пресета). Без override чипа нет — обычная строка.
Это даёт быстрый scan кто из учеников имеет нестандартный пресет.

### Кэш списка пресетов

Загрузка `presets_list` через apiVA при открытии AlbumDetailModal
параллельно с детьми. Один запрос на сессию модала.

### Шаг 2. API: новый action `update_child_preset`

Добавить новый блок ПОСЛЕ `delete_child` (~строка 1749) или ПЕРЕД
секцией УЧИТЕЛЯ. Паттерн копирует reset_child:

```typescript
// ----------------------------------------------------------
// update_child_preset — назначить override config_preset для ученика
// ----------------------------------------------------------
if (body.action === 'update_child_preset') {
  const { child_id, preset_slug } = body
  if (!child_id) {
    return NextResponse.json({ error: 'child_id обязателен' }, { status: 400 })
  }

  if (!(await assertChildAccess(auth, child_id))) {
    return NextResponse.json({ error: 'Ученик не найден' }, { status: 404 })
  }

  // preset_slug: непустая строка → resolve в id; пустая/null/undefined → сброс override (NULL)
  let configPresetId: string | null = null
  if (typeof preset_slug === 'string' && preset_slug.length > 0) {
    const preset = await resolvePresetBySlug(preset_slug)
    if (!preset) {
      return NextResponse.json(
        { error: `preset_slug ${preset_slug} not found` },
        { status: 400 },
      )
    }
    configPresetId = preset.id
  }

  const { error } = await supabaseAdmin
    .from('children')
    .update({ config_preset_id: configPresetId })
    .eq('id', child_id)

  if (error) {
    return NextResponse.json({ error: error.message }, { status: 500 })
  }

  await logAction(auth, 'child.update_preset', 'child', child_id, {
    preset_slug: preset_slug ?? null,
  })

  return NextResponse.json({ ok: true })
}
```

`resolvePresetBySlug` уже добавлен в 0.5.6.1 — переиспользуем.

### Шаг 5. UI: компонент `ChildPresetEditor` (или inline-блок)

Чтобы не загромождать expanded row, имеет смысл выделить логику в
отдельный inline-блок. Добавить ПОСЛЕ блока кнопок «Действия для X»
(после строки ~1207 в page.tsx) внутри `{canEdit && (...)}`:

```jsx
{/* Override пресета для ученика */}
<div className="flex flex-wrap items-center gap-2 mb-3 mt-2 pt-3 border-t border-gray-200">
  <span className="text-xs text-gray-500 mr-1">
    Пресет вёрстки:
  </span>
  <ChildPresetSelect
    child={c}
    presets={presets}
    albumPresetName={album.config_preset_name ?? null}
    onChange={async (newSlug) => {
      try {
        const r = await apiVA('/api/tenant', {
          method: 'POST',
          body: JSON.stringify({
            action: 'update_child_preset',
            child_id: c.id,
            preset_slug: newSlug,  // '' для сброса override
          }),
        })
        if (!r.ok) {
          const err = await r.json().catch(() => ({ error: 'unknown' }))
          onError(err.error ?? 'Ошибка обновления пресета')
          return
        }
        // Обновить локальное состояние children
        setChildren(prev => prev.map(ch => {
          if (ch.id !== c.id) return ch
          if (newSlug === '') {
            return { ...ch, config_preset_id: null, config_preset_slug: null, config_preset_name: null }
          }
          const found = presets.find(p => p.slug === newSlug)
          return {
            ...ch,
            config_preset_id: found?.id ?? null,
            config_preset_slug: found?.slug ?? null,
            config_preset_name: found?.name ?? null,
          }
        }))
        onNotify(`Пресет обновлён для ${c.full_name}`)
      } catch (e) {
        onError(e instanceof Error ? e.message : 'network error')
      }
    }}
  />
</div>
```

И определить компонент `ChildPresetSelect` (где-то рядом с другими
helper-компонентами AlbumDetailModal — например после `MiniStat` или
перед AlbumFormModal):

```typescript
function ChildPresetSelect({
  child,
  presets,
  albumPresetName,
  onChange,
}: {
  child: Child
  presets: PresetOption[]
  albumPresetName: string | null
  onChange: (newSlug: string) => Promise<void>
}) {
  // Текущее значение: если override установлен → его slug, иначе пустая строка
  const currentSlug = child.config_preset_slug ?? ''
  const [draftSlug, setDraftSlug] = useState<string>(currentSlug)
  const [busy, setBusy] = useState(false)

  // Синхронизация если child обновился извне
  useEffect(() => {
    setDraftSlug(child.config_preset_slug ?? '')
  }, [child.config_preset_slug])

  const dirty = draftSlug !== currentSlug

  return (
    <>
      <select
        value={draftSlug}
        onChange={(e) => setDraftSlug(e.target.value)}
        disabled={busy}
        className="text-xs px-2 py-1 border rounded"
      >
        <option value="">
          Использовать пресет альбома{albumPresetName ? ` (${albumPresetName})` : ' (не задан)'}
        </option>
        {presets.map(p => (
          <option key={p.slug} value={p.slug}>
            {p.name}
          </option>
        ))}
      </select>
      <button
        type="button"
        disabled={!dirty || busy}
        onClick={async () => {
          setBusy(true)
          try {
            await onChange(draftSlug)
          } finally {
            setBusy(false)
          }
        }}
        className="text-xs btn-secondary px-3 py-1.5 disabled:opacity-50"
      >
        {busy ? '...' : 'Применить'}
      </button>
      {dirty && (
        <span className="text-xs text-amber-600">Не сохранено</span>
      )}
    </>
  )
}
```
