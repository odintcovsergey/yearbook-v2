# Инструкция для подэтапа 0.11.0

> Это временный файл-инструкция. После выполнения подэтапа удалить и закоммитить вместе с основными изменениями.

## Цель подэтапа

Подготовить инфраструктуру для **soft-печати** (мягкие листы) и реализовать **S-Intro** — вступительную страницу для всех soft-альбомов кроме Mini-soft.

Это первый из четырёх подэтапов 0.11. Не добавляет новых комплектаций, только инфраструктуру:
- Переименование `SCENARIOS_LAYFLAT` → `SCENARIOS` (один словарь на оба print_type)
- Удаление поля `print_type` из `ScenarioDef` (теперь сценарий описывает обе печати)
- Новое поле `soft_overrides?: SoftOverrides` для override-логики soft vs layflat (вариант C из обсуждения)
- Новый тип `IntroSection` + функция `buildIntroSection`
- Soft-overrides для всех 4 текущих комплектаций (standard, universal, maximum, medium) с `intro_section: INTRO_SECTION_S_INTRO`

Один коммит:
```
feat(layout/0.11.0): soft-печать инфраструктура + S-Intro для standard/universal/maximum/medium
```

## Ключевые факты из БД (проверено через SELECT)

- **S-Intro**: page_role='intro', applies_to_configs=['standard','universal','maximum','medium','light','mini','individual','tryumo'], slot_capacity={photos_full:1}, is_fallback=false, mirror_for_soft=false
- **S-Intro-Old**: applies_to_configs=[] (пустой массив!), is_fallback=true, page_role='intro' — отсеивается фильтром `applies_to_config: 'standard'` потому что 'standard' не в []

То есть фильтр найдёт только S-Intro, что нам и нужно. S-Intro-Old остаётся как fallback для будущего.

### 2.4 Новая функция `buildIntroSection`

Добавить в build.ts (например, перед `buildTeacherSection`):

```typescript
/**
 * Сборка вступительной страницы S-Intro (только для soft-печати).
 *
 * S-Intro — одностраничный мастер с classphotoframe placeholder'ом.
 * Фото класса заполняется из common_photos.full_class[0]; если массив
 * пуст — null + warning class_photo_missing.
 *
 * Если S-Intro мастер не найден — warning + return (продолжаем без intro).
 */
function buildIntroSection(ctx: BuildContext, section: IntroSection): void {
  const filter = { ...section.filter, applies_to_config: ctx.config.config_type };
  const r = findMaster(ctx.config.template_set.spreads, filter);
  if (!r.ok) {
    pushWarning(ctx, {
      code: 'master_not_found',
      detail: `intro: ${section.filter.expected_name_hint ?? '?'}`,
    });
    return;
  }
  if (r.warning) pushWarning(ctx, r.warning);

  const data: Record<string, string | null> = {};
  if (hasPlaceholder(r.master, 'classphotoframe')) {
    const fc = ctx.input.common_photos.full_class;
    if (fc.length >= 1) {
      data.classphotoframe = fc[0];
    } else {
      data.classphotoframe = null;
      pushWarning(ctx, {
        code: 'class_photo_missing',
        detail: `master ${r.master.name} has classphotoframe but common_photos.full_class is empty`,
      });
    }
  }

  ctx.spreads.push({
    spread_index: ctx.spreadCounter.value++,
    template_id: r.master.id,
    template_name: r.master.name,
    data,
  });
}
```

`hasPlaceholder` уже существует в build.ts (добавлен в 0.10b.2). Не дублировать.
