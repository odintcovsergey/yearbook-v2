# 📸 Выпускной альбом — система отбора фотографий v2

## Что умеет система

- Родитель открывает уникальную ссылку на телефоне, без регистрации
- 6 шагов: контакт → портрет для страницы → обложка (с доплатой) → текст → 2 фото с друзьями → подтверждение
- Фото с друзьями — эксклюзивный выбор: выбранное одним родителем недоступно другим
- Ответственный родитель заполняет данные учителей
- Администратор управляет всем через /admin
- Экспорт CSV для автовёрстки в InDesign

## 🍎 Установка на Mac — пошагово

### Шаг 1. Установить Node.js

1. Откройте браузер, зайдите на **nodejs.org**
2. Скачайте версию с надписью "LTS" (большая зелёная кнопка)
3. Откройте скачанный файл `.pkg` и установите как обычную программу
4. Готово. Проверить: откройте **Терминал** (найдите через Spotlight: ⌘+Пробел, напишите "Терминал") и введите:
   ```
   node --version
   ```
   Должна появиться версия, например `v20.11.0`

### Шаг 2. Создать проект на Supabase (база данных + хранилище фото)

1. Зайдите на **supabase.com** → Sign Up (бесплатно)
2. Нажмите **New project**, введите название и пароль (запомните!)
3. Подождите 1–2 минуты пока проект создаётся
4. Перейдите в **SQL Editor** (левое меню) → нажмите **New query**
5. Скопируйте весь текст из файла `schema.sql` → вставьте → нажмите **Run**
6. Перейдите в **Storage** (левое меню) → **New bucket**:
   - Name: `photos`
   - Public bucket: **ВКЛ** (переключите!)
   - Нажмите **Create bucket**
7. Перейдите в **Settings → API** (левое меню) — здесь нужны три значения:
   - **Project URL** — начинается с `https://`
   - **anon public** key — длинная строка
   - **service_role** key — ещё одна длинная строка (держите в тайне!)

### Шаг 3. Настроить проект

1. Распакуйте папку `yearbook-v2` в удобное место (например, Рабочий стол)
2. В папке найдите файл `.env.example`
3. Скопируйте его и переименуйте копию в `.env.local`

   > ⚠️ Файл начинается с точки — он может быть скрыт. Если не видите:
   > в Finder нажмите **Cmd+Shift+.** чтобы показать скрытые файлы

4. Откройте `.env.local` в TextEdit (ПКМ → Открыть в программе → TextEdit)
5. Вставьте значения из Supabase:
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://abcdefgh.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
   SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...
   ADMIN_SECRET=МойСложныйПароль2025
   ```

### Шаг 4. Запустить на своём компьютере (для проверки)

1. Откройте **Терминал**
2. Перейдите в папку проекта (замените путь на свой):
   ```bash
   cd ~/Desktop/yearbook-v2
   ```
3. Установите зависимости (один раз):
   ```bash
   npm install
   ```
   Подождите 1–2 минуты
4. Запустите:
   ```bash
   npm run dev
   ```
5. Откройте браузер и зайдите на **http://localhost:3000/admin**
6. Введите пароль из `ADMIN_SECRET` — вы в системе!

### Шаг 5. Опубликовать в интернете (Vercel)

1. Зайдите на **vercel.com** → Sign Up (бесплатно, войдите через GitHub)
2. Нажмите **Add New → Project**
3. Нажмите **Browse** и выберите папку `yearbook-v2`
   (или загрузите через GitHub — нажмите **Import Git Repository**)
4. В разделе **Environment Variables** добавьте те же 4 переменные из `.env.local`
5. Нажмите **Deploy**
6. Через 2–3 минуты получите ссылку типа `https://yearbook-abc123.vercel.app`

## 🚀 Как работать с системой

### 1. Создать альбом
`/admin` → Создать альбом → заполните название, классы, режим обложки, дедлайн

### 2. Загрузить фотографии
Вкладка **Фото** → выберите тип → загружайте пачками

### 3. Разметить кто на каком фото
Подготовьте CSV:
```csv
child_name,photo_filename
Иванов Иван,IMG_001.jpg
Петрова Маша,IMG_001.jpg
```
Вкладка **CSV** → загрузите файл

### 4. Добавить учеников
Вкладка **Ученики** → добавляйте по одному или импортируйте (скоро)

### 5. Разослать ссылки
Вкладка **Ученики** → **Копировать ссылку** напротив каждого → отправьте в WhatsApp

### 6. Следить за прогрессом
Вкладка **Обзор** — видите кто выбрал, кто нет, сколько доплат

### 7. Экспортировать результат
**Обзор** → **Экспорт для вёрстки** → CSV с колонками под InDesign

## ❓ Частые вопросы

**Можно ли родителю поменять выбор?**
До нажатия "Подтвердить" — да, сколько угодно. После — нет. Если надо разблокировать — удалите `submitted_at` в Supabase для нужного ребёнка.

**Что если два родителя одновременно выбирают одно фото?**
Первый блокирует, второй видит фото как серое и недоступное.

**Как удалить тестовые данные перед боевым запуском?**
В Supabase → SQL Editor выполните:
```sql
delete from selections; delete from parent_contacts;
delete from student_texts; delete from cover_selections;
delete from photo_locks;
update children set submitted_at = null;
```

**Где хранятся фотографии?**
В Supabase Storage (bucket `photos`). Бесплатный план — 1 ГБ. Для 30 ГБ нужен Pro ($25/мес).

  [token]/page.tsx        — интерфейс родителя (6 шагов)
