## Контекст задачи

Дизайнер сверстала cover-мастера ПРАВИЛЬНО — единым полотном на ОДНОЙ широкой
мастер-странице (задняя слева, передняя справа, ~434×296 мм, bleed 18 мм). Но
парсер этого не понимал: `computeCoverZones` принимал только 2–3 страницы и при
одной широкой возвращал `null`, а коллизии меток получали суффиксы `_left/_right`.

Тестовый файл: `Мастер_Аква_меч_обложки.idml` (3 мастера: `C-Cover-Portrait`,
`C-Cover-Common`, `C-Cover-Design`). На момент изменения файла В РЕПО НЕТ — Сергей
приложит; интеграционный тест на него написан и пропускается (skip), пока файла нет.

## ✅ DONE — распознавание зон единого полотна (только парсер)

Изменены `lib/idml-converter/extract-geometry.ts` и `parse.ts`:

- **Новый случай n===1** (одна широкая страница). Зоны теперь считаются ВНУТРИ
  `extractPlaceholders` (нужны координаты слотов): добавлены `assignCoverZones`
  (диспетчер по числу страниц), `assignSinglePageCoverZones` (обёртка с валидацией)
  и чистая экспортируемая `computeCoverZonesSinglePage(page, slots)`.
- **Граница задняя/передняя — по контенту**: правый край самого правого `back_*`
  слота и левый край самого левого `cover_*`, граница посередине между ними. Если
  неоднозначно (нет таких слотов или перекрываются) — fallback на геометрическую
  середину рабочей ширины. Рабочая ширина = trim-страница (GeometricBounds), bleed
  в неё не входит по построению InDesign.
- **Зона слота — по X-центру** относительно границы: левее → `back`, правее →
  `front`. Корешок (spine) на полотне номинально 0 — реальная ширина подставляется
  при сборке (`computeSpineWidthMm`). Декор (`__under/__over/__fg`) распределяется
  по тому же X-правилу.
- **Валидация**: `back_*` ожидаются в back, `cover_*` — во front; расхождение →
  warning, парс не падает.
- **Суффиксы коллизий меток для обложек — по ЗОНЕ** (`_back/_spine/_front`), а не
  `_left/_right`. Единое полотно больше не делится на _Left/_Right. `dedupeLabels`
  получил флаг `isCover` и берёт суффикс из `ph.zone`.
- **Совместимость**: случаи n===2 (facing) и n===3 (с корешком) работают как
  раньше — `computeCoverZones(pages_x_ranges)` не тронут, зона = страница по x.
- `extractPlaceholders` теперь возвращает `{ placeholders, coverZones }`;
  единственный вызыватель — `parse.ts`. Из `parse.ts` убран прямой вызов
  `computeCoverZones` (логика ушла в extractPlaceholders).
