## Контекст задачи

Сергей попросил добавить в редактор обложек недостающие элементы управления,
которые есть в редакторе разворотов (приложил скрины): навигация ◀/▶, кнопка
«Стили текстов», «Вид» (полноэкранный просмотр), «Отменить»/«Повторить».

## ✅ DONE (этот инкремент)

Всё в `app/app/album/[id]/cover/page.tsx` + новый компонент.

1. **Навигация между обложками** — строка «◀ Назад / Обложка N из M / Вперёд ▶»
   под холстом. Хелпер `goToIdx(i)` (кламп + сброс кроп/текст/панель), им же
   пользуется лента-миниатюр.
2. **«Вид» — полноэкранный просмотр** — новый компонент
   `app/app/_components/CoverPreviewFullscreen.tsx` (аналог LayoutPreviewFullscreen,
   но для обложек): CoverCanvas mode='preview', листание ←/→/свайп, Esc, счётчик,
   fit-to-screen по аспекту (back+spine+front)/height. Кнопка «Вид» в шапке.
3. **Отменить / Повторить (undo/redo)** — история снимков {typePatches,
   studentPatches}. `pushHistory()` перед каждым действием (setTypeKey/
   setStudentKeys/handleTextStyleChange); кроп-жест коалесцируется через
   suspendHistoryRef (onGestureStart/End) → один шаг на жест. Undo/redo
   восстанавливают снимок И до-сохраняют изменившиеся области на сервер
   (`saveChangedScopes`, diff по JSON). Горячие клавиши Ctrl/⌘+Z, Ctrl/⌘+Shift+Z
   (не перехватываются при инлайн-редактировании текста и в «Виде»). История
   сбрасывается при загрузке. Кнопки в шапке со счётчиками. «Сохранено» → «✓ Сохранено».

## ⚠️ «Стили текстов» — НЕ добавлено осознанно

Глобальная модалка AlbumTextStylesModal стилизует только 4 СЕМАНТИЧЕСКИЕ группы
разворотов (studentname/studentquote/teachername/teacherrole), которые
`detectTextStyleGroup()` определяет по меткам. Метки обложек (`cover_student_name`,
`cover_title`, …) НЕ попадают ни в одну группу → кнопка была бы НЕРАБОЧЕЙ для
обложек. Для обложек уже есть поэлементная панель стиля (v192). Если нужна
ГЛОБАЛЬНАЯ стилизация текстов обложки — это отдельная фича (новые группы/поле),
не «добавить кнопку».

## ⚠️ Декор «Фёдорова Варвара» (из v192) — проверить вживую

Фикс v192 гасит декор-метки в null. На скрине Сергея от 18.06 текст ещё был —
вероятно скрин ДО деплоя. ВАЖНО проверить после деплоя: на C-Cover-Portrait
показывается реальное имя ученика (напр. «Смирнов Дмитрий») ИЛИ имя пропало.
Если пропало — значит в мастере настоящий слот имени назван `cover_student_name__under`
(метка декора), и это ошибка вёрстки → дизайнеру переименовать в `cover_student_name`.
