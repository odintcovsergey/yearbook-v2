# Система отбора фотографий для выпускных альбомов
# Обновлено: 08.04.2026 (вечер)

---

## КТО Я И ЧТО МЫ ДЕЛАЕМ

Я — фотограф/организатор выпускных альбомов. У меня штат фотографов и я работаю
с внешними фотографами (ретушь + вёрстка + печать). Мы делаем веб-систему для
отбора фотографий родителями вместо текущего неудобного процесса через Google Диск.

---

## ТИП АЛЬБОМА — «УНИВЕРСАЛ»

Структура альбома:
1. Обложка (портрет или общее фото класса или надпись)
2. Раздел учителей (ФИО, должность, портрет)
3. Личные страницы каждого ученика (1 страница на ученика)
4. Общий раздел (выбираем сами, вне системы)

Личная страница ученика содержит:
- 1 портрет (выбирает родитель из общего пула портретов класса)
- ФИ ученика
- Текст до 500 символов (пишет родитель)
- 2 фото с друзьями (выбирает родитель из пула 70-120 фото)

Правила блокировки фото:
- Портреты: общий пул для всего класса. 1 портрет = 1 ребёнок.
  Выбранные видны всем но серые и недоступны.
- Групповые фото: то же самое. 1 фото = 1 ребёнок. Эксклюзивный выбор.
- CSV разметка для портретов оставлена для будущей автовёрстки.
- Планируется вариант где одно фото могут выбрать несколько учеников — на будущее.

## СЦЕНАРИЙ — 6 ШАГОВ ДЛЯ РОДИТЕЛЯ

Шаг 1 — Контакты (имя родителя + телефон для допродаж)
Шаг 2 — Портрет для личной страницы (общий пул, блокировка как у групповых)
Шаг 3 — Портрет для обложки (4 варианта режима):
  - cover_mode = 'none'     → шаг пропускается
  - cover_mode = 'same'     → автоматически тот же что на странице
  - cover_mode = 'optional' → родитель выбирает: без портрета (бесплатно)
                              или с портретом (тот же или другой, +300р)
  - cover_mode = 'required' → все платят доплату
Шаг 4 — Текст от ученика (до 500 символов, необязательно)
Шаг 5 — 2 фото с друзьями (эксклюзивная блокировка)
Шаг 6 — Подтверждение (после этого изменить нельзя)

Отдельный сценарий: Ответственный родитель заполняет данные учителей
(ФИО, должность, портрет для каждого учителя).

## ИЗВЕСТНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

**Проблема:** Supabase новые ключи не работают
**Решение:** Legacy ключи из Settings → API Keys → Legacy tab

**Проблема:** Терминал открывается в домашней папке
**Решение:** cd ~/Desktop/yearbook-v2

**Проблема:** Фото загружены в Storage но не в базе
**Решение:** SQL запрос:
  insert into photos (album_id, filename, storage_path, type)
  select
    'ALBUM_ID',
    split_part(name, '/', 3),
    name,
    'portrait'
  from storage.objects
  where bucket_id = 'photos'
  and name like 'ALBUM_ID/portrait/%'
  and name not in (select storage_path from photos);

**Проблема:** storage_path в базе дублирует album_id
**Решение:** проверять что path = 'album_id/type/timestamp_filename.jpg'
