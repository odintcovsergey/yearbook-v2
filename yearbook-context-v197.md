## ✅ DONE

### 1. Расширен `printers.config` (jsonb, без миграции схемы printers)
- **lib/printers/types.ts**: к `sheet_types` добавлены `formats[]` (id, name,
  family `vertical_rect|square|horizontal`, page_w/h_mm, spread_w/h_px, work_w/h_mm,
  bleed_mm, safe_mm), `accept_mode` (`spread|page`), `file_format` (`jpeg|pdf`),
  `color` (`srgb`), `cover` (`flap_lr_mm`/`flap_tb_mm`). Тип листа теперь несёт
  `spine: { mode, ranges?, formula?, fixed_mm? }`. Поле `spine_ranges` помечено
  deprecated, но читается (legacy).

### 2. Режимы корешка (lib/printers/spine.ts)
- `resolveSpineMm(config, sheetTypeId, spreadCount)` — НОВОЕ имя, учитывает 3
  режима:
  - `ranges` — lookup по диапазонам (как было);
  - `formula` — `base_mm + step_mm × (разворотов / per_spreads)` (Булгак: +1мм/разв.);
  - `fixed` — постоянная ширина (ОкейКнига; Принт Мейтс = 0).
- `normalizeSpine(sheet)` — приводит legacy `spine_ranges` к `{mode:'ranges'}`.
- `resolveSpineFromRanges` оставлен как **@deprecated**-алиас (делегирует в
  resolveSpineMm). Вызов в `lib/cover/load-covers.ts` переключён на resolveSpineMm.
- Тесты: `lib/printers/__tests__/spine.test.ts` — 15 кейсов (ranges/formula/fixed
  + legacy-совместимость). Формула Булгака даёт base+1мм/разворот, ПМ fixed=0.

### 3. UI профиля типографии (app/super/printers/page.tsx)
Секции в карточке типографии: **Форматы блока** (добавить/удалить, семейство +
все размеры), **Приём и файл** (разворотами/постранично, JPEG/PDF, цвет), **Обложка
— загибы** (Л/П, В/Н), **Типы листов → корешок** (на каждый тип — селектор режима
диапазоны/формула/фикс с соответствующими полями). Старые профили (только
spine_ranges) открываются и нормализуются к ranges.

### 4. Выбор в заказе (app/app/page.tsx + app/api/tenant/route.ts)
- FormData: добавлено `format_id` (рядом с printer_id/sheet_type_id).
- UI-блок «Типография (формат и корешок)»: при выборе типографии показываются
  селекторы формата (если есть `config.formats`) и типа листа. Смена типографии
  сбрасывает и format_id, и sheet_type_id.
- API tenant: `format_id` в insert и в whitelist allowedFields (update).

### 5. Наполнение 4 типографий (scripts/seed-printers.ts)
Идемпотентный seed глобальных типографий (tenant_id NULL): Фабрика Фотокниги
(ranges по 3 типам листа, форматы 20×20/21×30, загиб 18/17), Булгак (formula
+1мм/разв., 5 форматов, загиб 15+safe10), Принт Мейтс (fixed 0, 4 формата),
Окей Книга (приём постранично, file_format=pdf, fixed, 2 формата).
**NB:** числа параметрические — точные диапазоны корешка/work-зоны/недостающие px
оставлены ЗАГЛУШКАМИ, Сергей сверит со справочником и поправит в UI.
