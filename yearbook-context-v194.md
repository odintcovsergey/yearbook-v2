## ✅ DONE (этот инкремент)

1. **lib/cover/text-styles.ts** — смысловые группы обложечного текста:
   title / name / subtitle / details (школа+город+год+класс) / spine / contacts.
   `detectCoverTextGroup(label)`, `parseCoverTextStyleOverrides(raw)`,
   `applyCoverTextStyles(data, placeholders, overrides)`. Ключевой приём:
   глобальные стили накладываются как НИЖНИЙ слой служебных ключей
   (__fontSize__/__color__/__halign__/__valign__/__font__), точечная правка
   (ключ уже в data) ПОБЕЖДАЕТ. Так не трогаем общий AlbumSpreadCanvas — он уже
   умеет точечные ключи.
2. **CoverCanvas** — проп `coverTextStyles`, применяет applyCoverTextStyles к
   data перед рендером. **CoverPreviewFullscreen** прокидывает его дальше.
3. **CoverTextStylesModal.tsx** — модалка по группам (шрифт/размер/цвет/h+v
   выравнивание, «сброс группы», «сбросить всё»), как у разворотов но для
   обложек. Оптимистичный preview + сохранение.
4. **lib/cover/load-editor.ts** — читает albums.cover_text_style_overrides
   (устойчиво к отсутствию колонки), возвращает `coverTextStyles`.
5. **API** `app/api/tenant`: action `cover_save_text_styles` (валидация object|null
   → update albums.cover_text_style_overrides). cover_editor уже отдаёт через
   loadCoverEditor.
6. **Редактор обложек** (page.tsx): кнопка «Стили текстов» в шапке, state
   coverTextStyles, модалка, проброс в CoverCanvas + CoverPreviewFullscreen.
7. Тесты lib/cover/__tests__/text-styles.test.ts (7).

## Как это сочетается с поэлементной панелью (v192)

- Поэлементная TextStylePanel пишет точечные ключи в scope='type' (cover_edits).
- Глобальная модалка пишет albums.cover_text_style_overrides (на весь заказ).
- Приоритет: точечная правка > глобальная > placeholder из IDML (как в разворотах).
- Панель показывает только точечный override (не глобальный) — консистентно с
  разворотами; «По умолчанию» в панели снимает точку → остаётся глобальный.
