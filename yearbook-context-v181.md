## ✅ DONE — Расширение каталога мастеров под набор «Аква меч»

**Проблема.** Дизайнерский набор «Аква меч» содержал мастера, которых не было в
реестре `lib/idml-converter/family-mapping.ts` → они грузились с
`slot_capacity = null`, и движок их не видел. Имена у дизайнера корректные —
расширили каталог под них. Связано с авто-зеркалом page-any (v180).

**Принцип совместимости (соблюдён):** ничего из okeybook-default не сломано.
Старые имена (`J-Quarter-Left/Right`, `J-Collage-4`, `G-Teachers-3x3/3x4/4x4`)
оставлены рабочими; новое добавлено рядом, с фолбэком. БД-схема не менялась.

### A. `J-Quarter` → симметричный `page-any`

- **Реестр:** добавлена запись `J-Quarter` (page-any, common, photos_quarter:2).
  `J-Quarter-Left/-Right` оставлены (совместимость).
- **`lib/rule-engine/sections/common.ts`:** у quarter-шага автопака появилось
  поле `preferAny: 'J-Quarter'`. Если `J-Quarter` есть в template_set — берём
  его на ОБЕ стороны (правую отдаёт авто-зеркало page-any), иначе старый путь
  с парой Left/-Right. См. `pickAutopackPage`.

### B. Коллажи `J-Collage-3 / 5 / 6` (был только 4)

- **Реестр:** добавлены 3/5/6 (page-any, common, photos_collage:N).
- **`common.ts`:** один collage-шаг автопака развёрнут в лестницу 6→5→4→3.
  Автопак берёт первый шаг, где хватает collage-фото И мастер есть в наборе →
  «самый крупный помещающийся из присутствующих» получается сам (как grid
  у учеников). `J-Collage-4` остаётся одним из вариантов.
- **Побочный фикс:** при откате правой страницы автопак не восстанавливал
  `available.collage` (предсуществующий баг утечки) — починено; collage добавлен
  в счётчик `common_autopack_underflow`.

### C. Учительская сетка на 6 — `G-Teachers-3x2`

- **Реестр:** добавлена `G-Teachers-3x2` (page-right, teacher_right, teachers:6,
  grid_modes 6/3x2).
- **`lib/rule-engine/sections/teachers.ts`:** новый helper
  `findCompactSubjectRightGrid` — для 5-8 предметников ищет компактную правую
  сетку (`min_fit` по teacher_right) и принимает её ТОЛЬКО если ёмкость < 9
  (крупные 3x3/3x4/4x4 принадлежат тиру 9+). Если нашлась (3x2): главный учитель
  один на левую (F-Head-WithPhoto), предметники → правая сетка. Иначе старый
  путь (F-Head-LargeGrid слева + общее фото справа). Решение согласовано в
  `pickLeftMaster` и `pickRightMaster` (оба зовут helper с тем же subjects).
- **Защита okeybook-default:** там нет 3x2 → `min_fit` находит только сетки
  cap ≥ 9 → отвергаются → поведение прежнее (предметники на левой LargeGrid).
