# knowledge/ — база знаний проекта yearbook-v2

Локальная резервная база знаний: «живой» снимок состояния проекта на диске и в Git
(два зеркала — GitHub + GitFlic). Нужна, чтобы быстро вспомнить контекст, поднять
проект с нуля и не терять историю решений, если что-то случится с машиной или
с подробными контекст-файлами.

## Структура

| Файл | Что внутри |
|---|---|
| `README.md` | этот файл — что это и как обновляется |
| `PROJECT_CONTEXT.md` | текущее состояние проекта (синхронизируется со свежим `yearbook-context-vN.md` + `MEMORY.md`) |
| `DECISIONS.md` | ключевые архитектурные/продуктовые решения с датами |
| `RECOVERY.md` | как восстановить проект с нуля (клон обоих зеркал, env-переменные, деплой) |
| `sessions/YYYY-MM-DD.md` | лог по датам: что делали за сессию |

## Как обновляется

В **конце каждой рабочей сессии** Claude Code:
1. Обновляет `PROJECT_CONTEXT.md` (актуальное состояние).
2. Дописывает `sessions/<сегодня>.md` (что сделано за сессию).
3. Коммитит и пушит в оба зеркала ОТДЕЛЬНЫМ коммитом:
   `docs(knowledge): обновление после сессии <дата>`.

Это ручное обновление силами Claude Code (не cron): происходит только когда идёт
работа с Claude Code. Подробнее правило — в корневом `CLAUDE.md` (раздел «База знаний»).

## Чего здесь НЕТ

Секретов, токенов, ключей и содержимого `.env*`. В `RECOVERY.md` — только НАЗВАНИЯ
переменных окружения, без значений.

## Где полная история

- Подробные контекст-файлы по этапам: `yearbook-context-v*.md` в корне (история не
  переписывается, каждый новый ссылается на предыдущий).
- Память Claude Code: `~/.claude/projects/.../memory/` (`MEMORY.md` — индекс).
- Рабочий гайд: корневой `CLAUDE.md`.
