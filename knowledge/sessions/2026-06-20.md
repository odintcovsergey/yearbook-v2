# Сессия 2026-06-20 — Типографская выгрузка + ускорение экспорта

ТЗ: ускорение рендера PDF + выгрузка под профиль типографии (нарезка по книгам,
имена, разворотами/постранично, формат, zip). Решено с Сергеем: сначала
**PDF-часть**, JPG — отдельным заходом (нужен растеризатор, в проекте только
sharp+pdf-lib). Постранично = сквозная нумерация страниц. UI = отдельная кнопка.

Ветка `feat/export-acceleration-naming` (НЕ в main — ждёт preview-прогон).

## Сделано (всё зелёное: 1107 тестов + smoke, tsc, build)

1. **Ускорение фото** (`perf(export)`): `lib/pdf-export/photo-embed.ts` —
   `collectPhotoUrlsFromSpreads` + `prefetchPhotoSources` (пул ~8, кэш по url с
   дедупом Promise'ов) + префетч в `pipeline.ts` до цикла рендера. Логика выбора
   оригинал/selection и ресэмплинг не изменены → вид PDF тот же. Чинит
   прод-таймаут экспорта (РИСК №1 из load-readiness).

2. **Нарезка per-student (модель)** (`feat(export)`): метка `personal`
   пробрасывается из движка в сохранённую вёрстку (`layout-to-buildresult.ts` +
   поле в album-builder `SpreadInstance`). `lib/export-typography/plan.ts` —
   `planTypographyExport`: нарезка на файлы по книгам (000 общая / 00X личные),
   имена КНИГА-НОМЕР, приём spread/page, рез is_spread на половины. 7 тестов.
   ⚠️ Старые альбомы нужно ПЕРЕСОБРАТЬ, чтобы метка появилась → личные книги.

3. **Рендер под формат** (`feat(export)`): `lib/export-typography/adapt.ts`
   (`adaptTemplateSetToFormat`); `pipeline.ts renderTypographyUnits` — по
   файлу=1 страница; «разворотами» = `renderCombinedWidePage` склеивает два
   разных мастера в один широкий холст (вылеты только по внешним краям);
   `index.ts exportAlbumTypography` — рендер→нарезка на одностраничные PDF.
   Рефакторинг: `buildRenderContext`/`resolveEffectivePlaceholders` вынесены,
   обычный PDF-экспорт функционально не тронут. Smoke-тест (5 кейсов) реально
   прогоняет рендер.

4. **Endpoint + zip** (`feat(export)`): `POST /api/layout?action=export_typography`
   — читает `printers.config` заказа (формат+приём), лимит 50 разворотов с
   честным сообщением, zip в YC, signed url. `handleExportPdf` не тронут.

5. **UI** (`feat(export)`): кнопка «🗂 В типографию (zip)» в `ExportPanel`.

6. **Обложки в zip** (`feat(export)`): `lib/export-typography/covers.ts`
   (`buildCoverRenderUnits`) — layoutCover (плавающий корешок) +
   adaptCoverToFormat + дедуп (портретные по людям → 00X-00, общая/дизайн →
   один 000-00). `pipeline.renderCoverPage` — НОВЫЙ серверный PDF-рендер
   обложки (раньше был только Konva-канвас + SVG-превью): фон + плейсхолдеры
   через общий drawPlaceholder. Номер 00X согласован с внутр. книгами через
   одинаковый порядок детей (class, full_name + фильтр покупки = student_index).
   UI показывает число обложек. Тесты: имена/дедуп/нумерация + smoke рендера.

7. **JPG-вывод** (`feat(export)`): `lib/pdf-export/rasterize.ts` —
   растеризация одностраничного PDF→JPEG 300dpi через **pdfjs-dist +
   @napi-rs/canvas** (НОВЫЕ зависимости!). Растеризуем НАШ же рендер → JPG=PDF
   1:1. Ленивый dynamic import; в next.config `serverComponentsExternalPackages`
   (иначе webpack падает на нативном .node). КРИТИЧНО: кириллица рендерится
   только при subset:false шрифтах (loadFonts так и делает; с subset:true pdfjs
   роняет глифы — проверено визуально). Мягкий fallback на PDF при сбое.
   endpoint читает file_format из printers.config; UI показывает PDF/JPG.

8. **Фиксы таймаута** (`fix(export)`): `export const maxDuration = 300` на роуте
   (без него Vercel рубил по короткому дефолту); параллельный ресэмплинг фото
   (resamplePhotoBuffer вынесен, prefetchResampledPhotos готовит ресэмплы пулом
   до рендера в resampledCache — применено и к обычному PDF, и к типографии);
   параллельная JPG-растеризация. Вид PDF не меняется.

## 🛑 РЕШЕНИЕ: работа по экспорту ОСТАНОВЛЕНА до переезда на Timeweb

Живой прогон на preview вскрыл стену: **бесплатный Vercel (Hobby) рубит функцию
на 60с**, а рендер даже 4 плотных разворотов (сетки по 12 фото) = ~55–70с →
«Failed to fetch»/500. `maxDuration=300` на Hobby обрезается до 60с. Это РИСК №1,
тупик бесплатного тарифа.

**Диагноз (важно):** код экспорта РАБОЧИЙ — локально на «Тест 24» собирает
правильные файлы (проверено: разворот отрендерился верно, PDF ~55с / JPG ~70с).
Проблема ЧИСТО во времени (лимит Vercel). Возможен ещё hard-crash native
@napi-rs/canvas при JPG на Vercel (не докопано — остановились). Старый
«Экспортировать» (один PDF) на проде работает — main не трогали.

**Решение Сергея (20.06):** НЕ дорабатывать экспорт под Vercel (ни «по кусочкам»,
ни платный Pro). Переезжаем на **Timeweb** в ближайшее время, экспорт делаем
**фоновой очередью уже там, без лимита времени**. Ветка
`feat/export-acceleration-naming` остаётся как есть (всё закоммичено, тесты
зелёные), в main НЕ вливаем. Новых веток/задач по экспорту до переезда не берём.
Ждём отдельного ТЗ на переезд. См. [[project_migration_to_ru]],
[[project_load_readiness]].

## НЕ сделано (после переезда / отдельные ТЗ)

- **Экспорт фоновой очередью** на своём сервере (Timeweb) — снимает лимит 60с.
- Вылеты (bleed/overhang) обложки + глобальные стили текстов обложки.
- Живой прогон типографской выгрузки целиком (упёрлись в лимит Vercel).

## Грабли (обложки)

- Серверного PDF-рендера обложки НЕ БЫЛО — только клиентский Konva
  (CoverCanvas) + SVG-превью (preview-svg.ts). Сделан новый рендер,
  переиспользует drawPlaceholder.
- Согласованность номера обложки 00X с внутренней книгой: обложки привязаны
  к child_id, внутренние книги к student_index движка. child_id НЕ доходит до
  движка (AlbumInput.Student без id; legacy-adapter роняет). Решено
  обходным путём — нумерация обложек тем же детерминированным порядком
  (class, full_name + filterChildrenByPurchase), что даёт student_index.
  Фрагильно при изменении порядка/мультисекциях — проверить вживую.

## Грабли

- `splitIntoBooks` (`lib/album-split`) работает со СТАРЫМ слоем (rule-engine
  SpreadInstance с left/right PageInstance), а PDF-рендер — со слоем
  album-builder (template_id+data). Для выгрузки сделан отдельный
  `planTypographyExport` на слое сохранённой вёрстки. Переиспользованы только
  хелперы имён (`formatBookId`/`bookFileName`).
- `.select('a, b' + 'c')` в supabase-js ломает вывод типов (GenericStringError)
  — select должен быть ОДНИМ строковым литералом.
