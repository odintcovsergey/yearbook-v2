# Сессия 2026-06-25 — фоны обложек в превью после переезда на Timeweb (Safari)

## Итог
Фоны (и портреты/декор) в миниатюрах обложек во вкладке «Обзор» **грузятся** —
проверено Сергеем в Safari без VPN. Причина оказалась многослойной, чинили по
слоям; финал — **same-origin прокси картинок превью** `/api/img`. Всё в проде
(main `4cee510`), оба зеркала.

## Хронология причин (почему «?» имел разные корни на разных экранах)
1. **Старый западный адрес фона.** `covers.background_url` хранит supabase-URL
   (`bnotiyhamfyllcrqwquq.supabase.co`) — из РФ без VPN заблокирован. Отсюда
   «фон только с VPN». Сервер теперь конвертит его в российский Timeweb при чтении.
2. **Сырой ключ в правках.** Портрет «синий ?» у учеников с поштучной правкой:
   `cover_edits` хранят `cover_portrait` КЛЮЧОМ; клиентский патч перекрывал
   подписанную базу сырым ключом → `<image href="ключ">` → 404. **Фотографии НЕ
   терялись — баг показа** (поэтому «сброс обложки Петровой» не понадобился).
3. **Safari + инлайн-SVG.** Картинки из чужого origin (хранилище) Safari в
   `<image>` грузил нестабильно: без CORS-заголовка не рисует, а Timeweb отдаёт
   `Access-Control-Allow-Origin` ТОЛЬКО на запрос с `Origin` (supabase слал всегда
   → раньше с supabase-URL было видно). `crossorigin="anonymous"` не помог —
   переводил запрос в CORS-режим и конфликтовал с no-cors кэшем холста (фон один
   раз загружен холстом без CORS → Safari отдавал тот кэш на cors-запрос из SVG →
   CORS-ошибка). Ни с `crossorigin`, ни без него стабильно не выходило.

## Коммиты (по слоям, все в проде)
- `7fd758a` — `lib/cover/resign-bg.ts`: пере-подпись фона из любой формы (ключ /
  старый supabase-URL / протухший presigned → свежий `resolveReadUrl`);
  `signCoverBg` проверяет карту подписей ПЕРВОЙ. Починило фон в редакторе (Konva)
  и превью разворотов.
- `1719dae` — анти-кэш (`cache:'no-store'`+`&_t=`) на запрос `cover_album_preview`.
- `76e4576` — `crossorigin`+`xlink:href` (ОТКАЧЕНО ниже, crossorigin ломал).
- `cd0da81` — `load-editor.ts`: подпись фото-меток в правках (`byType/byChild`
  через `resignCoverPhotoData`). Фикс «синий ?» портретов. Сохранение заново
  keyify-ит (`cover_save_edit`) → база остаётся на ключах.
- `0853f1c` — убрал `crossorigin` (он и ломал фон).
- `4cee510` ⭐ **ФИНАЛ** — `app/api/img/route.ts` (same-origin прокси) +
  `lib/cover/preview-svg.ts` заворачивает ВСЕ `<image>` в `/api/img?u=<подпись>`.
  Картинки превью идут с нашего домена → для Safari same-origin → ни CORS, ни
  кэш-граблей. Прокси: только `image/*`, белый список хостов (анти-SSRF),
  пересылает уже подписанную ссылку (новых прав не даёт). Холст редактора (Konva)
  грузит напрямую — не трогали.

## Урок на будущее
Картинки из ЧУЖОГО origin внутри инлайн-SVG в Safari — **сразу проксировать через
свой домен**, не возиться с `crossorigin`/`xlink`. Это снимает весь класс проблем
(CORS + кэш) разом. Портрет/QR/декор в одном SVG грузились, а фон нет — это и
вывело на кэш-конфликт, а не «CORS вообще».

## Что осталось (Сергей просил НЕ чинить)

  ~4330, модалка ~3848) — у Сергея пустая. Тот же путь данных

# Смена дизайна в редакторе — V1, Этапы 1–3 в проде (25.06.2026)

Новая фича: кнопка «Сменить дизайн» в редакторе альбома. Подход — **remap
(per-spread), НЕ rebuild** → ручные правки (тексты, кадрирование) сохраняются.
ТЗ и прогресс — `docs/tz-design-switch.md`. Модуль — `lib/design-switch/`.

## Что переносим / что из нового дизайна
- Переносим: контент `data[<label>]` (тексты, фото), кропы
  `__scale__/__offset__/__rotate__`, `__hidden__` (скрытие пустого слота — про
  контент, не геометрию; иначе пустые слоты вылезают после смены).
- Из нового дизайна: стили текста (`__fontSize__/__color__/__font__/__halign__/
  __valign__`), рамки, фоны, расположение слотов. `__pos__` — роняем (привязан к
  координатам старого мастера). Распределение НЕ пересчитываем. Обложки не трогаем.

## Коммиты
- **Этап 1 — coverCheck** (`d0f21d5`): `lib/design-switch/compatibility.ts`
  (`findReplacementMaster` по page_role+slot_capacity — переиспользуется remap'ом;
  `checkCoverage`) + `cover-check.ts` (загрузчик). Вариант B — по ТЕКУЩИМ
  разворотам (`album_layouts.spreads`), не пере-прогоном движка. Тексты отказа
  правило «нужное N + максимум M»: «…страница-коллаж из 5 фотографий, а в этом
  дизайне максимум 4 фотографии в коллаже…». `photos_friend` — мягкая ёмкость.
- **Этап 2 — remap** (`d6175f4`): `remapData` (template-replace) расширен
  аддитивным параметром `contentKeyPrefixes` (дефолт = прежнее поведение).
  `lib/design-switch/remap.ts` (ядро `remapAlbumSpreads`, кроп-онли+`__hidden__`)
  + `remap-album.ts` (загрузчик; `write:false`=DRY-RUN; `write:true` пишет
  `album_layouts.spreads`+`template_set_id` и `albums.template_set_id`;
  `has_user_edits` не трогает; фото КАК ЕСТЬ — пере-подпись отдельный долг).
- **Этап 3 — UI** (`cffabf3`): `/api/layout` действия `design_switch_check`
  (read-only) + `design_switch_apply` (пишет, guard как у save_album_layout,
  внутри повторный coverCheck = двойная защита). `DesignSwitchModal.tsx` + кнопка
  в шапке редактора (текущий дизайн помечен «Текущий» и заблокирован; перед apply
  досылаются несохранённые правки; после — reload). Прокликано локально.

## Проверки на проде (read-only / dry-run, БЕЗ записи)
- coverCheck «Тест 24» (дизайн «Акварельные мечты», 8 разв.): текущий → OK;
  «Белый» → отказ по 5-местному коллажу (у «Белого» только J-Collage-4) —
  **настоящий отказ**.
- apply DRY-RUN: «Белый» → ok=false/written=false (guard работает); сам в себя →
  8/8 remapped, lost:0, без записи.

## ОСТАЛОСЬ / зависимости
- На проде всего **2 дизайна** (Акварельные 24 мастера / Белый 37), совместимых
  пар НЕТ → happy-path между РАЗНЫМИ раскладками вживую не проверен.
- ⚠️ **Чистота переноса зависит от ПОЛНОТЫ наборов мастеров.** Нет точного
  близнеца типа → `findReplacementMaster` берёт ближайший достаточный → раскладка
  меняется (пример учителей: «Акварель» только на 6, «Белый» 9/12/16 + неточный
  тег F-Head-SmallGrid). Это не баг remap, лечится доработкой наборов.
- **Блокеры выравнивания дизайнов:** доработка мастеров «Акварели» (задание
  дизайнеру готово) + фикс «Белого»/**J-Collage-5** (бэклог).
- **Первая боевая запись** (design_switch_apply write) — на «Тест 24» по явному
  «ок» Сергея, после выравнивания. **Этап 4** (фоны нового дизайна в превью/
  экспорте, пере-подпись фото, граничные случаи) — делать на реальном кросс-
  дизайн переносе. В проде деструктивного пути нет (совместимых пар нет).
