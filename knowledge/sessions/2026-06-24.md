## Фикс блокера «серые фото» (Timeweb)
- **Корень:** `storageKeyFromUrl` (`lib/pdf-export/photo-embed.ts`) срезал имя
  бакета только для яндексовых хостов (`storage.*`). Для Timeweb path-style
  (`s3.twcstorage.ru/<bucket>/<path>`) бакет НЕ срезался → ключ не совпадал с
  `urlToFilename` → **оригиналы НИКОГДА не находились** (`no_original`) → экспорт
  падал на запасной **selection-WebP по подписанной ссылке (TTL 24ч)** → серые/
  низкокачественные фото, недетерминизм.
- **Фикс:** детектить path-style и для `*.twcstorage.ru` → срезать бакет.
  Проверено: на боевых данных старая логика 0/8 оригиналов, новая 8/8; прод-рендер
  через очередь — `no_original=0`, фото из оригиналов (подтверждено растеризацией
  PDF двумя движками pdfjs + Apple PDFKit + глазами на проде).
- **Улика Сергея «ZIP ок / PDF серо» — структурная, не случайная:** после фикса
  развороты в обоих путях берут оригиналы; остаточные warning'и типографии — это
  **обложки** (`back_logo`/`logo.webp` не загружен + `cover_*_under off-page`),
  которые PDF вообще не рендерит. Отдельное ТЗ.

## Грабли (важно)
- **postgrest.conf хранил СТАРЫЙ пароль gen_user.** После смены пароля (прошлая
  сессия) обновили `.env.local` и `backup.env`, но НЕ `/etc/postgrest/postgrest.conf`.
  Работающий PostgREST держал старое соединение; `systemctl restart postgrest`
  (делал для reload схемы под export_jobs) переподключился старым паролем → крэш →
  **сайт 502 на ~2 мин**. Фикс: пароль в conf. Урок: пароль gen_user живёт в ТРЁХ
  местах (`.env.local`, `backup.env` на VDS, `postgrest.conf` на VDS); для reload
  схемы лучше `systemctl reload postgrest` (не restart). См. [[project_offsite_db_backup]].
- **root-овый `shared/.env.production` ломает автодеплой.** Менял env под root (тюнинг
  порога для UX-теста) → файл стал root:root → сборка деплоя под юзером `yearbook`
  не смогла прочитать (`EACCES`). Плюс ручная пересборка под root оставила root-файлы
  в релизе (`rm` под yearbook → Permission denied). Фикс: `chown yearbook:yearbook
  shared/.env.production` + чистка релиза. **Урок: НЕ трогать под root файлы/папки,
  которыми владеет `yearbook` (shared, releases).** Прод не падал — защита деплоя
  (не переключать при ошибке сборки) сработала.
- Локальный `.env.local` смотрит на СТАРЫЙ Supabase (не Timeweb), а боевой
  PostgREST Timeweb закрыт снаружи → очередь/воркер тестируются только на сервере
  (или временным клоном ветки в `/tmp` против Timeweb).
