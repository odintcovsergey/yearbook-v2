# DECISIONS — ключевые решения проекта (с датами)

Журнал значимых архитектурных и продуктовых решений. Новые — сверху.
Полная летопись подэтапов — в `yearbook-context-vN.md`; здесь только то, что

## 19.06.2026 — Адаптация макета под формат типографии (v199)
- Адаптация мастера под формат заказа — только **uniform-scale** (равномерно),
  по меньшему коэффициенту work-зоны; контент центрируется, фон рисуется навылет.
  Растяжение (stretch) НЕ делаем.
- Адаптируем только **внутри одного семейства пропорций** (vertical_rect / square
  / horizontal). Между семействами (квадрат ↔ прямоугольник) — НЕ адаптируем,
  показываем предупреждение.
- Архитектура: адаптируем **сам мастер одним чистым преобразованием** перед
  подачей в холст (как `layoutCover` у обложки). Тогда превью и редактор
  подхватывают формат автоматически, координаты/кроп не ломаются.
- Ядро вынесено в `lib/format-adapt/` (чистые функции + тесты).
- Семейство дизайна — поле `format_family` (nullable, обратносовместимо: без него
  считается из пропорций). Миграция применяется вручную.

## 19.06.2026 — Обложки и типография (v186–v198)
- Обложка — отдельная система рендера (back + КОРЕШОК + front); корешок между
  страницами форматом НЕ масштабируется → нужна своя математика (адаптация обложки
  под формат осталась отдельным шагом после v199).
- Парсер обложек распознаёт единое широкое полотно как одну страницу (v186).
- Сущность «Типография» (`printers`): типы листов + диапазоны корешка (v189),
  профиль типографии с полями печати (v197). Заказ хранит `format_id`.
- Персональный раздел: признак `is_personal` + нарезка на per-student книги (v198).

## 17.06.2026 — Личный раздел и каталог мастеров (v181–v185)
- Конструктор личного раздела в структуре, несколько личных разделов, `multi_spread`
  (авто-коллажи + ручной выбор раскладок).
- Лимит фото «с друзьями» поднят 10 → 30.
- Вертикальная выключка текста из IDML + слоевой z-порядок декора.
- Расширение каталога мастеров (Quarter page-any, Collage 3/5/6, Teachers-3x2).

## Базовые продуктовые/архитектурные принципы (действуют постоянно)
- **Мультитенантность:** все БД-запросы tenant-aware через `auth.tenantId`
  (`lib/auth.ts` → `requireAuth`/`getAuth`).
- **Публичные страницы родителей/учителей неприкосновенны** — только аддитивные
  правки, логику отбора/submit/draft не менять:
  `/[token]`, `/teacher/[token]`, `/ref/[token]`, `/invite/[token]`, `/t/[slug]/[token]`.
- **Секреты только в env** (Vercel / `.env.local`), никогда в код и коммиты.
- **Сохранение данных и кроп** работают на оригинальных координатах; адаптация
  формата — только для отображения.
- **CSV-экспорт** (`/api/tenant` export_csv, `/api/admin` export) — контракт со
  связанным проектом автовёрстки InDesign; колонки менять согласованно.
- Этап 4.c (редирект /admin → /app) и 4.d (удаление legacy) — только по явной
  команде Сергея.
