> Причина многослойная: (1) `covers.background_url` хранит старый западный supabase-URL

> same-origin прокси `app/api/img/route.ts`:** все `<image>` превью заворачиваются в
> `/api/img?u=<подписанный-url>` → картинки идут с нашего домена → для браузера это
> «свои» → ни CORS, ни кэш-граблей. Прокси: только image/*, белый список хостов (анти-SSRF),
> пересылает уже подписанную ссылку. Холст редактора (Konva) грузит напрямую — не трогали.
> Прокликано Сергеем в Safari. УРОК: cross-origin картинки в инлайн-SVG → сразу проксировать
> через свой домен. Хвост (просил не чинить): кнопка «Превью обложки»-глазик в настройках

## Что это за продукт

Платформа для школьных фотографов: от сбора фото у родителей (вход по уникальной
ссылке) → автовёрстка альбома из учеников/учителей/общих разделов → редактор
разворотов и обложки → экспорт в PDF. Плюс кабинет партнёра, супер-админка,
обложки, типографии, реферальные программы, «Идеи».

## Стек

- **Frontend/Server:** Next.js 14.2 (App Router), React 18, TypeScript 5
- **БД/Storage:** Supabase (Postgres + Storage). Миграции применяет Сергей вручную
  через Supabase Studio → SQL Editor.
- **Канвас-редактор:** Konva 10 + react-konva
- **PDF-экспорт:** pdf-lib + @pdf-lib/fontkit + sharp
- **IDML-парсинг:** fast-xml-parser (preserveOrder)
- **Хранилище оригиналов/PDF:** Yandex Cloud S3 (@aws-sdk/client-s3 + presigner), приватный бакет + signed URL
- **Стили:** Tailwind 3 · **Тесты:** Vitest 4 (~1091 passed на 19.06)
- **Деплой:** Vercel (auto-deploy на push в `main`), тариф БЕСПЛАТНЫЙ (Hobby)
- **Git:** двойное зеркало — GitHub (origin) + GitFlic, push идёт в оба разом

## Известные ограничения / технический долг

- **Экспорт PDF синхронный** — главный риск. Падает по таймауту Vercel (300с) уже на
  ~27 разворотах (рендер ~5–6 с/страницу, фото грузятся последовательно). Большие
  альбомы (80+ разворотов) не собираются. Лечение: ускорить рендер (параллельная
  загрузка фото) и/или фоновая очередь рендера на Yandex Cloud. См. `DECISIONS.md`
  и `docs/tz-load-readiness.md`.
- **Экспорт не учитывает формат/формат файла типографии** (масштаб в файлы, JPG
  постранично) — отдельное «ТЗ экспорта», ещё не сделано.
- Серверный popup «собранная обложка» (cover_album_preview) не адаптирован под формат.
- Нет staging/Sentry; тестовая среда = preview-ссылки Vercel (план — dev-ветка + тестовая БД к приходу реальных партнёров).
